# GDD-028 — 格子战术战斗（Grid Tactical Battle）

> **Status**: Draft（2026-07-20；原型 prototypes/grid-battle 验证手感通过）
> **Governing ADR**: ADR-0018（确定性格子战术战斗引擎，supersede ADR-0012）
> **取代**: GDD-021（区域战斗）的战场/交战模型；复用 GDD-010/011/012/001/007/019

## 1. Overview（概述）

格子战术战斗是本游戏的核心差异化战斗玩法，取代命名区域战斗（GDD-021）。战场是一张**有真实地形的整数格子地图**（山/隘口/林地/粮仓/平地）。玩家作为最高指挥，为每支部队**设定目的地**，然后按**时间段（黎明/白昼/黄昏/夜）逐段推进**；部队按**兵种速度**沿寻路行军，骑兵快、步弓慢。行军途中**半路遭遇敌军会暂停让玩家临机抉择**；每个时间段，**相邻格的敌我互相交战**。地形与时机赋能策略：**藏兵林中诱敌入隘伏击、绕后焚敌粮仓断其补给**。**补给是硬约束**——充足则逐段回血，告急则部队叛逃、且强攻/被打伤亡数倍。胜负由布势、补给、时机与地形利用决定，而非数值堆叠。

## 2. Player Fantasy（玩家幻想）

"我是运筹帷幄、临阵决机的主将。" 玩家不是看一个已定结局被念出来，而是**在阵中随机应变**：诈败诱敌深入隘口再伏兵四起；派一支轻骑绕到敌后一把火烧了粮仓，看着敌军因断粮而哗变叛散；择夜色掩护突袭；补给将尽时决断是撤是拼。每一张地图因地形不同而有不同解法，同一张地图有多种打法。

## 3. Detailed Rules（详细规则）

### 3.1 战场与地形
- 战场为 `W×H` 整数格子网格（MVP 建议约 14×9）。每格一种 `TerrainKind`：
  - **平地**：可通行，无特性。
  - **山**：不可通行（逼出隘口这一唯一通道）。
  - **隘口**：可通行的狭窄通道；伏击的触发地形。
  - **林地**：可通行；单位在林中为**藏兵态**（反全知：未被侦破不入敌可见面），且是伏击发起地形。
  - **粮仓**：可通行的补给源，分属我/敌；焚毁或被断则该方补给下滑。
- 地图数据驱动，支持多张（不同地形布局→不同策略）。

### 3.2 部队与兵种
- 每支部队有：所属方、`TroopType`、位置（格）、兵力（整数）、目的地、姿态、存活。
- 兵种（数据驱动，示例值见 §7）：
  - **骑兵**：速度高（如 3 格/段），近战射程 1。
  - **步兵**：速度低（1 格/段），近战射程 1，兵力厚。
  - **弓兵**：速度低（1 格/段），**射程 2**（隔格放箭、不吃近战反击）。

### 3.3 时间与推进
- 战斗时钟用世界钟（GDD-001）：`WorldDay` + `DaySegment`（黎明→白昼→黄昏→夜→次日黎明）。
- **一次"进行" = 推进一个时间段**。每段依序结算：移动 → 遭遇判定 → 地形/时机涌现（伏击/火攻）→ 相邻交战 → 补给更新 → 回血/叛逃 → 存活/胜负判定。
- **夜**段：近战突袭加成（夜袭）。**白昼/黄昏**：可火攻（可视纵火）。

### 3.4 移动与寻路
- 玩家在布阵/行军阶段为部队**设目的地**（命令，前置校验：目的地须可通行）。
- 每段推进，部队按**速度**沿**确定性整数寻路（BFS）**向目的地移动，绕开不可通行格与被占格；骑兵一段可能即到，步/弓需 2-3 段。
- 到达目的地即停（据守）。行军中可改派目的地。

### 3.5 半路遭遇决策点
- 行军中的部队若与敌**新相邻**，暂停并给玩家抉择：**继续突进**（照原目的地打过去）/ **就地据守**（停在此处结阵）/ **后撤避战**（退一格避战）。
- 这是"当下参与感"的核心接缝——诈败诱敌、临机应变由此发生。

### 3.6 交战（相邻格结算）
- 每段末，**相邻（含斜角）的敌我单位互相造伤**。弓兵按射程可对射程内敌造伤而不吃近战反击。
- 伤害整数结算；多单位同时计算后统一写回，按 `(side, unitId)` 规范序（确定性）。

### 3.7 地形/时机涌现（兵法，非按钮）
- **伏击**：单位处**林地**、且相邻**隘口**中有敌 → 对该敌**伏击加成**（大幅倍率）。
- **火攻**：单位相邻敌**粮仓** + 时段为白昼/黄昏 → 焚仓（敌补给暴跌，粮仓标记焚毁）。
- **夜袭**：夜段近战伤害加成。
- 门不齐不成型、不提示（不剧透未成型）。

### 3.8 补给（硬约束）
- 双方各持**补给度**（整数 0–100）。每段：守住己方粮仓（未焚、无敌相邻）→ 回升；被焚/被断 → 下滑。
- **补给充足（≥ 高阈）** → 单位逐段少量回血（不越上限）。
- **补给告急（< 低阈）** → ① 部队**叛逃**减员（接 GDD-006 忠诚/凝聚）；② 该方单位**战斗伤亡按倍率放大**（强攻/被打数倍伤亡，接 GDD-011 士气疲劳）。
- 粮仓与补给线为**地图战略目标**（落地"绕后断粮"）。

### 3.9 反全知
- 玩家对敌只见情报投影（位置/兵力估计随侦察时效，GDD-007）；林中藏兵未被侦破不入敌可见面。类型层玩家可见结构不含敌真值。

### 3.10 胜负
- 一方单位全灭（战死+叛散）→ 对方胜；或达成地图目标（如夺占/守住关键格）。攻守统一，仅初始阵营角色不同。

## 4. Formulas（公式）

> 所有权威计算为**整数/定点**；乘子用整数百分比（`dmg * pct / 100`），不引入 float 到状态哈希路径（ADR-0004/0018）。

- **移动步数/段** = `min(speed, 沿寻路到目的地的剩余步数)`；寻路 = 4 向 BFS 避障避占。
- **单体基础伤害** `base = atk`。
  - 伏击：`base = base * AmbushPct / 100`（示例 250）。
  - 夜袭（近战 range=1，夜段）：`base = base * NightPct / 100`（示例 125）。
- **受伤方结算**：`taken = Σ base(来袭者)`；若受伤方补给告急：`taken = taken * StarvePct / 100`（示例 300）。`hp -= taken`。
- **补给更新/段**：`supply += healthy ? +Replenish : -Drain`（示例 +5 / −7），钳 [0,100]。`healthy = 己仓未焚 ∧ 无敌相邻己仓`。
- **火攻焚仓**：敌相邻己仓 + 白昼/黄昏 → 对方 `supply -= BurnLoss`（示例 38）、`burned=true`。
- **回血/段**（补给 ≥ HighThr，示例 50）：`hp = min(max, hp + Recover)`（示例 5）。
- **叛逃/段**（补给 < LowThr，示例 25）：`hp -= Desert`（示例 6）；到 0 记叛散。

## 5. Edge Cases（边界）

- **目的地不可通行**：设目的地命令拒绝（稳定错误码），不写入。
- **寻路无解**（被围死/全阻）：单位原地待命，不瞬移、不穿障。
- **目的地被占**：寻至相邻停；不叠格。
- **同段多方互破关键**：按 `(side, unitId)` 规范序 + 同时写回，结果可复现。
- **补给恰在阈值**：用半开区间（`≥ HighThr` 回血、`< LowThr` 叛逃），中间带无效果，避免抖动。
- **粮仓已焚再被"烧"**：幂等，不重复扣。
- **单位在林中同时相邻多个隘口敌**：伏击对每个满足门的敌各自结算（按规范序）。
- **夜段火攻**：不触发（火攻限白昼/黄昏）；夜段走夜袭。
- **战中存档续战**：`GridBattleState` round-trip 一致、确定性哈希不变。

## 7. Tuning Knobs（可调参数）

数据驱动配置（构建期不可变，ADR-0003）：
- **网格**：宽 W、高 H、地图地形布局、双方粮仓坐标。
- **兵种表**：每种 `speed`、`range`、初始 `atk`/`hp`。示例：骑(速3,程1)、步(速1,程1,兵厚)、弓(速1,程2)。
- **涌现倍率**：`AmbushPct`(250)、`NightPct`(125)。
- **补给**：`HighThr`(50)、`LowThr`(25)、`Replenish`(5)、`Drain`(7)、`BurnLoss`(38)、`Recover`(5)、`Desert`(6)、`StarvePct`(300)。
- **时间**：段序（DaySegment）、火攻可用时段、夜段定义。
> 用户裁定"数值都好说"——上列为原型验证值，正式版按平衡套件调。

## 8. Acceptance Criteria（验收标准）

1. **确定性**：同一初始 `GridBattleState` + 同一命令序，连推 N 段 → 同一状态哈希（可回放）。
2. **寻路**：单位从一侧到另一侧会绕山、经隘口；无解时原地待命，绝不穿障/瞬移。
3. **兵种定速**：骑兵到达目的地所需段数 < 步/弓（同距离）。
4. **伏击涌现**：林中单位相邻隘口内敌 → 伤害按 `AmbushPct` 放大；条件不齐不触发、不打标签。
5. **火攻断粮**：单位相邻敌粮仓 + 白昼/黄昏 → 敌补给按 `BurnLoss` 暴跌、粮仓焚毁（幂等）。
6. **补给硬约束**：补给 ≥ HighThr 逐段回血；< LowThr 逐段叛逃减员且受伤 ×StarvePct。
7. **半路遭遇**：行军单位与敌新相邻 → 产生决策点，三种抉择各自生效（继续/据守/后撤）。
8. **段序交战**：每段相邻敌我按规范序、同时写回结算，结果可复现。
9. **反全知**：类型层玩家可见结构不含敌真实兵力/林中藏兵（未侦破）。
10. **存档续战**：战中 `GridBattleState` 存读档 round-trip 一致、哈希不变。
