# GDD_014 — 战役与生涯

- 状态：Draft
- **Status**: Draft
- 范围：Meta（连接各战役为一段可持续人生；跨切片）

## System Purpose

把单场战役连接成一段**可持续的人生**。定义玩家从“城池太守”开局后，如何通过功绩、名望、关系与实战，沿**忠臣晋升**或**自立反叛**两条线延续游戏。本层是“可玩性 / 连接性 / 持续性”的来源——它让战斗的胜负、城池的经营、关系的积累**有处可写回、有目标可追求**。无本层，则各系统的“写回”全是浪费，游戏退化为一次性战斗 demo。

## Player Fantasy

“我从一座小城的太守起步，靠一场场仗打出来的战功和人脉，一步步爬到能左右天下的位置——或者，在旧主昏聩时，带着死心塌地的部下脱离他、自立为王。我的位置是我挣来的，不是系统发的。”

## Core Loop

开局守城承压 → 战后治理城池、发育资源人才 → 完成君主任务 / 积累战功攒功绩名望 → 晋升掌权 / 扩张夺城 → 终极抉择（继承基业 / 自立争霸）→ 新局面写回，进入更大棋盘。

## Main Rules

- **固定开局身份：城池太守。** 玩家空降为某正规势力一城太守，开局城池匹配历史禀赋（详见 GDD_015 世界模型）。
- **开局守城事件**（强制）：进入即触发历史敌军围城（一场 GDD_010 战役）。
  - 胜 → 解锁全城权限、开启生涯、获君主初始信任与功绩。
  - 败 → 失城罢官、沦为在野，保留少量核心部曲；可重新投效或流浪发育。**败局必须生成合法可继续状态**（撤退/求和/失职/流亡/投效/东山再起之一）。
- **双线并存、可随局势切换，互不强制：**
  - **忠臣晋升线**：依附君主，靠**功绩 + 名望 + 君主好感**逐级晋升官阶，终点为大都督乃至继承基业。
  - **自立反叛线**：不依赖官阶，靠**实战硬实力 + 僚属好感**，满足条件即可发动自立；结局由关系网决定。
- **功绩（Merit）**：作战胜利、完成君主任务、治理城池均可累积；是**忠臣线晋升的硬性指标**。
- **名望（Renown）**：大型战役胜利、平定叛乱、招揽贤才、治理盛世提升；影响**人才登用、部下信服、自立号召力**，且是晋升的软门槛。
- **君主好感（Lord Standing）**：独立于部下好感，由任务完成度、战功、治理质量决定；只影响**忠臣线晋升速度与授权**。
- **官职晋升梯队**（从太守起，逐级解锁带兵上限 / 治理范围 / 战略参与权）：
  1. 城池太守（单城，基础兵权/内政权）
  2. 资深太守（多城治理，可调配相邻城池兵力）
  3. 州刺史（一州数城，自主编制中型军团）
  4. 四方中郎将（势力核心将领，参与战略决策）
  5. 镇国将军（独领战区，独立作战调度权）
  6. 副都督（执掌部分主力军团）
  7. 大都督（总领全军，军政二把手）
  8. 继承基业（功绩/名望/民心/全军好感拉满，老君主病逝/退位后继承势力——忠臣线终极结局）
- **自立触发**（满足其一即可发动）：① 手握 ≥3 座自主掌控城池且兵粮达标；② 名望极高且核心文武好感普遍偏高；③ 主君昏庸、连续贬谪、任务苛刻触发割据剧情。
- **自立结局由好感分布决定**（关系网是核心变数）：
  - 全员高好感 → 文武集体拥立，无内乱，完整继承城池/兵力/人才，脱离原诸侯体系成立新势力。
  - 部分低好感 → 半数跟随，部分旧臣归降原主，损失部分兵力城池，开局需平乱御敌。
  - 全员低好感 → 众叛亲离，城池倒戈、兵力溃散，仅带少量亲卫沦为流浪势力，难度拉满。
- **自立后**：与旧主及诸侯转为敌对/中立，不再接君主任务，自定国策；可走 GDD_015 的历史分叉（架空线），以统一乱世为终极结局。

## Formulas

> 所有阈值、权重、映射来自版本化配置（ADR-0003）；本层结算确定性（ADR-0004），纳入状态哈希与存档 DTO（ADR-0005）。

### 变量定义

| 变量 | 含义 | 范围/单位 | 来源 |
|---|---|---|---|
| `merit` | 累计功绩 | ≥ 0 整数 | CareerState |
| `renown` | 名望 | ≥ 0 整数 | CareerState |
| `lord_standing` | 君主好感 | [0,1] 定点 | CareerState |
| `rank` | 当前官阶（0–7） | 枚举序数 | PromotionLadder |
| `affinity[i]` | 第 i 名僚属对玩家好感 | [0,1] 定点 | RelationshipState(GDD_006) |
| `cities_owned` | 自主掌控城池数 | ≥ 0 整数 | 世界模型 |

### 1. 忠臣晋升门槛（逐级）

```
can_promote(rank→rank+1) =
    merit ≥ merit_req[rank+1]
  ∧ renown ≥ renown_req[rank+1]
  ∧ lord_standing ≥ standing_req[rank+1]
```

- 任一不足则不可晋升；三项的 `*_req[]` 为按阶配置的递增数组。

### 2. 自立可发动判定

```
can_rebel =
     (cities_owned ≥ rebel_city_min ∧ supply_ready ∧ troops_ready)
   ∨ (renown ≥ rebel_renown_min ∧ avg(affinity) ≥ rebel_affinity_min)
   ∨ lord_oppression_flag
```

### 3. 自立结局分支（关系网决定）

```
loyal_ratio = | { i : affinity[i] ≥ defect_threshold } | / N
loyal_ratio ≥ hi   → 全员拥立
mid ≤ loyal_ratio < hi → 部分跟随
loyal_ratio < mid  → 众叛亲离
```

### 4. 官阶 → 兵权上限映射

```
troop_cap = base_cap × rank_cap_multiplier[rank] × city_support_factor(cities_owned, governance)
```

- 高阶解锁更高带兵上限，但俸禄消耗同步上升，需城池经济（GDD_004）支撑。

## Data Model

- `CareerState`：merit、renown、lord_standing、rank、所属势力、在野标志。
- `PromotionLadder`：官阶定义、各阶 `*_req` 门槛、带兵上限/治理范围/授权。
- `RetinueState`：核心部曲与僚属列表及其好感（引用 GDD_006）、可任免官职（城守/副将/内政主事/军师）。
- `RebellionState`：自立触发标志、发动时好感快照、结局分支、新势力初始态。
- `LordMissionLog`：已下发/完成/失败的君主任务及其奖惩。

## Player Inputs

接受/拒绝君主任务；分配城池治理与征兵；任免僚属官职；申请晋升（满足门槛时）；发动自立（满足条件时，带二次确认）；在野时选择投效目标或流浪发育。

## System Outputs

功绩/名望/君主好感变化；晋升/降职事件；自立结局与新势力建立；带兵上限/治理范围/授权变化；写回 GDD_005 身份变化、GDD_006 关系后果、GDD_004 城池控制权、GDD_015 世界态势。

## Dependencies

依赖 GDD_004（城市资源/控制权）、GDD_005（人物身份）、GDD_006（关系/好感/授权）、GDD_010（战役胜负产功绩名望）、GDD_015（开局禀赋与世界态势写回）、GDD_013（生涯状态存档）。被 GDD_016（敌方 AI 读势力态势）消费。

## Edge Cases

- 晋升门槛达标但君主势力已灭：转为在野，触发投效/自立选择。
- 自立发动瞬间好感临界：按结算时快照判定，发动后好感变化不回溯改分支。
- 在野期间无城无僚属：进入“流浪发育”最小态，仍可投效或经战功重建。
- 双线切换：忠臣可随时转自立；自立后不可回到原主忠臣线（关系已敌对）。
- 继承基业后玩家成为君主：晋升线终止，进入势力君主玩法（与自立后玩法合流）。

## Failure Cases

任务失败扣功绩、降君主好感、暂停晋升权限、轻微降辖区民心。自立失败（众叛亲离）不是卡死——沦为流浪势力是合法可继续状态。非法操作（门槛未达却申请晋升/自立）被拒，返回稳定错误码，无部分写入。

## Balancing Parameters

各阶 `merit_req/renown_req/standing_req`；自立 `rebel_city_min/renown_min/affinity_min`；结局分支阈值 `hi/mid/defect_threshold`；官阶兵权倍率与俸禄；任务奖惩数值；名望各来源权重。全部配置化并带范围。

## UI Requirements

显示生涯面板（官阶、功绩/名望进度、君主好感、距下一阶差距）；僚属好感与可任免官职；自立条件清单与当前满足度（达标项高亮）；自立发动前展示**结局风险预览**（按当前好感分布提示拥立/部分/众叛倾向，不泄露精确隐藏值）。

## AI Requirements

敌方/友方势力的官职与势力态势由 GDD_016 与 GDD_015 驱动；本层只提供玩家侧生涯结算。AI 君主对玩家的任务下发与信任变化使用可解释规则，不偷看玩家隐藏状态。

## Save / Load Requirements

保存 CareerState、RetinueState 好感快照、RebellionState、LordMissionLog、官阶与授权；round-trip 一致；与 GDD_015 世界态势、GDD_010 战役状态同一存档边界（GDD_013）。

## Test Requirements

晋升门槛逐级判定（达标/未达各项独立可验）；自立三分支由好感分布确定性产出；在野→投效→重建可继续；任务奖惩写回正确；同状态→同生涯结算（哈希一致）；存档 round-trip。

## MVP Scope

先实现：太守开局 + 守城开局事件胜败后果 + 功绩/名望两数值 + 忠臣线前 2–3 阶晋升 + 自立**触发判定与三分支结局**（结局后玩法可简化）。暂不做完整 7 阶全部权限差异、复杂任务库、继承基业后君主玩法。

## Future Scope

完整 8 阶梯队与各阶授权差异；丰富的君主随机任务库；继承基业/自立后的君主级国策与争霸；多势力并立的长线生涯；联姻/家族/师承等关系延伸。
