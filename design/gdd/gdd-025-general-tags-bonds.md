# GDD_025：武将标签与羁绊体系（General Tags & Bonds）

> **Layer**: Feature（人物层——把"条件涌现 + 反全知 + 人心杠杆"从主角/兵法推广到每一员武将）
> **Status**: Draft（2026-07-05）
> **Governing ADR**: 内联裁定复用既有基座（无需专属 ADR）——ADR-0004（确定性）· ADR-0006（种子化/反全知）· ADR-0011/0012（兵法条件/区域战斗）· ADR-0014（人心杠杆）。〔注：ADR-0015 已另作纪元与世界快照模型（GDD_026），非本篇契约。〕
> **消费/被消费**: GDD_005 人物 · GDD_006 关系 · GDD_010/021 兵法条件涌现 · GDD_024 人心杠杆 · GDD_014 生涯/自立 · GDD_015 事件（时代分段）

## 1. System Purpose（系统目的）

废除"统率/武力/智略/政治/魅力"数值面板。**武将 = 一束会在特定条件下发作的气质标签 + 一个藏着的心（野心/忠诚倾向）+ 一张羁绊网**——不是一张数值卡。玩家侧**看不到任何数值**，只靠**名声、事迹、情报**识人；系统侧由**标签→条件化效果**驱动，全部长在既有"条件涌现"引擎上。此为本作区别于三国志/Total War 的人物层根本设计。

## 2. Player Fantasy（玩家幻想）

"我不是在读一张吕布=武力100的卡。我知道吕布**反复无信**——所以他守的城，我不必强攻，散布流言许以厚利，他多半临阵倒戈；我也知道他【莽勇】，正面硬碰谁都吃亏。识人靠的是他是个什么样的人，不是一个数字。"

## 3. Core Loop（核心循环）

侦察/风闻得知某将的**名声与标签**（反全知，可不全）→ 据其气质判断如何用他/对付他（正面?攻心?伏击?）→ 战场上标签在**天时/地利/人和**条件齐备时**发作**（增益/连锁/负面）→ 隐秘的心（野心/忠诚）决定他能否被策反、会否自立 → 羁绊在同场时触发协同或崩解 → 战果与时代推进**改变其标签**（晚年多疑、失荆州而自负）。

## 4. Main Rules（主要规则）

- **R1 无数值面板（强制红线）**：玩家侧不显示任何 0–100/百分比能力数。武将以**名声档 + 标签 + 已见事迹**呈现。情报越浅，所知标签越少（反全知，GDD_007）。
- **R2 气质标签 = 条件化效果**：每将带若干标签，每个标签是**{触发条件 + 效果}**，而非常驻加成。触发条件复用既有门（天时/地利/人和/敌态）。门不齐则不发作——与火攻/伏兵同理（GDD_010/021）。
- **R3 隐秘的心（不显示）**：每将有**忠诚倾向**与**野心**两项**定性档**（如 忠义/摇摆/怀贰；安分/有志/狼顾），**内部驱动人心杠杆的策反难度门与自立倾向**（GDD_024/014），**绝不暴露为数字**。
- **R4 羁绊**：血脉/师徒/知己/仇怨四类。同场作战触发**协同效果**（分摊伤害/回血/同增）或**崩解效果**（一人亡则余者狂怒不可控；仇怨同场则互扣）。接 GDD_006 关系。
- **R5 无全能 + 专精上限**：标签是**专精化**的（猛将无谋、谋士无勇）；一员将不会"样样标签拉满"。底色（隐性，见 §6）决定其可有哪类标签的上限。
- **R6 负面特质**：傲物/刚愎/嗜杀/优柔等**减益标签**，接对应系统（傲物→难招揽、刚愎→拒谏、嗜杀→占城掉民心、优柔→错失战机）。
- **R7 时代分段 + 地域**：同一人不同人生阶段标签不同（时代推进/历史事件可改标签，接 GDD_015）；地域标签（善水战/善骑）接兵种·地形杠杆（ADR-0011，非克制表）。

## 5. Formulas（结构，非数值面板）

> 权威整数/定点（ADR-0004）。**对玩家不可见**；仅内部判定用**定性档 → 门/乘数**。

- **F1 标签发作**：`标签.触发条件成立(天时,地利,人和,敌态) → 施加标签.效果`。效果为**条件化乘数/状态**（如"对无名之将增伤"、"下回合混乱"），沿用 §兵法条件涌现的状态机，不叠常驻数值。
- **F2 隐秘心 → 策反门**（喂 GDD_024）：`策反可乘度 = g(忠诚倾向, 野心)`——忠义档→近乎不可策反；怀贰+狼顾→高可乘。映射为 GDD_024 的 `SubversionTargetProfile`（忠诚/怨恨/野心字段），**由定性档确定性生成，非玩家可见数**。
- **F3 羁绊触发**：`同场(A,B) ∧ 羁绊类型(A,B)=t → 效果_t`；成员阵亡触发崩解效果（确定性）。
- **F4 底色 → 标签上限**（隐性）：底色档（勇/谋/仁/志四向的定性倾向）决定该将**可携带哪类标签**（无全能）。底色隐藏，仅生成默认标签集用。

## 6. Data Model（数据模型）

- `GeneralTag`（枚举/标识）：具名气质标签（如 Reckless莽撞 / Awe威压 / LoneValor孤胆 / Fickle反复 / Arrogant傲物 …），各绑定**触发条件谓词 + 效果**。
- `TagEffect`：条件化效果描述（战斗状态/乘数/连锁），进条件涌现状态机。
- `LoyaltyLeaning` / `Ambition`（枚举定性档）：忠诚倾向（忠义/安分/摇摆/怀贰）、野心（无争/有志/雄图/狼顾）。**内部**，不显示。
- `BondType`（血脉/师徒/知己/仇怨）+ `Bond(a,b,type)` + `BondEffect`。
- `EraStage`：人生阶段（青年/盛年/晚年）——决定当前标签集（接 GDD_015 事件切换）。
- `GeneralDossier`：一员武将的**档案**——具名标签集 + 隐秘心（倾向/野心）+ 羁绊引用 + 当前阶段 + 地域。**取代**任何数值 stat 卡。
- 玩家侧投影 `GeneralReputationView`：名声 + **已知**标签（受情报深度过滤，反全知）+ 事迹，**无数值**。

## 7. Player Inputs / System Outputs

- **输入**：侦察某将（得更多标签）；据标签决策（正面/攻心/伏击/招揽）。
- **输出**：名声档 + 已知标签 + 事迹（无数字）；战场标签发作提示（复盘可见"因【威压】故…"）；策反可乘的**定性提示**（"此人似有二心"，非概率数）。

## 9. Edge Cases（边界）

- 情报为零 → 只知名声，不知任何隐性标签/心（反全知，不报错）。
- 无羁绊成员同场 → 无羁绊效果（正常）。
- 标签触发条件永不齐（如无水域将【锦帆】）→ 该标签本局不发作（设计意图）。
- 时代切换使标签变更 → 旧标签在途效果结算完再换（确定性）。
- 负面标签与正面标签冲突（如【莽撞】+【威压】）→ 各按自身条件独立发作，不互相抵消（涌现叠加）。

## 10. Failure Cases（失败即可继续，红线）

标签/羁绊只改变战斗与人心的**条件与效果**，不产生死局：策反失败可继续（GDD_024）、羁绊崩解（狂怒不可控）仍是可继续战局。绝不卡死。

## 11. Balancing Parameters（平衡参数）

各标签效果强度（条件化乘数/状态时长）· 隐秘心定性档 → 策反门映射 · 羁绊效果强度 · 负面标签惩罚度 · 底色→标签上限表。
> **反支柱护栏（W5）**：标签是**撬动战局的条件**，非"点了就赢"的按钮；强将标签仍需**门齐（天时/地利/人和）**方发作，且受兵法沙盒整体护栏约束。列入平衡验证用例。

## 12. UI Requirements（UI 需求）

武将名声卡：中文名 + **已知标签**（图标+一句触发/效果说明）+ 事迹时间线 + 隐秘心的**定性风闻**（"传言此人反复"）。**全程无数值面板**。反全知：未侦知的标签留白/存疑标注。键鼠可达。

## 13. AI Requirements（AI 需求）

敌方 AI 同样由标签/隐秘心驱动（反全知不作弊，ADR-0006/0013）：会据己方将标签选战法、据玩家将隐秘心尝试策反。LLM 仅装饰（武将台词），不参与结算。

## 14. Save / Load Requirements（存档需求）

`GeneralDossier`（标签集/隐秘心/羁绊/阶段）+ 玩家已知标签投影纳入统一存档（ADR-0009/0005），round-trip 一致、确定性哈希（ADR-0004）。

## 15. Test Requirements（测试需求）

- 反全知：情报浅则已知标签少（同将，不同情报深度）。
- 标签发作门：条件齐发作、门不齐不发作（如【锦帆】无水域不发）。
- 隐秘心→策反：忠义档近乎不可策反、怀贰+狼顾高可乘（喂 GDD_024，端到端）。
- 羁绊：同场触发协同/崩解；成员阵亡触发狂怒。
- 时代分段：阶段切换改标签集。
- 无数值红线：玩家侧投影不含任何能力数（结构性断言）。
- 存档 round-trip：档案一致。

## 16. MVP Scope（MVP 范围）

一批代表性标签（~12–16）各接条件/效果 + 隐秘心两档接策反 + 羁绊四类各一效果 + ~40 名将档案 + 反全知投影。**不做**：全谱数百标签、复杂多段连锁全集、全武将穷尽档案（→ 内容管线）。

## 17. Future Scope（后续范围）

标签连锁全谱 · 全武将档案 · 玩家可塑造部将标签（培养）· 标签驱动的历史分叉深化 · LLM 生成武将风闻台词。
