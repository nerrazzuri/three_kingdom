# GDD_021 — 战场区域部署与区域战斗（Zone Deployment & Zone Battle）

> **Status**: Draft（2026-07-04；核心战斗执行层。实现 GDD_010「兵法沙盒」的条件涌现于**区域部署 + 回合调整 + 敌方区域AI**；首次落地 GDD_016 敌方AI；统一攻守。待 /review-all-gdds + ADR。）
> **Epic**: （待建）epic-031-zone-battle-core
> **关系**: 本文是 GDD_010 的**具体执行层**——GDD_010 定"兵法=条件涌现、非按钮/非坐标摆阵"的原则，本文定"这些条件如何在**战场区域**上、经**回合部署与调整**、与**敌方区域AI**博弈中涌现"。复用 GDD_011 士气疲劳、GDD_007 情报反全知、GDD_002 天气、GDD_008 军师、GDD_016 敌方AI、GDD_019/ADR-0011 六维准备。

## 1. System Purpose（系统目的）

把战斗从「一击结算 / 脚本竖切」升级为**可玩的核心战斗层**：玩家将**支队（将 + 兵种 + 兵力）分派到战场几处命名区域**（指范围非坐标），逐回合**动态调整**应对敌军，在某区**凑齐条件**时涌现兵法（伏兵/断粮/夜袭）取胜。与敌方**区域AI**博弈是深度来源。攻守统一。这是全游戏的**核心玩法**，须最完善。

## 2. Player Fantasy（玩家幻想）

「我把宿将与步骑弓分派到战场要地——伏骑于隘、屯粮断于敌后、步卒当正面。敌重兵压我正面，我便令隘口伏骑抄他侧后；他分兵来断我粮，我调预备去堵。这一胜是我**一回合回合布势、应变、在他露破绽时发动伏兵**挣来的——不是拖格子微操，不是兵种克制表，更不是骰子。这比三国志更能排兵布阵，因为拼的是**布势与应变的谋略**。」

## 3. Core Loop（核心循环）

```
战前部署：把支队（将/兵种/兵力）分派到战场各区域（军师按区给方法·指范围）
  → 【回合循环，直至终局】
     ① 玩家调整：支队于相邻区间调动（在途延迟）/ 投预备队 / 改姿态（主攻·佯攻·守）
     ② 敌方AI行动：按其区域视野（反全知）效用评分 + 种子softmax → 增援/反扑/佯动/设伏/收缩
     ③ 同步结算：各区交战 → 减员·士气·疲劳；条件逐回合累积（伏兵蓄势·断粮累时）
     ④ 涌现触发：某区条件凑齐 → 兵法发动（伏兵突袭/断粮见效/夜袭）→ 大幅冲击
  → 终局：破城（守方关键区崩）/ 攻方溃退 / 超时（攻方未克退兵）
  → 结算：占城归属（GDD_019 占城C）/ 功绩名望 / 后果（GDD_010 §后果）
```

## 4. Main Rules（主要规则）

**R1 战场 = 有限命名区域集**。MVP 固定 ~5 类通用区域，**预留每城/每战场景自定义**（数据驱动，Future）。每区带：地形（坚城/隘口/粮道/遮蔽/开阔）、条件禀赋（此区可涌现哪些 TacticCondition）、软容量、邻接关系。

**R2 部署 = 把支队分派到区域**。支队 = {主将?（可空）/ 兵种混合 / 兵力 / 姿态}。玩家**指定区域（范围）**，**不指定坐标/朝向**（守设计锁：区域非格子）。

**R3 回合制确定性推进**：回合 = 玩家调整 → 敌AI → 同步结算 → 涌现。回合数有上限（超时=攻方未克退兵，防冗长）。全整数/定点、无旁路随机（ADR-0004）。

**R4 战中调整（核心好玩点）**：每回合玩家可**有限重部署**——① 支队只能调往**相邻**区域且**在途 1 回合**（不能瞬移全军）；② 投入预备队；③ 改姿态。调整有**真实代价**（调动中暴露/失位/疲劳），杜绝"无限反悔"。

**R5 条件按区涌现**（复用 GDD_010 §7 条件涌现结算 + TacticCondition 原子）：某区凑齐其门（地形 + 兵种份额 + 将属性 + 时机/天气 + 已侦察 + **累计回合**）→ 该区兵法成型 → 结算加成/一次性特效。门不齐 → 不成型、不打标签（延续 TR-battle-002 负向不变量）。**兵法仍是条件涌现，非按钮**。

**R6 敌方AI区域博弈**（GDD_016 + ADR-0006，首次落地）：敌方按**自身区域视野**（AiWorldView 反全知，未侦知玩家伏兵则不知）对区域动作（增援区X/反扑区Y/佯动/设伏/收缩）做**效用评分 + 种子softmax 选择**（不可预测却可复现）。敌方**也会**设伏/断粮/反扑，具渐进记忆（记玩家上回合动作）。LLM 仅装饰不决胜负。

**R7 攻守统一**：同一区域战斗引擎。**守城**=玩家为守方在自方区域（城墙/瓮城/侧门/粮仓/预备）布防，攻方AI来攻；**出征**=玩家为攻方。对称。

**R8 反全知（双向）**：玩家只见敌区域的情报投影（估计值/时效，GDD_007）；隐蔽区（伏兵）若敌未侦知则敌不知，反之亦然。类型层玩家可见面不含敌真实兵力/部署。

## 5. Formulas（公式，结构）

> 权威整数/定点（ADR-0004）；系数进 §11 Balancing。

- **F1 区交战**：`zone_atk_power` vs `zone_def_power`，各 = Σ 支队(兵力×士气×(1+已成条件加成)×地形/姿态修正)（复用 GDD_010/019 战力式 + GDD_011 士气疲劳）。确定性比较 → 减员/推进量。
- **F2 条件回合累积**：伏兵突然性需连续 `N_ambush` 回合该区隐蔽未破；断粮达宽限需累计 `M_starve` 回合切断敌粮道区。累计中断则重置（确定性）。
- **F3 敌AI效用**：`U(action) = Σ w_i·factor_i`（区域战略价值 / 己方该区兵力 / 受威胁度 / 情报置信），`seed = Hash(worldTick, battleRound, factionId, actionId)`，`P = softmax(U/τ)`，注入式确定性流取样（ADR-0006 同款，非掷骰）。
- **F4 涌现触发**：区已成条件集 ⊇ 某 TacticTag 必要条件 → 发动，效果 = 一次性大幅战力/士气冲击（如伏兵突袭令敌该区士气骤降 + 攻方战力乘数）。

## 6. Data Model（数据模型）

- `BattleField`（区域集 + 邻接图 + 场景标注；MVP 固定模板，预留场景注入）
- `Zone`（id / `TerrainKind` / 条件禀赋集 / 软容量 / 邻接）
- `Detachment`（`OffensiveGeneral?` / `TroopComposition` / 兵力 / `Posture` / 所在区 / 在途目标区+剩余回合）
- `ZoneEngagementState`（该区双方支队 / 累计回合计数（伏兵·断粮）/ 已成条件集）
- `BattleClock`（当前回合 / 回合上限）
- `DeploymentCommand`（初始部署 / 调动 / 投预备 / 改姿态；前置校验、稳定错误码、无部分写入）
- `EnemyZoneAi`（`AiWorldView` 区域投影 + 效用评分 + 种子流 + 渐进记忆）
- `BattleResolutionState`（聚合权威态，可存档，确定性哈希）
- 复用：`TacticCondition`/`TacticTag`（GDD_010）· `OffensiveGeneral`/`TroopType`/`TroopComposition`（ADR-0011）· 士气疲劳（GDD_011）· 情报投影（GDD_007）· `WeatherType`（GDD_002）· `OpponentModel`/`AiWorldView`（GDD_016）

## 7. Player Inputs / System Outputs

- **输入**：战前部署（派支队到区）/ 每回合调整（调动/投预备/改姿态）/ 请军师建议 / 结束回合 / 战中存档。
- **输出**：各区态势（我/敌投影/条件进度）/ 涌现兵法事件 / 减员士气变化 / 终局判定 / 结算（占城/功绩/后果）/ 确定性哈希。

## 9. Edge Cases（边界）

- 支队调往非相邻区 → 拒（稳定错误码）。在途支队本回合不参与任一区结算（失位代价）。
- 某区无我方支队 → 敌可长驱（如空虚粮道被断）。
- 伏兵被侦破（敌侦察成功）→ 突然性条件重置、不成型。
- 超时未破城 → 攻方退兵（失败可继续，红线）。
- 战中存档后读档续战 → 状态哈希一致、AI 种子/记忆恢复、后续推进确定。
- 双方同回合互破关键区 → 定义确定性优先序（守方判定先于攻方，或按发起序）。

## 10. Failure Cases（失败即可继续，红线）

攻方战败/超时 → 折损退兵，必生成合法可继续状态（再备战/改守/转攻他城）。守方失城 → 走 GDD_014 防守后果，不切死局。绝不卡死。

## 11. Balancing Parameters（平衡参数）

> **打磨定值（2026-07-04，`ZoneBattleConfig.Default`）**——数据驱动，权威整数/定点（ADR-0004）。理据在下。

| 参数 | 字段 | 定值 | 理据 |
|---|---|---|---|
| 回合上限 | `BattleClock` | 6 | 防冗长，逼攻方"速破"（久拖则守方胜=退兵，红线可继续）。 |
| 区域数 | `BattleField` | 5（正面/粮道/侧翼/掩护/预备） | 够博弈纵深，不至散乱。 |
| 调动在途回合 | — | 1 | 调动有失位代价（在途不参战），防"无脑横跳"。 |
| 伏兵蓄势 `N_ambush` | `AmbushChargeRounds` | 2 | 突然性须连续隐蔽 2 回合，被侦破则重置。 |
| 断粮累计 `M_starve` | `StarveRounds` | 2 | 断粮达效须持续切断 2 回合。 |
| 每条件加成 | `ConditionBonusEach` | +15% | 成型兵法条件的战力增益（可叠加）。 |
| 基础减员率 | `AttritionRate` | 20% | 均势败方每回合损失；随战力比放大至 `AttritionCap`。 |
| 减员上限 | `AttritionCap` | 60% | 悬殊不瞬灭，但快速崩解。 |
| 败方士气跌 | `MoraleDropOnLoss` | 10%（上限 45%） | 留博弈余地，不单回合归零。 |
| **姿态战力** | `Assault/Hold/Feint Mod` | **1.25 / 1.10 / 0.75** | **W5 关键**：主攻战力最高（速破），坚守居中（据工事久持），佯攻最低（诱敌/掩护蓄势）。 |
| **姿态疲劳倍率** | `Assault/Hold/Feint FatigueMul` | **2.0 / 0.5 / 1.0** | 权衡代价：主攻耗力最快、坚守省力——速攻 vs 久持成真选择，无单一姿态占优。 |
| 每回合疲劳 | `FatiguePerRound` | 5%（×姿态倍率） | 疲劳基础增幅。 |
| **疲劳战力权重** | `FatiguePowerWeight` | **0.5** | 有效战力 = 名义×(1−疲劳×0.5)；令疲劳成真代价（久战侵蚀战力、疲兵可被翻盘），非死数值。 |
| **城防加成** | `FortifiedDefenseBonus` | **+35%** | 守方据坚固地形（城门正面）得工事加成——破坚城须真优势而非均势（W5）。 |
| 涌现士气冲击 | `EmergenceMoraleShock` | 30% | 兵法成型对被打击方一次性士气冲击。 |
| 敌AI 温度 / 权重 | `EnemyAiConfig` | 见 GDD_016 落地 | 种子softmax，不可预测可复现。 |

> **反支柱护栏（W5）**：延续 GDD_019——战斗产出速率不压过非战斗来源；战中调整有代价（在途失位 + 姿态疲劳权衡），防"无脑堆兵+反复横跳"必胜。**验证**：`ZoneBattleBalanceTests`——① 主攻在开阔交战胜坚守（无单一姿态占优）② 坚守耗力慢于主攻 ③ 疲劳侵蚀战力可翻盘 ④ 坚城前城防之利令守方反胜 ⑤ 同守备（400）下唯"准备"决定胜负（备足900破城 / 裸战120退兵）。

## 12. UI Requirements（UI 需求）

- **部署界面**：战场区域**示意图**（区域块 + 地形标注 + 邻接，**非坐标格子**）+ 派支队到区 + 军师按区建议（指范围、给缘由/条件/风险，**无胜率**，GDD_008）+ 预计成型条件。
- **战中界面**：回合状态 + 各区态势（我方 / 敌方情报投影 / 条件进度 ✓✗）+ 调整命令（调动/投预备/改姿态）+ 涌现兵法提示 + 敌动向（投影，反全知）。
- 反全知：隐蔽区/敌真值不直接显示；过时情报文本标注（非纯色，色盲可辨）。
- 键鼠可达（technical-preferences）；1080p/1440p/4K 可读。

## 13. AI Requirements（AI 需求）

敌方区域AI（GDD_016 §MVP"便宜80%"首次落地）：**反全知**（AiWorldView 不接受真值，未侦知玩家隐蔽区）· **种子softmax**（ADR-0006，不可预测可复现）· **渐进记忆**（记玩家上回合区域动作）· 会**增援/反扑/佯动/设伏/收缩**· 效用评分数据驱动。LLM 仅装饰（敌将台词），不参与胜负结算。守城时玩家为守方、攻方由此 AI 驱动（对称）。

## 14. Save / Load Requirements（存档需求）

`BattleResolutionState` 纳入统一存档信封（ADR-0009/0005）：区域态/支队/累计回合/已成条件/AI 种子与记忆/回合钟，round-trip 一致、确定性哈希（ADR-0004）。支持**战中存档续战**。

## 15. Test Requirements（测试需求）

- **确定性**：同种子 + 同玩家指令序 → 同结果（哈希一致）。
- **条件按区成型门**：各条件独立可验（门齐成型、门不齐不成型）。
- **战中调整合法性**：调动只邻接 / 在途延迟 / 越界拒（稳定错误码、无部分写入）。
- **敌AI**：确定性 + 反全知（不读真值）+ softmax 种子复现 + 记忆生效。
- **攻守对称**：同引擎攻/守两向可验。
- **涌现触发**：门齐才发、门不齐不发。
- **负向不变量**：数据模型无坐标/朝向字段；无兵种克制减益（延续 ADR-0011）。
- **失败可继续**：败/超时必含 ≥1 合法可继续命令。
- **存档**：战中 round-trip + 续战确定。

## 16. MVP Scope（MVP 范围）

- 固定 5 类区域（正面/关城·侧翼隘口·敌粮道·遮蔽高地·预备后方）；回合上限 6；支队调动在途 1 回合。
- 战中调整三类：调动 / 投预备 / 改姿态。
- 条件复用现有 TacticCondition 原子（假退伏击/断粮/夜袭链），按区 + 按回合累积涌现。
- 敌AI"便宜80%"：增援/反扑/收缩 + 简单设伏，反全知 + 种子softmax。
- 攻守各一场景：虎牢关攻 / 汜水关守。接 GDD_019 六维准备为初始部署输入、占城C为攻城结算。
