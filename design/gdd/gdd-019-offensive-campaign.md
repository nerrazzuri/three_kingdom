# GDD_019 — 出征攻城（Offensive Campaign）

> **Status**: Reviewed（2026-07-04；焦点跨系统审查 gdd-cross-review-2026-07-04.md 裁定 CONCERNS→5 Warning 全闭——与 GDD_004/010/014/015 无阻断冲突：占城经004控制权、自立倾向 rebellion_lean 已接 014 触发、目标合法性经015；ADR-0010 Accepted）
> **Epic**: epic-029-offensive-campaign-loop
> **关系**: 本文是"进攻侧"的循环规则，**复用**既有系统而非新造——出征战斗走 GDD_009/010/011，占城走 GDD_004，回报走 GDD_014，情报走 GDD_007。本文只定义**把它们连成"主动攻城"的规则 + 占城归属**。

## 1. Overview（概述）

出征攻城是太守的**主动进取能力**：经君主授权率军攻打敌城，用**在治理/备战/情报中真实创造的条件**打赢攻城战，胜则占城（归属见 §占城 C）、得功绩名望而升官，败则折损但可继续。它把此前**只有防守**（GDD_014 太守开局守城）的循环补成**攻守双向**，为玩家提供持续的目标与成长纵深。

## 2. Player Fantasy（玩家幻想）

「我不是坐守孤城等人来打的太守——我**主动请缨出征**，靠事先修好的工事、备足的粮、探明的敌情、设好的伏，**打下一座城**。这一胜是我**挣来的**（我创造了条件），不是骰子给的。打下来君主可能赏给我治理，也可能收走——收多了，我便有了自立的心。」

## 3. Core Loop（核心循环）

```
治理/征兵/备战/侦察（攒资源·攒条件·攒情报）
  → 请缨 + 【君主授权出征】选目标敌城
  → 攻城战（GDD_010 兵法沙盒；你创造的条件决定胜负）
  → 胜：占城（§占城 C）+ 功绩/名望 → 升官（GDD_014）→ 更大战区/出征权/直辖城
     败：折损、退兵，但必留合法可继续状态（不卡死）
```

## 4. Main Rules（主要规则）

**R1 授权前置**：出征须君主授权（属朝廷政令任务，GDD_014 忠臣线）。未授权/超出权限（如攻打非授权目标、越战区）→ 拒绝，稳定错误码，无出征。授权额度/范围随官阶放宽（GDD_014 Rank）。

**R2 目标合法性**：目标须为世界模型中登记的敌方控制城（GDD_004 控制权 / GDD_015 世界投影）。己方城、盟友城、不可达城不可攻。

**R3 闭合因果（核心）**：攻城战的初始条件**由玩家的准备态派生**，不再脚本固定——
- 备战计划（GDD_009 CommittedPlan：设伏/分兵/调粮）→ 战斗初始兵法条件与布局。
- 城市/后勤态（GDD_004/012：征募的兵、转运的粮）→ 你的可投入兵力与续航。
- 情报态（GDD_007：探明的敌军强度/性格/补给）→ 军师可识别的可行招式 + 敌方预设可信度。
- **准备充分 → 触发兵法优势、以弱胜强；裸战 → 战力硬碰、可能败。** 胜负经 GDD_010 确定性结算（无胜率、条件涌现）。

**R4 占城归属（方案 C）**：见 §占城 C。

**R5 回报与失败**：胜 → 功绩/名望增（非战斗源之外的战功来源，GDD_014）→ 升官门槛推进（epic-022）。败 → 折损兵力、退兵；生涯不切死局，退兵后可再备战/再请缨/或转守（失败可继续，红线）。

**R6 反全知**：出征前对敌城的了解**只经玩家情报知识**（GDD_007 投影）——你可能情报有误（敌比预估强/弱），这本身是风险。军师建议不报成功率。

### 占城 C（占领归属规则，权威）

攻城战胜、敌城陷落后，城池控制权经 **GDD_004 唯一权威 + 控制权变更事件**（ADR-0008）转移。**归属**按方案 C：

- **前两座**由玩家攻下的城 → **默认归玩家直辖**（`GrantToPlayer`）。给启动动力。
- **第三座起** → 由君主**种子化确定性随机**（ADR-0006 同款注入式随机流）在 `{君主直辖, 交玩家直辖}` 间取舍，权重可受玩家名望/君主好感/城池战略价值调制（数据驱动）。可复现、不可预测、非掷骰。
- 君主直辖的城 → 记入君主势力（GDD_015 投影），玩家得功绩/名望/赏赐但不直辖。
- **自立动机累积**：每当玩家攻下的城被君主收走，累积一个"自立倾向"量（喂 GDD_014 自立线的触发条件）。屡抢战果 → 玩家更易/更想自立。这是**特意的张力**。

## 5. Formulas（公式）

> 权威路径整数/定点（ADR-0004）；此处给结构，具体系数进 Tuning Knobs / 配置。

- **F1 出征战力（进攻方）**：`atk_force = f(投入兵力, 士气, 补给续航, 兵法条件加成)`——复用 GDD_010 战力式，进攻方=玩家投入。
- **F2 兵法条件派生**：备战计划中的每条命令（设伏/断粮/分进）在开战态里满足 GDD_010 招式的**涌现前提条件**（位置/隐蔽/时机/补给等）。前提齐备时，**结算加成来自 GDD_010 §7 的条件涌现结算路径**（非任何"计策按钮"）；`TacticRecognizer` 只在**战后只读打复盘标签**（火攻/诱敌/断粮…），不参与结算、不改状态——二者须分述，勿混为一。
- **F3 占城归属取舍（C，第三座起）**：
  ```
  seed = Hash(worldTick, playerFactionId, cityId, conquestIndex)
  p_grant = clamp( base_grant + w_renown·名望norm + w_standing·君主好感norm + w_value·城价值norm , 0, 1 )
  归属 = seeded_bernoulli(seed, p_grant) ? GrantToPlayer : LordKeeps
  ```
  前两座 `conquestIndex < 2` → 恒 `GrantToPlayer`。
- **F4 自立倾向累积**：`rebellion_lean += lean_per_seizure`（每次被君主收走战果）。

## 6. Data Model（数据模型，草案）

- `CampaignAuthorization`（君主授权：目标城集合/战区/额度，随官阶）
- `OffensiveTarget`（目标敌城 id + 当前情报视图）
- `ConquestRecord`（已攻下城计数 conquestIndex + 各城归属结果）
- `OwnershipVerdict { GrantToPlayer | LordKeeps }`（占城 C 判定输出）
- 复用：CommittedPlan（GDD_009）、BattleSnapshot（GDD_010）、CityControlAuthority（GDD_004）、CareerState（GDD_014）

## 7. Player Inputs / System Outputs

- **输入**：请缨/接受出征政令、选目标城、（复用）备战计划、开战、解析、结算。
- **输出**：攻城战果、城池控制权变更事件（GDD_004）、功绩/名望增量（GDD_014）、占城归属判定、自立倾向增量、失败续局选项（GDD_010 §后果）。

## 8. Tuning Knobs（可调值）

- 各官阶授权额度/战区范围
- 占城 C：`base_grant`、`w_renown`、`w_standing`、`w_value`、前 N 座默认归玩家的 N（默认 2）
- `lean_per_seizure`（自立倾向累积速率）
- 闭合因果各准备维度 → 战力/条件的映射系数

> **反支柱护栏（W5，强制）**：出征是**强战功→功绩/名望**通道。平衡时须保证**出征的功绩/名望产出速率不压过治理/招揽/外交/平叛等非战斗来源**——否则最优玩法退化为"只刷出征"，踩中 game-pillars 反支柱（承接 GDD_014 Balancing N10 与 2026-06-24 W5）。护栏具体化：单次出征功绩产出设上限 + 出征有真实成本（兵粮消耗/时间/失败折损），使"刷出征"不优于均衡经营。列入平衡验证用例。

## 9. Edge Cases（边界情况）

- 情报错误：敌远强于预估 → 可能大败（合理风险，非 bug）；败后可继续。
- 授权撤销：出征中君主撤令 → 须定义（建议：已开战不受影响，未开战则取消）。
- 前两座之一战败未占：conquestIndex 只在**成功占城**时递增（默认归玩家名额不被失败消耗）。
- 占城后立即被攻（敌反扑）：走守城（GDD_014 防守侧），控制权可再变。

## 10. Dependencies（依赖）

GDD_004（城池控制权/占城写入）· GDD_007（情报/反全知）· GDD_008（军师）· GDD_009（备战）· GDD_010/011（战斗/士气）· GDD_012（补给）· GDD_014（生涯/授权/回报/自立）· GDD_015（世界投影/目标合法性）· ADR-0004/0006/0008 · 拟 ADR-0010（占城归属契约）。

## 11. Failure Cases（失败即可继续，红线）

出征败 → 折损退兵，**必生成合法可继续状态**：再备战、改守、转攻他城、或（若倾向足）自立。绝不卡死、不切死局。

## 12. Acceptance Criteria（验收，可测）

- **AC-1 授权门**：未授权/越权出征被拒（稳定错误码）；授权范围随官阶正确放宽。
- **AC-2 目标合法**：只可攻登记的敌控城；己方/盟友/不可达城被拒。
- **AC-3 闭合因果**：相同种子下，**不同准备态 → 不同攻城胜负/兵法识别**（准备充分可胜、裸战可败）；全程确定性、无胜率数字。
- **AC-4 占城 C**：前两座恒归玩家直辖；第三座起按 F3 种子化取舍（同种子同结果，可复现）；占城经 GDD_004 控制权变更事件写入。
- **AC-5 回报联动**：胜→功绩/名望→晋升门槛推进；被君主收走战果→自立倾向累积。
- **AC-6 失败可继续**：败局必含 ≥1 合法可继续命令。
- **AC-7 反全知**：出征前敌情只经玩家知识投影，类型层取不到敌方真值。
- **AC-8 存读档**：出征态（授权/目标/占城计数/归属）存读档 round-trip 一致，确定性哈希。
