# GDD_026：空降者·纪元开局与一生（Arrival, Era & Lifespan）

> **Layer**: Meta（开局与人生框架——把"多剧本/势力选择"升级为"任选时空、活出一生"）
> **Status**: Draft（2026-07-05）
> **Governing ADR**: 拟 ADR-0015（纪元与世界快照模型）· 复用 ADR-0004（确定性）· ADR-0005（存档版本）· ADR-0009（CampaignSession 装配）
> **消费/被消费**: GDD_001 游戏时间（扩公元纪年层）· GDD_003 世界地图 · GDD_005 人物 · GDD_014 生涯/自立 · GDD_015 历史事件（改按公元年）· GDD_025 武将标签（EraStage/时代分段）· #1 PlayableStart（泛化）· #2 武将档案（补生卒/布防）

## 1. System Purpose（系统目的）

把开局从"扮演某个具体诸侯"改为**"一个与三国演义无关的空降者，任选年份与城池，作为某势力的太守起家，活出他的一生"**。选年 = 选一张**世界快照**（势力/城/在世武将/前方大势）；选城 = 在该快照里**任取一城为太守**（君主治所除外），**该年该城在职武将尽归其麾下**；一生 = 空降者**有寿命**，寿终或退隐即一世落幕，可传承子嗣续局或重开。此为本作"人生模拟 × 战略沙盒"的顶层框架。

## 2. Player Fantasy（玩家幻想）

"我选公元 190 年、乱世方启——落在徐州下邳做个太守。城里几员部将从此听我调遣。我这一生约莫四五十年：是安做汉臣熬资历，还是趁群雄割据自立门户？我会亲眼看着官渡、赤壁、三分在我面前展开，也会老去。等我寿终，我的儿子接过这份基业——或者，我换个年份、换座城，再活一次全然不同的人生。"

## 3. Core Loop（核心循环）

选**锚点年**（世界快照）→ 选**一城**为太守（属该势力，君主治所除外）→ 该城该年**在职武将归你** → 在有限的一生里治城/征战/晋升/施计/自立（既有全循环）→ **公元年推进**，天下大势（历史事件）在你面前展开、武将随龄变化 → **寿终/退隐** → 一世结算（官阶/领土/生涯成就定评）→ **传承续局** 或 **换时空重开**。

## 4. Main Rules（主要规则）

- **R1 广义时代 + 锚点年（强制）**：时代取广义三国 **184–280**（群雄割据 → 三分 → 归晋）。世界快照只在**精选锚点年**提供（命名剧本：184 黄巾 / 190 讨董 / 200 官渡 / 208 赤壁 / 220 三分 …），**非连续任意年滑块**（内容与确定性代价）。首发仅 190 讨董（复用现有 17 席/37 城骨架）。
- **R2 公元纪年层**：世界钟在既有"日-段"（GDD_001）之上叠一层**公元年锚**：`当前年 = 起始年 + 已历年数`。历史事件（GDD_015）与空降者寿命**改按公元年计**；战役/治理仍走日-段。年推进为**确定性纯函数**（ADR-0004）。
- **R3 任选城起家（君主治所除外）**：玩家可在快照里选**任意一座非君主治所的城**为太守；开局即**属该城所属势力**，君主为该势力之主。君主亲镇的治所不可选（玩家始终是**太守而非君主**，接 GDD_014 太守→晋升→自立的生涯弧）。
- **R4 该年该城武将归你**：所选城在**该锚点年在职**的具名武将，开局即为玩家**部曲/僚属**（接 GDD_025 档案 + GDD_014 僚属）。武将**在职与否**由其**生卒年**与**任职城×锚点年布防表**决定。
- **R5 空降者有寿命（一生）**：空降者以某**入场年龄**降临，拥有一段**寿命**（确定性派生自人设 + 会话种子，非纯随机）。寿终或主动**退隐**即一世结束。选年因此**有取舍**——早入乱世活得久、历经更多；晚入则跑道短。
- **R6 传承而非纯重开**：一世落幕经**一世结算**（生涯成就/官阶/领土定评）后，默认**传承子嗣或择一亲信续局**（继承部分基业/官阶，接 GDD_014），玩家亦可**换时空重开**。
- **R7 武将随龄变化（接 GDD_025 R7）**：`EraStage`（少/壮/暮）由武将当前年龄派生，喂标签的时代分段（暮年多疑、盛年锐进）；历史事件仍可改标签。
- **R8 反全知不破（接 GDD_007/025）**：选城时**只见名声与已知信息**——武将的隐藏量级档（战阵档/谋略档）与隐秘的心（忠诚/野心）**绝不因"归你管"而暴露为数字**；仍须相处/情报方知深浅。

## 5. Core Loop 结构（数据流，非数值面板）

```
锚点年 Y ── 世界快照(Y) ─→ {势力/城归属 + 各城君主 + 各城在职武将(按生卒×布防)}
                                   │
玩家选城 C(非治所) ──────────────→ PlayableStart(玩家势力=Owner(C,Y), 治所=C, 部曲=Generals(C,Y))
                                   │
空降者(入场年龄, 寿命L=f(人设,种子)) ─→ 一生 [Y, Y+L]：治城/征战/晋升/自立（既有全循环）
                                   │
公元年推进 ──→ 历史事件(按年) + 武将随龄 + 逼近寿终
                                   │
寿终/退隐 ─→ 一世结算 ─→ 传承续局 或 换时空重开
```

## 6. Data Model（数据模型 · 权威来源）

- **纪元锚（Calendar，权威）**：`StartYear`（锚点年，如 190）、`YearsElapsed`（已历年数，随年度推进增），派生 `CurrentYear = StartYear + YearsElapsed`。入存档（DTO 版本化，ADR-0005）。
- **世界快照（Snapshot，配置/只读）**：`anchorYear → { FactionRecord[]（势力+君主+城）, CityOwnership[], CityGarrison[（城→在职武将 id）] }`。现有 `World` 表（#1）泛化为**按年**的快照集；不可变配置，构建期校验（ADR-0003）。
- **武将生卒/布防（配置，扩 GeneralDossier）**：每将新增 `BirthYear`、`DeathYear`；`Placement: anchorYear → CityId`（该年任职城）。**权威来源=GeneralDossiers**（#2），构建期与快照城归属自洽校验。
- **空降者一生（Life，权威）**：`EntryAge`、`Lifespan`（派生 `DeathYear = StartYear + Lifespan`）、`Retired`（是否已退隐）、`HeirChain`（传承链，接 GDD_014）。入存档。
- **消费者**：PlayableStart 生成（#1）· 武将目录/僚属（#2/GDD_014）· 历史事件调度（GDD_015）· EraStage（GDD_025）。

## 7. Player Inputs（玩家输入）

- **选锚点年**（开局，从命名剧本中选）。
- **选城**（开局，快照内任一非治所城；反全知只见名声/已知信息）。
- **（一生中）主动退隐**（可选，提前结束一世换取生涯定评）。
- **（一世落幕后）传承续局 / 换时空重开**。

## 8. System Outputs（系统产出）

- 开局：一份程序化 `PlayableStart`（玩家势力/治所/部曲）+ 空降者一生参数（入场年龄/寿命）。
- 运行中：当前公元年、距寿终年数（定性提示，非精确倒计时——见 §11）、按年展开的历史事件、随龄变化的武将 EraStage。
- 落幕：一世结算报告（官阶/领土/生涯成就）+ 传承句柄。

## 9. Formulas（公式 · 均附依据，禁硬编码）

> 数值皆为**平衡旋钮**（§12），此处给结构与默认值 + 依据；实现取自版本化配置（CON-5）。

- **F1 当前年**：`CurrentYear = StartYear + YearsElapsed`。
- **F2 年推进边界**：`YearsElapsed` 于**年度边界**递增。默认 `1 公元年 = 一个年度回合`（由 `SegmentsPerYear` 个日-段构成的季节循环；`SegmentsPerYear` 为旋钮）。依据：把细粒度日-段（战役/治理）与粗粒度年（大势/寿命）解耦，年级足够表达"一生"而不使回合爆炸。
- **F3 空降者寿命（确定性）**：
  `Lifespan = LIFESPAN_BASE + round( (2·u − 1) · LIFESPAN_SPREAD )`，其中 `u = DeterministicRandom(sessionSeed ⊕ SALT_LIFE).NextUnit() ∈ [0,1)`。
  默认 `LIFESPAN_BASE = 48`、`LIFESPAN_SPREAD = 7` → `Lifespan ∈ [41, 55]`（整年）。
  `DeathYear = StartYear + Lifespan`；`DeathAge = EntryAge + Lifespan`（默认 `EntryAge = 20` → 卒于 61–75 岁）。
  依据：汉末士人多卒于六七十岁；给玩家一世约 40–55 活跃年，够历经数个锚点大势又不至无尽。种子化 → 存读档一致（同人设/会话恒定）。
- **F4 武将在世/在职**：`Available(g, Y) ⟺ (g.BirthYear + SERVICE_MIN_AGE ≤ Y) ∧ (Y ≤ g.DeathYear)`。默认 `SERVICE_MIN_AGE = 16`。依据：未及冠不列朝、已卒不在世。
- **F5 EraStage（接 GDD_025）**：`age = Y − g.BirthYear`；`Young: age < STAGE_PRIME_MIN`、`Prime: STAGE_PRIME_MIN ≤ age ≤ STAGE_LATE_MIN`、`Late: age > STAGE_LATE_MIN`。默认 `STAGE_PRIME_MIN = 30`、`STAGE_LATE_MIN = 50`。依据：与既有 EraStage 三档对齐，盛年 30–50。
- **F6 可选城集**：`Selectable(Y) = { c ∈ Snapshot(Y).Cities | c ≠ 任一势力君主治所 }`。依据：R3 玩家为太守非君主。

## 10. Edge Cases（边界情形）

- **空城**：所选城该年无具名在职武将 → 只带默认僚属（Aide）起家，合法。
- **君主治所**：不入 `Selectable`；UI 列出但标注"君主亲镇，不可为太守"（AC-5 可见性）。
- **入场即近寿终**：晚锚点年 + 低寿命 → 一生极短，仍合法（选年取舍的体现）；给出定性预警。
- **武将该年已卒/未出仕**：不进任何城布防，不可归你（F4）。
- **锚点年不含某城**：早年快照城域较小 → 只提供该快照存在的城（F6 以快照为准）。
- **传承链断绝**：无子嗣/亲信可继 → 回退换时空重开（非崩溃）。
- **年推进跨多个历史事件**：一次年度推进跨多事件 → 全部按年触发并入通报流（GDD_015 既有序）。

## 11. Failure Cases（失败/反挫）

- **寿终非"失败"**：寿终是一生的**自然落幕**，走结算+传承，非 game-over 死局（与"败战可继续"同哲学）。
- **不给精确死亡倒计时**：只给定性档（"春秋鼎盛 / 年事渐高 / 风烛残年"），避免玩家把一生当秒表倒数（反全知精神延伸至己身）。
- **退隐是主动权衡**：过早退隐锁定较低生涯定评——是取舍非惩罚。

## 12. Balancing Parameters（平衡旋钮 · 均可版本化配置）

| 旋钮 | 默认 | 依据（链 §9） |
|---|---|---|
| `EntryAge` | 20（弱冠） | 可仕又留足跑道（F3） |
| `LIFESPAN_BASE` | 48 年 | 汉末寿数中位偏低（F3） |
| `LIFESPAN_SPREAD` | ±7 年 | 一生 41–55 活跃年（F3） |
| `SegmentsPerYear` | 待定（战斗/治理节奏定） | 年级与日-段解耦（F2） |
| `SERVICE_MIN_AGE` | 16 | 及冠出仕（F4） |
| `STAGE_PRIME_MIN` | 30 | EraStage 盛年下界（F5） |
| `STAGE_LATE_MIN` | 50 | EraStage 暮年下界（F5） |

## 13. UI Requirements（界面）

- **开局选择屏**：① 锚点年选择（命名 + 一句话情境，泛化现有 `ScenarioChoiceView`）；② 地图选城（列可选城，标注势力/名声/在职武将概览；君主治所置灰说明）。全程**反全知**——不显任何能力数值。
- **HUD**：常显当前公元年 + 空降者年龄/寿数定性档；天下大势通报按年流入（复用 EventNotices）。
- **一世结算屏**：官阶/领土/生涯成就 + 传承入口。

## 14. AI Requirements（AI）

- 无新增战斗 AI。对手扩张（GDD_017/018 争霸）继续按年推演。历史事件调度改按公元年门（GDD_015）。

## 15. Save / Load Requirements（存档）

- 新增权威字段：`StartYear`、`YearsElapsed`、`EntryAge`、`Lifespan`、`Retired`、`HeirChain` → 显式版本化 DTO，逆序迁移链（ADR-0005）。快照/生卒/布防为**配置**（构建期），不入存档（指纹校验，ADR-0003）。
- round-trip：同存档恢复后当前年/寿数/在职武将集恒等（确定性，ADR-0004）。

## 16. Test Requirements（测试）

- **选城生成**：任选非治所城 → PlayableStart 玩家势力=该城 Owner、治所=该城、部曲=该年该城在职武将；君主治所不入可选集。
- **在世/在职**：某将在锚点年前未生/已卒 → 不在世不布防；生卒边界值。
- **寿命确定性**：同会话种子 → 同寿命/卒年（存读档一致）；不同人设 → 不同寿数（在区间内）。
- **年推进**：跨年触发按年历史事件；EraStage 随龄跨档。
- **一世落幕**：寿终 → 走结算+传承而非崩溃；无继承人 → 回退重开。
- **存档 round-trip**：纪元/一生字段恢复一致。

## 17. MVP Scope（最小可玩）

- 单锚点年 **190 讨董**（复用现有 17 席/37 城，补该年在职 48 将布防）。
- **任选非治所城**起家 + 该城该年武将归你。
- **公元纪年层** + **空降者寿命/寿终** + 按年历史事件。
- 现有 48 将补 `BirthYear/DeathYear` + 190 布防。
- 传承取最小实现（单继承人或重开）。

## 18. Future Scope（后续）

- 增补锚点年 200/208/220/184（各一张快照 + 该年布防），逐年扩。
- 武将谱扩向 ~250，补全生卒/历年布防。

- "布衣无城"空降变体（不做太守、从游侠/寒门起）。
