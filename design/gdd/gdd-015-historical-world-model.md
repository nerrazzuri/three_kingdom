# GDD_015 — 条件历史世界模型

- 状态：Implemented（跨系统审查 + 架构审查 2026-06-24 通过；ADR-0007 条件历史模型 + ADR-0008 控制权契约 Accepted）
- **Status**: Implemented
- 范围：Meta（世界骨架与历史推进；跨切片）

## System Purpose

提供玩家所处世界的**结构与方向**：谁占哪座城、谁与谁敌对、现在是哪一年什么局势、下一步会发生什么大事。世界大势严格沿《三国演义》时间线推进（**大势随历史**），为玩家的判断、布局、目标提供**廉价而丰满的骨架**；同时定义历史事件的**条件化触发**——条件还在则历史继续，玩家强大到改变了某事件的前置条件则该事件分叉（**个人靠抉择**）。本层把“严格演义”与“可自立改写”这对矛盾，用**触发条件 + 玩家触及边界**统一起来。

## Player Fantasy

“我活在一个真实推进的三国里——黄巾、董卓、官渡、赤壁照着历史发生，我知道天下是谁的、该往哪使劲。但如果我真的强到提前灭了孙权，赤壁就不会发生了——历史是大势，不是铁轨。”

## Core Loop

世界时间推进（GDD_001）→ 到达某历史事件时间窗 → 检查其前置条件 → 条件成立则按历史结局触发、条件被玩家破坏则走分叉分支 → 更新势力/城池归属与态势 → 写回玩家可感知的世界状态与新目标。

## Main Rules

- **演义时间线骨架**：黄巾之乱、群雄逐鹿、董卓乱政、官渡、赤壁、三分、北伐、归晋等关键节点为**历史事件**，各自带时间窗与前置条件。
- **事件四元组**：每个历史事件 = `{时间窗 + 前置条件 + 正常结局 + 分叉结局}`。
  - 到达时间窗且**前置条件全部成立** → 触发**正常结局**（历史照常）。
  - 前置条件被玩家行动破坏 → 触发**分叉结局**（历史岔开），并置该事件 `diverged` 标志，下游依赖它的事件**重新评估**自己的前置条件。
- **玩家触及边界（保持便宜的关键）**：事件的前置条件按**玩家当前体量/势力圈**判定可达性。玩家**够不着**的事，其前置条件**恒成立**——一个开局的县城太守撼不动官渡赤壁，历史就在轨上跑（零额外成本）。**只有当玩家的势力圈直接触及某事件的前置条件主体**（如已据有/灭掉相关势力或城池），该事件才可能分叉。
  - 推论：玩家“提前灭孙权 → 赤壁不成立”天然要求玩家**已强大到能打下江东**（即已走到自立晚期），故历史分叉的成本被**推迟到晚期**才支付。
- **分叉范围控制（省 scope）**：晚期分叉默认**只让玩家自己的势力圈脱稿**；远处地图按历史或抽象推进，**不要求全图架空史模拟**。具体脱稿深度为可配置策略。
- **势力与城池归属**：世界模型对每座城维护**归属势力 + 守备**，随历史事件与玩家/AI 行动更新；这是玩家“知道城属于谁”“该打谁”的来源。
- **开局禀赋**：每座可开局城池有匹配历史的初始数据（文武团队、兵力、城防、产出），供 GDD_014 太守开局绑定。

## Formulas

> 历史推进确定性（ADR-0004）：同一存档、同一玩家行动序列 → 同一历史走向。事件定义与阈值来自版本化配置（ADR-0003），纳入存档（ADR-0005）。

### 变量定义

| 变量 | 含义 | 范围/单位 | 来源 |
|---|---|---|---|
| `now` | 世界时间 | WorldTime | GDD_001 |
| `event.window` | 事件触发时间窗 | [start,end] | 配置 |
| `event.preconds` | 前置条件集合 | 谓词列表 | 配置 |
| `reachable(e, player)` | 玩家是否触及事件前置主体 | 布尔 | 世界模型×GDD_014 |

### 1. 事件触发判定

```
on time_window_enter(e):
    if ¬reachable(e, player):            # 玩家够不着 → 前置恒成立
        fire(e.normal_outcome)
    else if all(e.preconds hold):        # 玩家在场但未破坏前置
        fire(e.normal_outcome)
    else:                                # 玩家破坏了前置
        mark e.diverged
        fire(e.divergence_outcome)
        reevaluate(downstream(e))         # 依赖 e 的下游事件重检前置
```

### 2. 城池归属（只读投影，订阅 GDD_004 控制权变更）

> **权威边界（跨系统裁定 2026-06-24）**：城级控制权的**唯一权威是 GDD_004**（独占「控制权变更事件」）。
> 本世界模型在战略尺度**只读反映**归属，**不独立写** `city.owner`——历史事件结局、GDD_010 战果、GDD_014 夺城
> 一律经 GDD_004 控制权变更事件落地后，由本模型订阅同步。

```
on GDD_004.ControlChanged(city, newOwner, garrison):   # 唯一更新源
    world.city[city].owner    ← newOwner               # 只读投影
    world.city[city].garrison ← garrison
# 历史事件的 owner_change 不直接写世界模型，而是发起 GDD_004 控制权变更，再回流至此
```

## Data Model

- `HistoricalEvent`：id、时间窗、前置条件谓词、正常结局、分叉结局、diverged 标志、下游事件引用。
- `WorldState`：当前时间、各势力存续与疆域、各城归属/守备、已触发/已分叉事件集合。
- `FactionRecord`：势力 id、君主、存续状态、领有城池、对玩家关系。**本模型为势力实体（含玩家自立新势力）的创建与存续的唯一权威**（裁定 2026-06-28，类比 ADR-0008 城池归属契约）：GDD_014 自立成立时向本模型**发起**创建请求，由本模型创建 `FactionRecord` 并写势力存续；014 只读，不独立创建势力。城池归属仍只读反映、订阅 GDD_004。
- `CitySeed`：可开局城池的历史初始禀赋（供 GDD_014 绑定）。
- `ReachPredicate`：判定玩家势力圈是否触及某事件前置主体的规则。

## Player Inputs

玩家不直接编辑历史；通过 GDD_010 战役、GDD_014 夺城/灭势力**间接**改变前置条件。玩家可查看世界态势（已知部分，受 GDD_007 情报限制）。

## System Outputs

历史事件触发/分叉通知；势力存续与城池归属变化；新战略目标与窗口；写回 GDD_014（生涯所处态势）、为 GDD_016（敌方 AI）提供势力级态势输入。

## Dependencies

依赖 GDD_001（时间）、GDD_003（地图/城池拓扑）、GDD_004（订阅城级控制权变更事件以同步城池归属，本模型不独立写归属）。被 GDD_014（开局禀赋/态势写回）、GDD_016（势力态势）、GDD_007（玩家对世界的认知受情报限制）消费。

## Edge Cases

- 玩家势力圈恰好在事件时间窗边缘触及前置主体：以时间窗内最后一次判定为准，结果确定。
- 多个下游事件依赖同一被分叉事件：按事件 id 稳定序重评估，避免顺序不确定。
- 玩家自立后与历史势力全面敌对：其势力圈扩大，更多事件变为 `reachable`，分叉面随之扩大（成本随体量上升，符合设计）。
- 历史事件与玩家局部战役同时争夺同一城：归属以全局结算顺序（GDD_001 日界顺序）裁定，不并发写。

## Failure Cases

事件定义缺前置或缺分叉分支 → 配置校验拒绝（不得只有正常结局却允许被玩家破坏）。历史推进异常 → 回滚到上一安全世界状态，不产生半更新态。

## Balancing Parameters

各历史事件时间窗与前置条件；`reachable` 判定的体量门槛；晚期分叉脱稿深度策略；开局城池禀赋数值。全部配置化。

## AI Requirements

历史事件的“正常结局”不依赖玩家，由脚本/抽象推进；玩家不在场的势力混战可用**抽象结算**（不必跑完整战役）。AI 势力的扩张/结盟在玩家够得着的范围内由 GDD_016 战略层驱动，够不着的范围按历史抽象推进。

> **抽象结算随机源声明（ADR-0004，ADV-7）**：抽象结算的任何随机性**只经注入式 `IDeterministicRandom`**（种子由世界状态派生）、在声明检查点取值，**禁止旁路随机源**（System/UnityEngine.Random），结果**纳入状态哈希**——保证「同一行动序列→同一历史走向」（§Test Requirements）。实现见 `StrengthAbstractResolver`（epic-012 story-005，已落地并有确定性测试）。

## Save / Load Requirements

保存 WorldState（时间、势力、城池归属、已触发/已分叉事件集合）、HistoricalEvent diverged 标志；round-trip 一致；与 GDD_014 生涯、GDD_010 战役同一存档边界（GDD_013）。

## Test Requirements

够不着的事件前置恒成立、照常触发；玩家破坏前置 → 走分叉且下游重评估；同一行动序列 → 同一历史走向（确定性）；城池归属随事件/战果正确更新；存档 round-trip；配置校验拒绝缺分叉分支的事件。

## MVP Scope

先实现：**单一历史战役框**（如讨董/汜水关一带，对应当前竖切舞台）的世界快照——少数势力、若干城池归属、1–2 个带条件的历史事件、玩家够不着则照常触发。验证“条件触发 + 触及边界”机制成立。暂不做全 96 年时间线与全图分叉。

## Future Scope

完整 184–280 演义时间线事件网络；多战役框可选开局（群雄割据/官渡/赤壁/北伐等）；玩家势力圈脱稿后的局部架空史推进；势力混战抽象结算器；信使/情报延迟下的世界认知。
