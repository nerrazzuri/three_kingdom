# GDD_027 — 武将归属·任用·全局融入（General Affiliation, Appointment & Integration）

> 让 500 员武将从「资料库 + 地图棋子」变成**贯穿全局的 NPC 连接层**：每将有归属、有角色，参与内政、军师、招揽、派系、战斗、历史事件。武将是把所有系统串起来的连接组织。

## 1. Overview（概述）

每名在世武将，在任一纪元年份，要么**在职**于某势力某城、担当某角色（守将/先锋/谋士/内政/斥候/水军…），要么**在野**（未仕，可被发觉、可招揽）。归属由 `baseFaction`（一人一势力标签）+ 生卒年 + 纪元世界盘**纯函数派生**，不依赖任何场景。城以「武将册」为组织单位（**每城 ≤ 20 员，均由太守调拨**）；内政官、军师、军队主将/副将皆从该城册点选。此归属层被六个系统按需消费：招揽、内政、军师、派系关系、战斗、历史事件。

## 2. Player Fantasy（玩家幻想）

「我招揽的每一员将，都真的在我麾下做事——郭嘉替我出谋、糜竺替我理财、张辽替我领一路兵；他们彼此有恩怨、有忠奸；乱世的大事因人而起、因人而变。500 位三国人物不是名册，是我的班底与对手。」

## 3. Detailed Rules（详细规则）

### R1 归属派生（Affiliation）
- `AffiliationOf(将, 年)`：
  - 未及冠或已故（生卒门，GDD_026）→ **不存在**（不入任何册）。
  - `baseFaction` 在该纪元世界盘存续 → **在职**：城 = 该纪元布防城（若有），否则该势力治所；角色由标签派生（R3）。
  - `baseFaction` 该纪元已灭/未立 → **在野**（可招揽池）。
  - 未登记 `baseFaction` 者（名士/隐士/女性/方士/异族散人）→ 默认**在野**（=可招揽中立人才）。
- **著名叛降**（张辽 吕布→曹、姜维 魏→蜀、黄权 蜀→魏…）用**少量纪元覆盖**校正 baseFaction（数据，非引擎）。

### R2 城武将册（City Roster，组织单位）
- 每城持一份武将册，**上限 20 员**。
- NPC 城：由归属派生填充（该城在职将），超 20 则按战阵/谋略档 + 相关度取前 20。
- 玩家城：**太守调拨**——玩家把已招武将指派入册（≤20），可在自己控制的城间调动。

### R3 角色派生（Role，从标签，无新增数据）
- 守将（善守/傲骨）· 先锋（骑锋/莽撞/绝世武）· 谋士（远图/诡谋/经天纬地谋）· 内政（仁德/善内政类）· 斥候（夜袭/善奇袭）· 水军（水战）。一将可兼数役，取其最强项为主役。

### R4 任用（Appointment，玩家侧）
- 玩家从城武将册点选：**内政官 1（治该城内政）· 军师（献策）· 军队主将 + 可选副将（出征/守城，1~2 人/军）**。
- **多军并进**：不同城/不同册可各出一军，同时在途。
- 未任用之将留册待命（仍计忠诚、可触发事件、守城时按角色顶上）。

### R5 六消费方（全部按需查询归属层，互相解耦）
1. **招揽**：在野在世将 = 可招池；难度自隐秘心（忠诚倾向/野心）+ 玩家名望 + 历史亲和。
2. **内政**：城内政官属性 → 改征粮/民心/城防/治安产出。
3. **军师**：献策军师谋略档 → 决定可提策集 + 策质量（诸葛之策 ≠ 马谡之策）。
4. **派系关系**：麾下羁绊/忠诚 → 内部凝聚、背叛风险、外交挂钩。
5. **战斗**：军队主将 + 副将（1~2 人），档/标签 → 战力/行为；多军多线。
6. **历史事件**：事件锚定具名武将（桃园/温酒/三英/三顾/许攸夜奔/白门楼/赤壁/水淹/走麦城/五丈原…），按谁在世/在哪/属谁/玩家扮谁触发；产出登场（Introduce）/移籍（Reassign）/斩杀（Slay）效果，经确定性重放成覆盖层改写归属（ADR-0016 D6）。

## 4. Formulas（公式）

- **在职判定**：`Available(将,年) ∧ FactionAlive(baseFaction,年) → 在职`；否则若 `Available` → 在野。
- **册裁剪**：城在职将 > 20 → 按 `key = max(战阵档, 谋略档)` 降序 + id 序稳定，取前 20。
- **内政加成**（P4 落地时定稿，均版本化）：产出 × (1 + 内政官内政系数)，系数自角色/标签档映射（如 善内政 +0.3、仁德 民心额外 +）。
- **招揽难度**（P2）：净意愿 = 意愿(自忠诚倾向逆) − 阻力(自野心/傲物) + 玩家名望档 + 历史亲和；定性化为 易招/尚可/难招（无数字，反全知）。
- **军师策质量**（P5）：可提策集 = f(谋略档档位)；质量系数 = 谋略档 → CombatProwess 右偏抽取（复用 ADR-0004）。
- 均为整数/定点，禁 float 权威路径（ADR-0004）。

## 5. Edge Cases（边界）

- 势力中途覆灭 → 其在职将转在野（可被别家招揽/触发流亡事件）。
- 城易主（占城）→ 该城武将册归属随城主（守将或降或走，由事件/施计决定，P8）。
- 同一将 baseFaction 该年不存在但被纪元覆盖指到存续势力 → 以覆盖为准。
- 玩家城武将册满 20 再调入 → 拒（须先调出）。
- 在野将无城 → 不在战略图城棋子，但入「可招人才」视图。

## 6. Dependencies（依赖）

GDD_025（武将标签/隐秘心/羁绊，数据源）· GDD_026（生卒门/纪元盘/反全知登场）· GDD_004（城市经济，P4 接入）· GDD_008（军议，P5 接入）· GDD_006/023（关系/外交，P6）· GDD_019/021（出征/区域战，P7）· GDD_015（条件历史，P8 事件）· GDD_020（招揽，P2 统一）· ADR-0016（本系统契约）。

## 7. Tuning Knobs（可调旋钮）

城武将册上限（默认 20）· 内政官各标签加成系数 · 招揽难度各档阈值 · 军师各谋略档可提策集 · 各役标签→角色映射 · 事件触发条件阈值。全部版本化配置（ADR-0003）。

## 8. Acceptance Criteria（验收）

- AC1 `AffiliationOf(关羽,190)=在职·刘备阵营·先锋役`；`AffiliationOf(姜维,190)=不存在(未生)`；`AffiliationOf(司马徽,任意)=在野`。
- AC2 某城在职将 >20 时，册裁剪到 20 且稳定可复现。
- AC3 内政官为善内政之将时，同一征粮令产出高于无内政官/庸才（P4）。
- AC4 军师为经天纬地谋者，可提策集严格超集于庸才军师（P5）。
- AC5 在野在世将可被招揽，难度随隐秘心/名望定性变化（P2）。
- AC6 一军至多主将+副将 2 人；可同时存在多军在途（P7）。
- AC7 演义事件引擎完整：招牌关目表（≥10 桩：桃园/温酒/三英/三顾/许攸夜奔/白门楼/赤壁/水淹/走麦城/五丈原）锚定具名武将，按在世/归属/纪元/玩家势力触发，并产出 typed 效果——登场/移籍/斩杀；效果经确定性重放（`OverridesAt`）累积成覆盖层，优先于 baseFaction 派生（斩杀→Absent、移籍→换势力），可推演不入档（ADR-0016 D6）。
- 全部前 8 阶段纯 C# 无场景依赖、单测覆盖；表现层（P9）★编辑器验证。
