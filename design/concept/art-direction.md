# 美术方向（概念阶段锁定）

## 决策摘要

- MVP 使用 2D 为主的图卷式界面与区域/路线战役地图，不制作连续实时 3D 大军战场。
- 人物使用 2D 半身肖像；部队使用旗帜、轮廓和军阵符号，不制作逐兵单位资产。
- 历史可信的服饰、器物和物质因果构成基底，演义感来自构图、叙事和气氛，不来自超自然法术特效。
- 美术验收首先检查军情可读性、人物状态和后果表达，其次检查装饰完成度。

## 目标

建立“演义感、军情可读性、人物温度”并存的视觉语言。美术首先服务于信息判断与后果表达，其次服务于历史氛围和戏剧张力。

## 视觉支柱

1. **演义写意**：以中国传统设色、水墨留白、木刻线性和军图符号为灵感，避免照搬特定艺术家的风格。
2. **人物可信**：人物肖像强调年龄、身份、经历和情绪，不采用同质化英雄脸或过度玄幻装备。
3. **军情清晰**：地形、道路、风向、天气、视野、补给和士气必须在不同缩放层级保持辨识。
4. **后果留痕**：城损、疲惫、缺粮、关系破裂与声望变化应有持续视觉反馈，不只在结算弹窗出现。

## 视觉定位

整体采用“军府案上展开的活图卷”：地图保留手绘地貌和纸张质感，情报以批注、印记、虚实线和修改痕迹叠加；人物与事件以更具情感的肖像和局部插图进入。画面可以写意，但信息语义必须精确。

三种视觉语气对应三类内容：

- **事实**：稳定轮廓、实线、清晰标签和较高确定性对比。
- **推测**：虚线、淡墨、范围、问号或来源标记，绝不伪装成事实。
- **命令与承诺**：朱批、箭头、印信和资源标记，明确表示尚未结算的玩家计划。

## 色彩与材质

- 世界与城市：低饱和土色、纸张与矿物颜料质感作为基底。
- 势力识别：使用受控的旗帜色与纹样，不仅依赖红绿差异。
- 战术信息：高对比但数量受限的功能色；危险、未知、推测和确认信息必须使用不同形状或纹理冗余编码。
- 天气时段：通过光照、雾、地表和天空共同表达，不能只加全屏滤镜。

## 表现层级

- 战略层：图卷式地图，强调路线、城市联系和势力压力。
- 城市层：以功能区和状态痕迹表达生产、民心、治安与防御，不追求逐户模拟。
- 战役层：MVP 使用区域节点与路线连接表达地形、移动、视野、伏击点和补给线；不追求连续空间微操。未来是否增加格网只在 GDD_003/GDD_010 证明必要时评估。
- 人物层：半身肖像、关系标记与简洁动态表情支持人物叙事。

## MVP 镜头与资产形式

- 世界/战役视角：固定或有限缩放的 2D 地图，不依赖自由旋转镜头。
- 城市表达：一张可分区的城市状态图，使用受损、缺粮、拥挤、修复等叠加层，不建逐户 3D 城市。
- 人物：少量 2D 半身肖像，使用表情、伤病、疲惫或身份标记表达状态；不建立完整 3D 角色管线。
- 部队：旗号、兵种轮廓、人数/士气/军纪信息和阵态符号组合；缩放时保持语义不丢失。
- 事件：关键转折使用局部插图、镜头推进、纸面动画和音效强化，不用技能爆炸演出。

## UI 方向

界面借鉴军令、简牍、地图批注和印信的层级逻辑，但不得牺牲现代信息架构。装饰不能遮挡数字、状态来源和因果解释。所有关键状态需要文本或图形双重表达。

### 信息编码

| 信息类型 | 主编码 | 冗余编码 |
|---|---|---|
| 势力归属 | 旗色与边框 | 纹样、旗号文字、形状 |
| 情报置信度 | 线条实虚与清晰度 | 文本等级、来源、时效 |
| 补给状态 | 路线连续性与粮袋标记 | 数值、趋势、警告文字 |
| 士气/疲劳 | 独立仪表与人物/军阵姿态 | 文本状态、图标形状 |
| 天气/风向 | 地图环境与方向箭头 | 文字、预测区间、时间标签 |
| 命令状态 | 朱批与路径预览 | 待提交/已承诺/执行中标签 |

不能只用红/绿表达敌我、危险或成功。关键路径必须支持键盘焦点、文本缩放和减少动态模式。

## 动效与音频协同

- 动效强调命令传递、时间消耗、情报更新和条件成立，不用夸张技能爆发掩盖规则。
- 音频以环境、军阵节奏和器乐质感建立张力；重要提示需同时提供视觉反馈。

## MVP 资产边界

只制作一座小城、少量关键人物、有限地形与天气、军队标识、状态图标和必要界面原型。优先验证信息可读性，不建立全时代资产库。

### 优先级

1. 战役区域、路线、视野、风向、天气和补给的可读原型。
2. 事实/推测/命令三类信息的统一视觉语法。
3. 人物关系、疲劳、伤病和执行意愿的状态表达。
4. 城市粮食、民心、工事和受损痕迹。
5. 关键事件插图和氛围润色。

如果第 1—4 项无法在灰盒或低保真资产中清楚表达，不进入第 5 项量产。

## 禁止方向

- 计策以 MOBA/RPG 技能图标、冷却圈和爆炸特效为主要表达。
- 颜色成为唯一状态编码。
- 为追求写意而隐藏精确规则与信息来源。
- 在玩法视角未锁定前批量生产战场资产。

## 历史与演义边界

- 武器、服饰、军制、运输和城防以可研究的历史可信性为基础，但不追求博物馆级逐项复刻。
- 演义式夸张用于天气气氛、构图、人物气势、文字和事件节奏。
- 核心规则不使用法术、瞬移、凭空召唤天气或单将消灭整军等超自然解释。
- 传闻和人物崇拜可以夸张，但界面必须区分传闻与已证实事实。

## 美术验证任务

- 在不打开详细面板时，测试玩家能否识别城、敌营、补给线、可疑伏击区和当前风向。
- 在移除颜色后，测试玩家能否依靠纹样、形状和标签区分势力与状态。
- 向玩家展示事实、传闻和军师推测，确认三者不会混淆。
- 放大文本并启用减少动态后，核心命令流程仍可完成。
- 战后仅查看视觉复盘，玩家能指出主要条件及其发生顺序。

## 已分派的后续决策

- GDD_003/GDD_010：区域节点、路线容量、可达性、视野和战术覆盖的精确规则。
- UX_SPECS：地图图层、军议、计划提交和复盘的信息结构。
- ASSET_SPECS：分辨率、切图、状态变体、动画预算和命名规范。
- 技术 ADR：2D 渲染管线、字体、资源加载与本地化方案。

这些决定不阻断概念阶段；在对应文档完成前不得批量生产资产。
