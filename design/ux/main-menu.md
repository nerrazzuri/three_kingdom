# UX Specification: 主菜单 (MainMenuScreen)

> **Status**: Approved
> **Author**: ux-designer + user
> **Last Updated**: 2026-06-21
> **Screen / Flow Name**: `MainMenuScreen`
> **Platform Target**: PC（Windows/macOS/Linux）· 键鼠为主 · Gamepad 部分 · 无触控
> **Related GDDs**: `design/gdd/gdd-013-save-load.md § UI Requirements`（存档槽/继续）；`design/gdd/game-concept.md`（基调/幻想）
> **Related ADRs**: ADR-0005（存档版本与迁移——继续/读档须经版本化存档端口）
> **Related UX Specs**: `hud.md`（开始游戏后进入）、`pause-menu.md`（共享设置面板）、`interaction-patterns.md`
> **Accessibility Tier**: Comprehensive（WCAG 2.1 AA + 减少动态 + 文本缩放 + 高对比，见 `design/accessibility-requirements.md`）

## 1. Purpose & Player Need

**玩家需求**：玩家启动游戏时，需要在一个安静、不喧哗的入口处，确认「我的进度安全」并以最少步骤回到自己的故事，或从容开启新局。本作不是争分夺秒的动作游戏——主菜单应像翻开案头那卷未合的图卷，沉静、有历史质感，不催促。

**玩家目标**：3 步内继续上次进度（Continue）或选择存档读取；新玩家能一眼看出从哪里开始。

**游戏目标**：在进入世界模拟前，安全地经版本化存档端口（ADR-0005）加载权威状态；向玩家传达本作基调（不完全信息、谋略、历史演义），为五状态氛围（art-bible §2）的「生活/观察」起调。

---

## 2. Player Context on Arrival

| 问题 | 答案 |
|---|---|
| 玩家此前在做什么？ | 刚启动游戏（冷启动）；或从游戏内「退出到主菜单」返回 |
| 情绪状态 | 平静、期待；返回时可能带轻微疲惫（刚结束一段对局） |
| 认知负荷 | 低——无实时威胁，无时间压力 |
| 已有信息 | 是否有可继续的存档（须一眼可见） |
| 最可能想做的 | Continue（回到最近存档）——主用例 |
| 最担心的 | 误覆盖存档、找不到上次进度、新建时不知如何开始 |

**情绪设计目标**：沉静、从容、有掌控感——玩家应感到「我的故事在这里等我，没有催促」。

## 3. Navigation Position

```
MainMenuScreen（根）
  ├── NewGameFlow（开局设置 → 进入 HUD）
  ├── LoadGameScreen（存档槽列表，复用 GDD_013 槽位模式）
  ├── SettingsScreen（与 pause-menu.md 共享同一设置面板组件）
  ├── CreditsScreen
  └── QuitConfirm（P9 确认对话）
```

**Modal behavior**: Non-modal 根屏（无世界在其后）。子项以 Overlay push 方式叠加，Esc/Back 逐层返回。

**Reachability — 入口**：

| 入口 | 触发 | 备注 |
|---|---|---|
| 游戏冷启动 | 启动可执行文件 | 主入口；启动画面后落到此屏 |
| 游戏内退出到主菜单 | pause-menu.md →「退出到主菜单」（经 P9 确认） | 须先处理未保存状态（提示保存或丢弃） |

## 4. Entry & Exit Points

**Entry**：

| 触发 | 来源 | 转场 | 传入数据 | 备注 |
|---|---|---|---|---|
| 冷启动 | 启动画面 | 淡入 250ms（减少动态：即时） | 存档存在性扫描结果（有无 Continue 目标） | Continue 项依此 enable/disable |
| 退出到主菜单 | 游戏内（pause-menu） | 淡入 250ms | 无（世界状态已在退出前经存档端口处理） | 退出前由 pause-menu 负责 P9 保存确认 |

**Exit**：

| 退出动作 | 目标 | 转场 | 返回/保存数据 | 备注 |
|---|---|---|---|---|
| New Game | NewGameFlow → HUD | Replace | 新局种子 + 初始场景 id | 经 Application 层创建初始权威状态 |
| Continue | HUD | Replace | 最近存档 slot id | 经 SiegeSaveSerializer 版本化读档（ADR-0005）；失败走错误态 |
| Load → 选槽 | HUD | Replace | 选定 slot id | 同上；不兼容/损坏槽走 P9 + 错误说明 |
| Settings | SettingsScreen | Overlay push | 无 | 设置改动即时保存到玩家档（非单槽） |
| Quit | 桌面 | 经 QuitConfirm（P9） | 无 | 默认焦点落在「取消」 |

## 5. Layout Specification

```
┌────────────────────────────────────────────────────────────┐

│                                                            │
│        三国演义 · 兵法沙盒                                    │  ← 标题区（T1 大标题，山水墨）

│                                                            │
│              ●[ 继续 ]                                       │  ← 主操作（primary，每屏≤1；首焦点）
│               [ 新的局 ]                                     │     菜单列（左中，垂直）
│               [ 读取存档 ]                                    │
│               [ 设置 ]                                       │

│                                                            │
│                                          v1.0 · 存档兼容 OK  │  ← 角注（C1，淡墨灰）
```

### 5.2 Zone Definitions

| 区 | 描述 | 约占 | 滚动 | 溢出 |
|---|---|---|---|---|
| 背景区 | 全屏图卷底图 + 雾叠加（非交互） | 100% | 否 | 按分辨率缩放，保持图卷比例 |
| 标题区 | 游戏名 + 分隔墨线 | 上 ~25% | 否 | T1 字号，超长靠版式不截断 |
| 菜单列 | 6 项垂直按钮 | 左中 ~40%×50% | 否 | 文本缩放 150% 下行高自适应 |
| 角注区 | 版本号 + 存档兼容状态 | 右下 | 否 | C1 字号 |

### 5.3 Component Inventory

| 组件 | 类型 | 区 | 用途 | 必需 | 复用 |
|---|---|---|---|---|---|
| 背景图卷 | Image + Overlay | 背景 | 基调（§2.1） | 是 | 否（`bg_*` / `overlay_fog_*`） |

| 继续 按钮 | Button(primary) | 菜单 | 读最近存档 | 是 | 标准按钮（primary 变体） |

| 版本/兼容角注 | Text(C1) | 角注 | 存档兼容提示 | 是 | BodyText |

**首焦点**：`继续`（若无可继续存档则禁用，并显示「无存档」语义标签，首焦点改落 `新的局`）。

## 6. States & Variants

| 状态 | 触发 | 视觉变化 | 行为变化 | 备注 |
|---|---|---|---|---|
| 有存档 | 扫描到可继续槽 | `继续` 启用，为首焦点 + primary | Enter 直读最近槽 | 默认常见态 |
| 无存档 | 首次游玩/无槽 | `继续` 禁用 + 「无存档」标签，首焦点移到 `新的局` | 继续不可触发 | 不显示无提示的灰按钮（带语义标签，§Standard Controls） |
| 读档中 | 选 Continue/Load 后 | 菜单淡出 + 加载指示（带文字「读取中」+ 阶段说明） | 仅可取消 | >500ms 才显示，避免闪烁 |
| 读档失败/不兼容 | 存档损坏或版本不兼容（ADR-0005 拒绝） | 错误态：图标 + 可行动说明（「此存档来自更新版本，无法读取」）+ 重试/选其他槽 | 仅重试/返回可用 | 不暴露技术细节；不静默失败 |
| 退出确认 | 选 退出 | QuitConfirm 模态叠加 | 背景锁定 | 默认焦点=取消（P9） |

## 7. Interaction Map

### 7.1 导航输入

| 输入 | 平台 | 动作 | 视觉响应 | 备注 |
|---|---|---|---|---|
| ↑/↓ / D-Pad | All | 菜单项间移焦 | 焦点框移动（朱批朱红 2px 描边 + ≤2% 缩放，art-bible §7.2） | 列内循环 |
| 鼠标移动 | PC | 悬停高亮 | 高亮，但**不移焦点**（仅点击移焦） | 无 hover-only |
| 鼠标左键 | PC | 选中并激活 | 按下态闪 → 激活 | — |
| Enter / A | All | 激活当前项 | 转场 | — |
| Esc / B | All | 从子屏返回上一层；根屏无操作 | — | 根屏 Esc 不退出游戏（防误退） |

### 7.2 动作输入

| 输入 | 上下文 | 动作 | 响应 | 备注 |
|---|---|---|---|---|
| Enter / A | `继续` 聚焦 | 读最近存档 | 进读档态 → HUD | 失败走错误态 |
| Enter / A | `退出` 聚焦 | 触发 QuitConfirm | P9 模态 | 默认焦点取消 |

### 7.3 状态限制

| 状态 | 输入限制 | 原因 |
|---|---|---|
| 读档中 | 仅取消可用 | 防并发读档 |
| 退出确认模态 | 仅确认/取消 | 模态锁定背景 |
| 错误态 | 仅重试/返回 | 无有效数据可导航 |

## 8. Data Requirements

| 数据 | 来源系统 | 更新频率 | 拥有者 | 格式 | 缺失处理 |
|---|---|---|---|---|---|
| 存档槽存在性/元数据 | Infrastructure 存档端口（SiegeSaveSerializer） | 屏打开时扫描 | 存档子系统 | 槽列表：{slot, 时间戳, 场景名, schema 版本, 兼容标志} | 空列表 → `继续` 禁用 + 无存档态 |
| 最近存档目标 | 同上 | 屏打开时 | 存档子系统 | slot id 或 null | null → 同上 |
| 当前 schema 版本 | 存档版本常量（ADR-0005） | 静态 | Domain SaveVersion | 版本号 | 用于角注兼容提示 |

> 本屏只读存档元数据，不写任何权威状态；New Game/读档经 Application 层创建/恢复状态。

## 9. Events Fired

| 玩家动作 | 事件 | 载荷 | 接收系统 | 备注 |
|---|---|---|---|---|
| 选 New Game | NewGameRequested | {seed?, scenario_id} | Application（场景工厂） | 创建初始权威状态 |
| 选 Continue/Load | LoadGameRequested | {slot_id} | Application（经存档端口） | 失败回错误态，不部分加载 |
| 确认退出 | QuitGameRequested | {} | App 生命周期 | 经 P9 确认后 |

## 10. Transition & Animation

| 转场 | 触发 | 类型 | 时长 | 缓动 | 减少动态跳过？ |
|---|---|---|---|---|---|
| 屏进入 | 冷启动/返回 | 淡入 | 250ms | 墨入 ease-out | 是（即时） |
| 子屏 push/pop | 选项/返回 | 左右滑 | 200ms | 纸落 | 是（即时） |
| 雾叠加漂移 | 常驻 | 极缓循环 | 连续 | 雾散 linear | 是（静止） |
| 按钮按下 | 激活 | 缩放 95% | 60/60ms | — | 否（触觉反馈，art-bible §5.3 即时反馈不省） |

所有动画遵循 P12：可跳过、不影响任何状态；减少动态开关下退化为即时（art-bible §7.3）。

## 11. Input Method Completeness Checklist

**键盘**
- [x] 全部项可经 ↑↓/Tab + Enter 到达与激活
- [x] 焦点顺序与视觉阅读顺序一致（上→下）
- [x] 鼠标可达的均键盘可达
- [x] 焦点全程可见（朱批朱红 2px，对比 ≥3:1）
- [x] 模态内焦点受限（QuitConfirm 焦点陷阱）

**Gamepad**
- [x] D-Pad/左摇杆可达全部项
- [x] A=确认 / B=返回，符合平台惯例

**鼠标**
- [x] 全部项有悬停态
- [x] 点击命中区 ≥32×32px（按钮高 ≥44px）
- [x] 右键 = 无操作（非未定义）

**触控**：不适用（无触控平台）

## 12. Screen-Level Accessibility Requirements

**对比度**（art-bible §4，实测优先）：

| 文本元素 | 背景 | 要求 | 现估 | 通过 |
|---|---|---|---|---|
| 标题 T1（山水墨） | 宣纸白底图 | 4.5:1 | ~11:1 | [x] |
| 菜单项标签（山水墨） | 旧绢米按钮底 | 4.5:1 | ~9:1 | [x] |
| 主按钮 primary 焦点态 | 朱批朱红边/底 | 4.5:1（大字）/3:1 图形 | 需实测 | [ ]（实现期实测） |
| 角注 C1（淡墨灰） | 宣纸白 | 4.5:1 | 需实测（淡墨灰偏浅） | [ ]（若不过改山水墨） |

**色盲安全**：菜单纯文字 + 形状，无颜色编码信息；`继续` 的「主操作」靠 primary 形状 + 位置 + 首焦点，非仅颜色（P3）。

**焦点顺序**：1 继续 → 2 新的局 → 3 读取存档 → 4 设置 → 5 制作名单 → 6 退出 →（循环）。

**屏幕阅读器播报**：

| 状态变化 | 播报 | 时机 |
|---|---|---|
| 屏打开（有存档） | 「主菜单。检测到存档：[场景名]，[时间戳]。继续 已选中。」 | 焦点落定 |
| 屏打开（无存档） | 「主菜单。无存档。新的局 已选中。」 | 焦点落定 |
| 聚焦菜单项 | 「[项名]。[启用/禁用及原因]」 | 焦点到达 |
| 读档失败 | 「读取失败：[可行动原因]。」 | 错误态渲染 |

**文本缩放**：100/125/150% 三档；菜单列行高自适应，150% 下不截断不重叠（art-bible §7.1，9 组合验证关卡）。

**认知负荷**：单一信息流（6 项线性菜单）——远低于 7±2。

## 13. Localization Considerations

- 全文本来自本地化字符串，无图内文字；中文为基准。
- 容忍 +40% 扩展（英文/德文本地化时）。

| 文本 | 中文基准 | 上限 | RTL | 溢出 | 风险 |
|---|---|---|---|---|---|
| 游戏标题 | 9 字 | 24 | 居中 | 不截断（缩字号 ≥90%） | 低 |
| 菜单项（「读取存档」等） | 2–4 字 | 16 | 镜像 | 按钮宽自适应 | 中（德文长） |
| 角注兼容提示 | ~10 字 | 30 | 右对齐 | 换行 | 低 |

## 14. Acceptance Criteria

**布局/渲染**
- [ ] 1080p/1440p/4K 三分辨率下无重叠/截断
- [ ] 16:9 与 21:9 下背景图卷不变形拉伸
- [ ] 全部状态（有/无存档/读档中/失败/退出确认）正确渲染

**输入**
- [ ] 全部项纯键盘可达（↑↓/Tab + Enter）
- [ ] 全部项 Gamepad（D-Pad + A/B）可达
- [ ] 焦点全程可见，根屏 Esc 不退出游戏

**事件/数据**
- [ ] New Game/Load/Quit 事件以正确载荷在各退出路径触发
- [ ] 本屏不直接写任何权威状态（仅经 Application）
- [ ] 损坏/不兼容存档走错误态，不部分加载、不崩溃

**无障碍**
- [ ] 全部文本达 §12 对比度（实测）
- [ ] 主操作不靠颜色单一通道区分
- [ ] 屏幕阅读器播报存档状态
- [ ] 减少动态下转场即时，雾叠加静止
- [ ] 文本 150% 无溢出/重叠（9 组合）

**本地化**
- [ ] 无文本溢出（含最长译文）
- [ ] 全部文本走本地化字符串
