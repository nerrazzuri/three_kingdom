## 1. HUD Philosophy

**本作与屏上信息的关系**：**HUD 即被批注的军府图卷本身**（art-bible §1.1）。这是策略沙盒——「完整且分层的态势感知」就是游戏，但态势是**不完全信息**的：玩家只见自己合法知道的（己方精确账本 vs 敌方探报推测，P10）。HUD 不是覆盖在世界上的装饰，它就是玩家用来判断、布局、承诺、复盘的那张图。

与动作游戏相反，本作**无实时压迫**——回合制，时间只在玩家推进时前进（P8）。因此 HUD 的设计约束不是「别挡视野」，而是「**别替玩家判断、别泄露不该知道的真值、别把多维状态压成一个数字**」（P11/P10/P6）。

**可见性原则——拿不准时显示还是隐藏？** **CONTEXTUAL（情境化）**：信息按五状态（art-bible §2 生活/判断/行动/战争/战果）显隐与强调切换。常驻的只有「时间/期限条」与「己方关键状态摘要」；其余随玩家所处状态浮现。

**必要性法则**：「一个 HUD 元素值得常驻，当且仅当——**玩家若看不到它，就会在不完全信息下做出本可避免的误判，或被迫离开当前情境去别处翻找。**」纯粹「让画面热闹」的元素一律退为按需/可展开。

### 4.2 Element Detail（要点块）

**时间/期限条（P8）**
- 视觉：顶栏满宽水平条；左=当前日+时段（更漏意象），中=时间轴（已提交行动实线 / 预测虚线），右=活跃主期限 + 当前计划总耗时。
- 更新：仅在玩家推进时段时前进；**无实时倒计时压迫**；浏览/暂停态自由。
- 紧急态：期限临近 → 该期限标转警戒琥珀 + 沙漏图标 + 文字（脉冲 2 次后驻留，非常驻闪烁，art-bible §5.4）。
- 交互：点击期限项跳转相关面板；键盘可聚焦遍历。

**敌方探报卡（P10 + P1/P2）**
- 视觉：与左侧己方账本**布局对称、数据形态不同**——己方精确值/完整账本；敌方仅区间（「兵力 ~8k–12k」）+ 来源图标 + 时效（「2 时段前」，过期高亮）+ 置信（来源可靠性标识，非孤立百分比）。
- 更新：侦察命令刷新；随时段衰减（时效权威归 gdd-007）。
- 设计锁：**绝不显示敌方精确实时真值**；冲突报告并列不合并（P1）。

**命令路径（P4）**
- 草稿态：朱批朱红粗虚线 + 「草稿」水印，可自由增删、Esc 整体放弃，无副作用。
- 承诺态：实线 3px + 箭头 + 印信封存（印定动效，art-bible §5.3），原子生效后常驻于地图，不可静默回退。
- 提交前走 P7 风险预览（列风险因素，**不给单一成功率**）+ P9 跨期限确认。

**军师入口（gdd-008 / P11）**
- 内容：观察 + 候选路线（所需/风险/操作）+ 缺失情报 + 置信 +「不替你定计/不保证」免责。
- 行为：判断态前 3 回合自动浮现（新手上手），之后按 `?` 调出。**不排优劣、不选最优、不暴露隐藏真值、不可一键转为完整命令**（P11 否决标准）。

## 5. HUD States by Gameplay Context

对齐 art-bible §2 五状态：

| 情境 | 显示 | 隐藏 | 修改 | 进入转场 |
|---|---|---|---|---|
| 生活/观察（§2.1） | 时间条（淡）、地图底图 | 敌方探报、命令路径、士气疲劳、警告 | 己方账本淡至 60%；光照暖、低对比 | 无威胁时淡入，400ms（减少动态：即时） |
| 判断/布局（§2.2） | 时间条、己方账本、敌方探报、军师入口、情报来源标 | 命令承诺态 | 图卷放大至战役区；外围压暗 20–30% | 进入判断时图卷平移 250ms |
| 行动/承诺（§2.3） | 命令路径(朱批)、己方账本、风险预览(P7) | 军师自动浮现（转按需） | 已承诺路径高亮跳出；背景压暗 | 印信封存 350ms |
| 战争/应变（§2.4） | 时间条、敌方探报、迷雾/天气层、士气疲劳军纪、期限警告 | 军师自动 | 迷雾激活、对比来自天气；敌方仅探报 | 战事触发即时显示（无淡入，态势须即时） |
| 战果/延续（§2.5） | 因果流(P5)、地图标记更新、关系/城损变化 | 战时执行层 | 胜:微回暖；败:偏冷灰但仍有高亮（「尚有余地」） | 结战后停顿，因果链逐步展开（可跳过） |
| 全屏模态（军议/暂停/读档） | 无 HUD（对应屏接管） | 全部 HUD | 世界暂停于其后 | HUD 150ms 隐去 |

## 6. Information Hierarchy

| 元素 | 优先层 | 理由 | 隐藏时替代 |
|---|---|---|---|
| 时间/期限条 | MUST KEEP（任何非全屏态） | 一切承诺有时间代价，失去则无法判断期限 | 无替代 |
| 命令路径（构建/提交中） | MUST KEEP | 看不到已承诺即可能重复/冲突部署 | 无 |
| 敌方探报（判断/战争态） | MUST KEEP（相关态） | 不完全信息下的核心判断依据 | 无（但保持「未知」诚实标注） |
| 己方账本 | SHOULD KEEP | 决策需己方精确状态 | 可折为摘要，展开见全账本 |
| 士气疲劳军纪 | SHOULD KEEP（战争态） | 影响可行动作 | 展开式，非常驻全维 |
| 军师入口 | CAN HIDE（按需 `?`） | 玩家可主动调用 | `?` 随时调出 |
| 因果流 | CAN HIDE（战果态外） | 复盘用，非实时决策 | 可整体跳过/回看 |
| 通知 toast | CAN HIDE（高强度时入队） | 战时低优先通知入队，战后批量放出 | 队列保留 |

## 8. Feedback & Notification Systems

| 通知类型 | 触发系统 | 位置 | 时长 | 进/出动效 | 同时上限 | 优先级 | 队列 | 可关 |
|---|---|---|---|---|---|---|---|---|
| 侦察情报抵达 | 情报(gdd-007) | 右下 toast | 2500ms | 墨入 150ms / 淡出 300ms | 3 叠 | 低 | FIFO；战时入队（回合制无「战斗中」，按「应变态」入队） | 自动 |
| 命令校验失败 | Application 命令路径 | 元素旁 + toast | 持续至处理 | 轻震 1 次 + 红框 | 1 | 高 | 即时，不入队 | 可行动错误文字(P7) |
| 期限/断粮/补给警告 | 时间/后勤(gdd-001/012) | 时间条 + 边缘 | 条件存续 | 警戒琥珀脉冲 2 次后驻留 | 每类 1 | 临界 | 不入队不合并 | 否（条件解除自消） |
| 外交回应（gdd-012§8） | 后勤外交 | 右下 + 在途条 | 4000ms | 墨入 | 1 | 中 | 不与情报合并 | 否 |
| 条件链应验 | 战斗(gdd-010) | 相关节点 | 单次 | 节点高亮扩散（§5.4） | 1 | 中 | 不入队 | 否 |

**队列规则**：① 应变（战事）态下「低」优先通知入队，态势缓和后批量放出（≤3 连续）。② 同类 500ms 内合并（「情报 ×3」）。③ 临界警告（断粮/期限/校验失败）永不入队/合并，立即显示。

## 9. Platform Adaptation

| 平台 | 安全区 | 分辨率范围 | 输入 | HUD 备注 |
|---|---|---|---|---|
| PC Windows（1920×1080 参考） | 3% | 1280×720 ~ 3840×2160 | 键鼠（Gamepad 可选） | 须各分辨率正确缩放；21:9 下中央图卷不拉伸、左右账本不超宽；cert 前测 1280×720 |
| PC macOS/Linux | 3% | 同上 | 键鼠 | 同上 |
| Steam Deck（1280×800） | 5% | 固定 | 控制器 | 小屏最小字号关键——全元素测此分辨率；C1 11px 须实测可读，不足则该信息降级为图标+tooltip |

**HUD 可重定位要求**（无障碍）：玩家须可在 HUD 布局设置中至少重定位——时间/期限条、己方账本、军师入口。设置存玩家档（非单槽），跨会话生效。

## 10. Accessibility — HUD Specific

整合 `design/accessibility-requirements.md` + art-bible §7。

### 10.1 色盲模式

| 元素 | 仅颜色风险 | 修复 |
|---|---|---|
| 三语气线（事实/推测/命令） | 颜色区分语气 | 线型（实/虚/箭头）+ 图标 + 标签（art-bible §3.3/§7.4） |
| 势力标记（魏/蜀/吴） | 旗色区分 | 纹样（网格/山/波浪）+ 边框形状 + 文字印记（art-bible §4.4/§4.5） |
| 期限/断粮警告 | 警戒琥珀 | 沙漏/叉号图标 + 文字 |
| 情报四层（事实/报告/推测/未知） | 配色 | 线型 + 来源图标 + 时间戳/置信文字（P1/P2） |
| 命令草稿 vs 承诺 | 朱批色态 | 「草稿」水印 vs 印信图形 + 线型（P4） |

### 10.2 文本缩放

100% 基准；125%/150% 全 HUD 文字按比例放大，容器自适应或内部滚动，不截断不重叠。在途条/敌报卡 150% 下高度自增；中央世界锚的命令路径标签若 150% 超出节点区则降级为图标+tooltip（art-bible §7.1）。

| 元素 | 100% | 125% | 150% | 溢出 |
|---|---|---|---|---|
| 时间/期限条文字 | 过 | 过 | 待实测 | 条高自适应，不压中央 |
| 己方账本/敌报卡 | 过 | 待实测 | 待实测 | 卡高自增，内部滚动 |
| 因果流文字 | 过 | 待实测 | 待实测 | 叠加面板内滚动 |

### 10.3 运动敏感（减少动态）

| 动效 | 严重度 | 减少动态停用？ | 替代 |
|---|---|---|---|
| 期限警告脉冲 | 轻 | 是 | 静态琥珀框 + 文字（不脉冲） |
| 节点高亮扩散（应验） | 中 | 是 | 静态高亮 |
| 命令路径锁定/印信 | 轻 | 是（保留状态结果） | 即时切到承诺态 |
| 迷雾/天气漂移 | 中 | 是 | 静态叠加 |
| 因果流逐步展开 | 轻 | 是 | 一次性全展示，可回看 |
| 通知 toast 滑入 | 轻 | 是 | 即时出现于终位 |

全局禁止 >3 次/秒闪烁（WCAG 2.3.1，art-bible §7.3）。

### 10.4 字幕/旁白

本作弱音频依赖（art-bible §6.1）。事件叙事文本、军师话语、通知**均以文字为权威载体**，完全静音可通关。若后续 audio 团队加入语音事件，须遵循字幕规格（底部居中、≥24px、半透明底 70%、说话人前缀冒号、非颜色区分）。

### 10.5 HUD 可见性控制（无障碍菜单）

| 设置 | 范围 | 默认 | 效果 |
|---|---|---|---|
| HUD 全局不透明度 | 0–100% | 100% | 同步缩放全 HUD alpha |
| HUD 文本缩放 | 100/125/150% | 100% | 缩放全 HUD 文字，布局自适应 |
| 通知详尽度 | 全部/仅重要/关 | 全部 | 仅重要=期限+校验失败+外交回应 |
| 减少动态 | 开/关 | 关 | 见 §10.3 |
| 高对比模式 | 开/关 | 关 | 应用高对比主题（art-bible 高对比变体） |
| 地图层开关（天气/补给/视野/控制） | 各开/关 | 视野开其余按需 | 多层独立开关，各有纹样体系（P3） |

## 12. Acceptance Criteria

**布局/可见性**
- [ ] 全 HUD 元素在 1280×720 / 1080p / 1440p / 4K 下不重叠、不截断、不超安全边距
- [ ] 任一情境（§5 六态）下无两元素重叠
- [ ] 生活态占屏 <12%，战争态 <26%
- [ ] 中央 40% 区无常驻面板（仅瞬时标记/路径）
- [ ] 21:9 下中央图卷不拉伸、左右卡不超宽

**情境正确性**
- [ ] 每个 §5 情境只显示规定元素，转场计时符合 §10.3/art-bible §5.3
- [ ] 全屏模态（军议/暂停/读档）正确隐去全部 HUD
- [ ] 战果态因果链可逐步展开且可整体跳过（终值不变，P12/ADR-0004）

**不完全信息（核心设计锁）**
- [ ] 敌方探报**绝不**显示精确实时真值（仅 P1 推测/报告 + P2 区间/时效/来源）
- [ ] 军师入口不提供「最优解/一键布阵」（P11）
- [ ] 己方=完整账本、敌方=探报，视觉对称数据形态不同（P10）
- [ ] 多维状态（士气/疲劳/军纪、关系）分列不合并（P6）

**无障碍**
- [ ] 全 HUD 文字对最暗与最亮图卷区均达 4.5:1（实测）
- [ ] 无任一元素仅靠颜色区分（去色后信息仍可得）
- [ ] 减少动态停用 §10.3 全部动效，无信息丢失
- [ ] 文本 150% 不致任何 HUD 文字溢出/重叠（9 组合）
- [ ] §10.5 全部可调项可用且跨会话持久
- [ ] 完全静音下全程可玩，无信息丢失（§10.4）
- [ ] 全 HUD 键盘可达、焦点可见（朱批 2px ≥3:1），无 hover-only

**通知**
- [ ] 同类 500ms 内合并
- [ ] 应变态低优先通知入队、缓和后放出
- [ ] 临界警告（断粮/期限/校验失败）立即显示，绕过队列
- [ ] 同时 toast ≤3
- [ ] 场景切换清空陈旧通知

**平台**
- [ ] 1280×720（Steam Deck）无元素裁切/重叠；C1 实测可读
- [ ] 时间条/己方账本/军师入口可重定位且设置持久
