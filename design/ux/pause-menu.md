# UX Specification: 暂停菜单 (PauseMenuScreen)

> **Status**: Approved
> **Author**: ux-designer + user
> **Last Updated**: 2026-06-21
> **Screen / Flow Name**: `PauseMenuScreen`
> **Platform Target**: PC（Windows/macOS/Linux）· 键鼠为主 · Gamepad 部分 · 无触控
> **Related GDDs**: gdd-013 存档读档（§ UI Requirements）；gdd-001 时间（暂停语义——回合制，时间本不流逝）
> **Related ADRs**: ADR-0005（存档版本与迁移——保存/读取经版本化端口、原子写）
> **Related UX Specs**: `hud.md`（其上叠加）、`main-menu.md`（共享设置面板组件、退出目标）、`interaction-patterns.md`
> **Accessibility Tier**: Comprehensive（见 `design/accessibility-requirements.md`）

> **Scope boundary**: 本屏为 **Overlay（覆盖世界，游戏暂停）**。虽叠加于游戏世界之上，按模板规则作为「屏规格」处理，并在导航位置记录其叠加关系。

## 1. Purpose & Player Need

**玩家需求**：玩家在对局中途需要一个「安全的停顿点」——把手从案上挪开去喝口茶、保存进度、调一下设置，或决定收手——而**不担心**世界在背后偷偷推进、不担心误覆盖存档。本作是回合制，时间只在玩家推进时前进，所以暂停的真正价值不是「冻结实时威胁」，而是**提供一个明确的、有掌控感的中断与退出契约**。

**玩家目标**：1 步暂停（Esc）；3 步内完成「保存并安全退出」或「调整一项设置后继续」。

**游戏目标**：在玩家中断时，保证权威状态可经版本化存档端口安全落盘（ADR-0005 原子写）；任何退出/读档路径都不丢失或部分写入状态。

## 2. Player Context on Arrival

| 问题 | 答案 |
|---|---|
| 玩家此前在做什么？ | 在 HUD 中游玩（生活/判断/行动/战争任一态）按下 Esc |
| 情绪状态 | 多样：从容（生活态）到紧张（战争态中途想保存喘息） |
| 认知负荷 | 中——脑中可能正持有一个未提交的部署草稿或一个判断 |
| 已有信息 | 知道自己刚才在做什么；可能有未保存进度 |
| 最可能想做的 | 继续（Resume）——多数暂停只是临时停顿；其次保存 |
| 最担心的 | 误覆盖存档、丢失未提交草稿、误退出丢进度、设置改坏找不回 |

**情绪设计目标**：安心、可控、可逆——玩家应感到「我可以安全地停在这里，什么都不会在背后改变，我的每个选择都有明确后果说明」。

## 3. Navigation Position

```
HUD（游玩中）
  └── PauseMenuScreen（Overlay，世界暂停于其后）
        ├── SaveGameScreen（存档槽列表，P9 覆盖确认；复用 GDD_013 槽位）
        ├── LoadGameScreen（与主菜单共享组件；P9 + 丢未保存进度确认）
        ├── SettingsScreen（与 main-menu 共享同一设置面板组件）
        └── QuitToMenuConfirm / QuitToDesktopConfirm（P9）
```

**Modal behavior**: Overlay（渲染于世界之上，游戏暂停）。可由 Esc/B 直接 Resume 关闭。子项以 push 叠加，逐层 Esc 返回。

**Reachability — 入口**：

| 入口 | 触发 | 备注 |
|---|---|---|
| 游玩中按 Esc | HUD 任一非全屏态 | 唯一入口；全屏模态（军议/已在子屏）内 Esc 先返回该模态 |

## 4. Entry & Exit Points

**Entry**：

| 触发 | 来源 | 转场 | 传入数据 | 备注 |
|---|---|---|---|---|
| 按 Esc | HUD 游玩态 | Overlay push，背景调暗 60%（减少动态：即时） | 当前是否有未提交草稿、当前可保存性 | 草稿存在性影响退出/读档的丢失警告 |

**Exit**：

| 退出动作 | 目标 | 转场 | 返回/保存数据 | 备注 |
|---|---|---|---|---|
| 继续 (Resume) | HUD（恢复原态） | Overlay pop，恢复亮度 | 无（世界状态不变，草稿保留） | Esc 亦触发 |
| 保存 | SaveGameScreen → 回暂停 | push | 选定 slot；经存档端口原子写 | 覆盖走 P9；写中给反馈，失败给可行动原因 |
| 读取 | LoadGameScreen → HUD | replace | 选定 slot | 有未保存进度先 P9 警告；不兼容/损坏走错误态 |
| 设置 | SettingsScreen → 回暂停 | push | 无 | 改动即时存玩家档 |
| 退出到主菜单 | MainMenuScreen | replace（经 P9） | 无 | 有未保存进度提示保存/丢弃；默认焦点安全项 |
| 退出到桌面 | 桌面 | 经 P9 | 无 | 同上 |

| 区 | 描述 | 约占 | 滚动 | 溢出 |

| 组件 | 类型 | 区 | 用途 | 必需 | 复用 |

## 6. States & Variants

| 状态 | 触发 | 视觉变化 | 行为变化 | 备注 |
|---|---|---|---|---|
| 默认（无草稿） | Esc 暂停、无未提交草稿 | 态摘要仅显示日/期限 | 全项可用 | 常见态 |
| 有未提交草稿 | 暂停时存在 PlanDraft（P4） | 态摘要显示「未提交草稿：[名]」| 退出/读取项触发额外丢失警告（P9） | 防止误丢草稿 |
| 保存中 | 选保存并确认槽 | 加载指示 + 文字「保存中」 | 仅取消可用 | 原子写；>500ms 才显示 |
| 保存失败 | 存档端口写失败 | 错误态 + 可行动原因（非「出错了」） | 重试/返回 | 不部分写入（ADR-0005） |
| 读取/退出确认 | 选读取/退出且有未保存进度 | P9 模态：说明将丢失什么、是否可逆 | 背景锁定 | 默认焦点=取消 |

## 8. Data Requirements

| 数据 | 来源系统 | 更新频率 | 拥有者 | 格式 | 缺失处理 |
|---|---|---|---|---|---|
| 当前日/时段/主期限 | 时间系统(gdd-001) | 暂停打开时快照 | Domain 时间 | {日, 时段, 期限列表} | 无期限→只显日时段 |
| 未提交草稿存在性 | 战前准备(gdd-009) | 暂停打开时 | Domain（草稿态 P4） | bool + 草稿名 | 无→默认态 |
| 存档槽列表/元数据 | 存档端口(SiegeSaveSerializer) | 进入保存/读取屏时 | 存档子系统 | 槽元数据 + 兼容标志 | 空→保存可建新槽，读取显空态 |
| 当前可保存性 | Application | 暂停打开时 | Application | bool（如战役结算中途可能限制） | 不可保存→保存项禁用+原因标签 |

> 本屏只读上述快照 + 触发保存/读取/退出事件；不直接写权威状态（保存经存档端口，退出/读档经 Application）。

## 9. Events Fired

| 玩家动作 | 事件 | 载荷 | 接收系统 | 备注 |
|---|---|---|---|---|
| 确认保存 | SaveGameRequested | {slot_id, overwrite:bool} | 存档端口 | 原子写；成功/失败均反馈 |
| 确认读取 | LoadGameRequested | {slot_id} | Application（经存档端口） | 先 P9 丢失警告（如有草稿） |

| 确认退出到桌面 | QuitToDesktopRequested | {} | App 生命周期 | 经 P9 |
| Resume | GameResumed | {} | HUD 状态机 | 恢复原态 |

## 12. Screen-Level Accessibility Requirements

**对比度**（art-bible §4，实测优先）：

| 文本元素 | 背景 | 要求 | 现估 | 通过 |
|---|---|---|---|---|
| 「暂停」标题（山水墨） | 宣纸色面板 88% | 4.5:1 | ~9:1 | [x] |
| 态摘要/草稿提示（山水墨/淡墨灰） | 面板 | 4.5:1 | ~9:1（淡墨灰须实测） | [ ]（淡墨灰不过则改山水墨） |
| 继续 primary 焦点态 | 朱批朱红 | 大字 4.5:1 / 图形 3:1 | 需实测 | [ ] |
| danger 按钮标签 | 按钮态色 | 4.5:1 | 需实测 | [ ] |

**色盲安全**：菜单纯文字 + 形状；danger（退出）靠 danger 形状变体 + 位置 + P9 确认，非仅颜色（P3）。草稿提示靠文字 + 图标。

**焦点顺序**：1 继续 → 2 保存 → 3 读取 → 4 设置 → 5 退出到主菜单 → 6 退出到桌面 →（循环）。P9 模态打开时焦点陷入模态，默认落在安全项（取消）。

**屏幕阅读器播报**：

| 状态变化 | 播报 | 时机 |
|---|---|---|
| 暂停打开 | 「游戏已暂停。当前 [日/时段]。[有未提交草稿：名 / 无草稿]。继续 已选中。」 | 焦点落定 |
| 聚焦退出项 | 「[退出到主菜单/桌面]。破坏性操作，需确认。」 | 焦点到达 |
| P9 打开 | 「确认：[将发生什么 + 是否可逆 + 草稿处置]。取消 已选中。」 | 模态渲染 |
| 保存成功/失败 | 「已保存到 [槽]。」/「保存失败：[可行动原因]。」 | 事件确认后 |

**文本缩放**：100/125/150%；模态面板高度随文本自适应，150% 下不截断不重叠；态摘要草稿名超长截断 + tooltip（art-bible §7.1）。

**减少动态**：§10 全部转场即时；背景暗化保留（静态，非动画），属可用性必要（区分暂停态）。

**认知负荷**：态摘要（1）+ 菜单（1）= 2 信息流，低。安心锚（显示「世界未变」）主动降低焦虑。

## 13. Localization Considerations

- 全文本来自本地化字符串，无图内文字；中文基准；容忍 +40% 扩展。

| 文本 | 中文基准 | 上限 | RTL | 溢出 | 风险 |
|---|---|---|---|---|---|
| 标题「暂停」 | 2 字 | 12 | 居中 | 不截断 | 低 |
| 菜单项（「退出到主菜单」） | 6 字 | 24 | 镜像 | 按钮宽自适应，缩字号 ≥90% 后截断 | 中（德文长） |
| 态摘要/草稿提示 | ~12 字 | 40 | 右对齐 | 草稿名截断+tooltip | 中 |
| P9 后果说明 | ~20–40 字 | 120 | 右对齐换行 | 模态内换行不截断 | 低 |

## 14. Acceptance Criteria

**布局/渲染**
- [ ] 1080p/1440p/4K 下面板居中、无重叠/截断
- [ ] 全部状态（默认/有草稿/保存中/保存失败/退出确认）正确渲染
- [ ] 背景世界正确暗化 60% 且静止（验证：暂停期间无任何权威状态推进）

**输入**
- [ ] 全部项纯键盘可达；Esc 即时 Resume，绝不从屏内直接退游戏
- [ ] Gamepad（D-Pad + A/B）可达
- [ ] 暗化背景点击不误关菜单

**事件/数据**
- [ ] 保存为原子写：失败不留部分状态（ADR-0005），给可行动原因
- [ ] 有未提交草稿时，退出/读取均先经 P9 明确草稿处置
- [ ] 本屏不直接写权威状态（保存经端口、退出/读档经 Application）
- [ ] 读取不兼容/损坏存档走错误态，不崩溃、不部分加载

**无障碍**
- [ ] 全部文本达 §12 对比度（实测）
- [ ] 破坏性退出不靠颜色单一通道，且必经 P9（默认焦点安全项）
- [ ] 屏幕阅读器播报暂停态、草稿存在性、P9 后果
- [ ] 减少动态下转场即时（背景暗化静态保留）
- [ ] 文本 150% 无溢出/重叠

**本地化**
- [ ] 无文本溢出（含最长译文）
- [ ] 全部文本走本地化字符串
