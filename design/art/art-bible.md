## Section 1 — 视觉身份声明

### 1.1 核心视觉规则

**每一帧画面都是军府案上那张正在被批注的活图卷：底层是手绘地貌与纸张质感，中层是批注、印记与修改痕迹，顶层是人物与事件带来的情绪温度——三层共存，信息语义精确，画面可以写意，但不能模糊。**

这条规则是所有视觉决策的唯一锚点。当两个方案之间存在争议时，选择更接近"一张被实际使用的军事图卷"的那个：它有磨损、有修改、有多人的笔迹，但每一条线都有明确目的。

### 1.2 支撑原则

**原则 A：信息即美术**

军情可读性是视觉质量的第一标准，而非装饰完整度。

**锚定支柱**：视觉支柱「军情清晰」× 游戏核心幻想（玩家自己创造条件，而非系统赋予技能）

任何视觉元素——无论多精美——如果遮挡了地形通道、模糊了补给线连续性，或令三类信息语气（事实/推测/命令）相互混淆，都必须修改甚至移除。

**设计测试**：当某个装饰纹样使玩家在 5 秒内无法判断一条路线是"已确认通道"还是"推测通道"时，本原则要求删除或缩减该纹样，优先改善线条可读性。

---

**原则 B：笔迹分层语气**

三种视觉语气（事实·推测·命令）在形状、线条、颜色和排版上必须全时可区分，绝不因追求风格统一而压缩差异。

**锚定支柱**：视觉支柱「演义写意」× 游戏设计支柱「后果留痕」

图卷上有多种笔迹：制图者的稳定墨线（事实）、侦察手的潦草标注（推测）、主将的朱批箭头（命令）。三种笔迹共存于同一张纸上，但不会被混淆，因为每种笔迹都有自己的材料特征。本游戏的视觉语言须达到同等清晰度。

**设计测试**：当一条新 UI 元素无法被立即归类为"事实/推测/命令"三者之一时，本原则要求在落稿前先确定其语气归属，再决定线条、虚实和颜色。

---

**原则 C：历史质感承载演义气势**

物质层（地貌、器物、服饰）须有历史可信的质感作为基底；演义式的气势和夸张通过构图、光照、事件叙事方式注入，而非通过违反物质规律的视觉特效。

**锚定支柱**：视觉支柱「人物可信」×「演义写意」× 设计边界（历史可信性与演义界限，参见 art-direction.md §历史与演义边界）

一件铠甲的纹路须有实物依据，但将领登场时的构图仰角与光源角度可以充满戏剧性。法术光球不出现，但狂风骤雨的地图变色可以渲染感十足。

**设计测试**：当某个视觉效果（光效、粒子、色调）只能用"这是三国演义嘛"解释、而无法用"战场上可能有这种自然现象"解释时，本原则要求将其替换为天气、火光、尘埃、旗号等有物质依据的形式。

## Section 2 — 氛围与气氛

五种状态须在视觉上可区分：相邻状态之间的切换应有明显的氛围转变信号，而非仅靠 UI 面板内容的变化。每种状态的光照、饱和度、颜色重心须有可测量的差异。

---

### 2.1 生活 / 观察

**情绪目标**：在场、从容、细察——玩家以人物身份生活在世界中，处理日常职责与关系，感受到江山辽阔与时局暗涌并存。

**光照特征**：
- 时段：晨光或午前，暖调斜射偏东
- 冷暖：中暖偏黄（色温 5500–6500K 参考值），宣纸底色在此状态下最明亮
- 对比：低对比度；阴影柔和，光源不强调方向性压迫

**氛围形容词**：沉静、宽阔、温厚、有纹理、可呼吸

**承载氛围的具体视觉元素**：城镇功能分区在低强调色下细节可见（屋顶、炊烟、墙垣损痕叠加层处于低不透明度）；人物肖像表情平稳或若有所思；地图图卷铺展、无朱批路径叠加；补给线以淡墨灰点线呈现，不触发警戒色。

---

### 2.2 判断 / 布局

**情绪目标**：聚焦、权衡、紧迫中的冷静——玩家在不完全信息下推演形势，如在案前展开情报批注。

**光照特征**：
- 时段：傍晚至夜烛，暖调集中，光源方向朝向屏幕正前方或微俯斜
- 冷暖：烛光暖调中心，外围偏冷（形成内暖外冷的渐变晕染）
- 对比：中对比；光源集中在图卷/面板区域，地图边缘压暗约 20–30% 亮度

**氛围形容词**：专注、压缩、半明半暗、蓄力、批注式

**承载氛围的具体视觉元素**：图卷局部放大至战役区域；侦察虚线（推测层，推测蓝灰色）出现并明显区分于底层；情报面板前置，城市背景退入阴影；军师图标在场；信息来源标记（P1 四层）视觉权重提升。

---

### 2.3 行动 / 承诺

**情绪目标**：承诺落地的重量感——命令已下，资源已投，无法收回，玩家感受到决策的不可逆性。

**光照特征**：
- 时段：日落或清晨出发，强调极端色温（深橙落日或寒白晨光）
- 冷暖：双极化——图卷核心保留暖调，命令路径（朱批色）引入红橙强调
- 对比：高对比；已承诺的命令路径在地图上明亮跳出，背景相对压暗

**氛围形容词**：决断、沉重、朱批、路径感、不可逆

**承载氛围的具体视觉元素**：朱批朱红（命令色）在图卷上明显出现——箭头实线化、路径节点锁定动画、印信封存效果；人物肖像表情转向凝重或坚定；资源消耗数字以醒目方式更新（P4 已提交态视觉）；草稿水印消失，转为"已承诺"标签。

---

### 2.4 战争 / 应变

**情绪目标**：不完全信息下的紧张执行——玩家无法全知，只能在变化的局势中调整，感受兵法沙盘的不确定性张力。

**光照特征**：
- 时段：由行动时段与天气系统共同决定（晨雾/烈日/夜袭三种基准）
- 冷暖：
  - 晨雾：冷蓝灰，对比低，细节模糊
  - 烈日：暖黄高对比，阴影深重
  - 夜袭：极暗背景，仅局部高亮（火把、旗帜）
- 对比：高对比来自天气而非美化滤镜；迷雾区对比急降

**氛围形容词**：焦灼、多变、局部可见、迷雾压迫、气候驱动

**承载氛围的具体视觉元素**：迷雾/视野遮挡在地图上明显激活（P1 未知层），边缘柔化羽化；风向箭头与天气图标置于前景层；敌方信息仅以报告/推测层呈现（P10，无精确真值）；玩家部队图标处于执行状态（区别于草稿状态，P4）；期限警报以警戒琥珀色出现于时间状态条（P8）。

---

### 2.5 战果结算 / 失败延续

**情绪目标（胜/利）**：不是欢庆，而是条件应验的满足感——玩家认出"那是我提前布好的局"。

**情绪目标（败/延续）**：惨败后的可玩感——局面艰难，但仍有路可走（投效、撤退、东山再起）。

**光照特征**：
- 时段：结战后的时间停顿（由战斗时段决定，夕阳或黎明均可）
- 胜：光照稍暖，整体饱和度轻微回升，城镇视角短暂出现
- 败：光照偏冷灰，灰度上升，仅某些区域高亮（强调"尚有余地"而非"一片废墟"）
- 对比：中低，情绪趋于沉淀，不做视觉庆典式闪烁

**氛围形容词（胜）**：应验、落定、从容复盘、清明

**氛围形容词（败）**：沉降、余地、不甘、延续

**承载氛围的具体视觉元素**：战后复盘因果链（P5 逐步展开）；地图标记状态更新（城损叠加层、旗帜变更、补给线断裂视觉化）；人物关系变化标识（声望损益、受伤、疲惫）出现在肖像右下角叠加区；失败状态不切至空白界面，图卷仍在展开，新局面标记已起。

## Section 3 — 形状语言

### 3.1 人物半身像剪影哲学

**比例框**：3∶4（竖向），标准参考尺寸 256×340px。全部肖像在此比例下绘制，不拉伸适配其他容器；容器裁切时从中上方裁，保留头部与肩部。

**缩略图可读性要求**：

| 尺寸 | 可读性要求 |
|---|---|
| 64×85px（列表卡） | 在 2 秒内识别主要「阵营–职能」类型（武将/谋士/文官/君主/女性） |
| 128×170px（面板） | 区分具体人物原型，无需文字标签辅助 |
| 256×340px（全肖像） | 表情细节、伤病状态、服饰纹路可读 |

关键区分区域集中在**头部上方 1/3**（头饰/盔缨/发髻形态）和**肩线宽度**（武将宽、谋士窄）。这两个区域须在 64px 宽度下保持轮廓干净，不被装饰细节淹没。

**身份与原型区分特征**：

| 原型 | 剪影关键特征 | 形状词汇 |
|---|---|---|
| 武将 | 宽肩铠甲轮廓，头盔盔缨向上延伸，下颌线硬朗 | 宽、厚重、扩张 |
| 谋士 | 窄肩，纶巾或高冠，手持羽扇剪影水平延伸 | 竖向、精炼、水平延展 |
| 君主 | 高冠居中，整体对称，肩线略宽于谋士 | 端正、对称、权威 |
| 文官 | 幞头或进贤冠，双肩平直，衣裾线垂直 | 平、方整、规则 |
| 女性人物 | 发髻轮廓独特（区别于全部男性头饰类型），领口形状和男性明显不同 | 曲线、细腻、区别型 |

**状态变体规则**：同一人物在伤病、疲惫、声望破损状态下，**肖像底图不更换**（避免 SKU 爆炸）。通过叠加层（`charfx_*` 类图标）在肖像右下角固定区域标注状态，区域尺寸 24×24px，最多同时叠加 3 个图标，超出时优先显示最高严重度。

---

### 3.2 地图 / 战役层几何

**区域节点**：
- 形状不规整，反映真实地貌轮廓（山地=三角形组、平原=宽阔多边形、河湖=波曲围合）
- 城市节点：八边形底座，边长不等，历史官印感；不用圆形（圆形保留给特殊事件/关键人物标记，避免歧义）
- 节点内部图标区固定于中央，文字标签居于节点正下方（不遮挡节点面积 60% 以上）

**路线 / 道路**：
- 主路（已确认通道）：饱满笔触，边缘略有压感变化，模拟毛笔线条；宽度 4–6px（@1x）
- 小路（可疑/推测路线）：细线，边缘更飘逸，宽度 2–3px（@1x）；与推测蓝灰色配合使用
- 线条不用纯直线；允许微曲以表达地形蜿蜒感，但信息读取优先于装饰弧度

**视野 / 迷雾**：
- 视野边界：软边，8px 羽化渐变；不用硬边裁切
- 迷雾区：半透明白雾叠加层（60–70% 不透明度，保留底图隐约可见），叠加细腻纸纹噪波，模拟图卷空白区
- 伏击点/危险标记：尖锐 V 形或叉形（锐角传达危险），与路线的圆润曲线形成对比

**补给线**：
- 活跃补给线：间隔点线（2px 点 + 4px 间距），沿路线走向；颜色使用矿土褐
- 断裂补给线：叉号图标叠加 + 警戒琥珀色强调；同时配文字提示（P3 冗余编码）

### 3.3 UI 形状语法

**三类语气的线条规则**（直接对应 art-direction.md 三语气体系与 P1 四层标识）：

| 语气 | 线型 | 粗细 | 颜色 | 附加形状 |
|---|---|---|---|---|
| 事实 | 实线 | 2px | 山水墨 H25 S8 L18 | 实心点、清晰标签 |
| 推测 | 虚线 4px-on / 4px-off | 1px | 推测蓝灰 H215 S18 L55 | 问号图标、来源标记 |
| 命令 | 实线 + 箭头 | 3px | 朱批朱红 H12 S72 L40 | 印信图形、路径箭头 |

**圆角语义**：

| 元素类型 | 圆角半径 | 语义 |
|---|---|---|
| 信息卡片 / 数据面板 | 4px | 温和、中性、内容为主 |
| 确认/来源标签 | 2px | 轻微区分，非强势 |
| 危险警告 / 错误提示 | 0px | 锐角传达紧迫，不柔化 |
| 图标容器 | 2px | 统一标准 |
| 势力标记（曹魏） | 0px | 体现军纪严整 |
| 势力标记（蜀汉） | 4px | 五峰形顶边，见 4.4 |
| 势力标记（东吴） | 波曲边 | 波浪顶边，见 4.4 |

**装饰角线规则**：UI 面板允许在四角使用墨角装饰纹，固定为 12×12px 角标，**纯装饰，不承载信息**（符合 P3——纯装饰色彩无需冗余编码）。装饰纹不得遮挡面板内任何文字或数值。

**线条粗细层级总则**：

| 层级 | 用途 | 粗细（@1x） |
|---|---|---|
| L1 | 屏幕区域主分隔 | 3px |
| L2 | 面板边框 | 2px |
| L3 | 内容分隔线 | 1px |
| L4 | 注释/辅助/推测线 | 1px（虚线） |

### 3.4 主形状 vs 支撑形状

**主形状（吸引视觉注意，前景层）**：
- 人物肖像：最高局部对比度，色彩最暖
- 命令路径（朱批）：唯一使用高饱和红橙的线条元素；在地图上拥有最高视觉权重
- 战术焦点节点（被选中/悬停区域节点）：金白色选中光晕

**支撑形状（退入背景，底层/中层）**：
- 地形底图：低饱和度，高统一性，颜色接近宣纸与矿土褐
- UI 面板背景：半透明宣纸色（85–92% 不透明度，使地图底图隐约可见）
- 推测层叠加（虚线、淡蓝）：视觉权重显著低于事实层

**格式塔原则应用**：

| 原则 | 应用方式 |
|---|---|
| 接近性 | 同一势力的节点和路线保持势力色调，强化"领土聚合感" |
| 相似性 | 同类信息（补给线/道路/命令路径）使用同类线条形状，仅以线型/颜色区分语气 |
| 图底关系 | 地图底图始终是底；UI 面板是浮层；人物肖像是最前景——层级不得颠倒 |
| 封闭性 | 区域节点的围合轮廓暗示"可进入的地形单元"；开放轮廓暗示"边界/争议区" |

### 4.5 色盲安全

**高风险色对与冗余备份**：

| 色对 | 风险类型 | 问题描述 | 冗余备份方案 |
|---|---|---|---|

## Section 5 — 动效规范

> 动效契约的**行为基线**在 `design/ux/interaction-patterns.md` §Animation Standards（P12 可跳过约束、timing 表）已定义；本节补充**视觉层具体规格**：运动哲学、缓动曲线、状态转场帧数、强调动效。
> **核心动效哲学**：动效服从「军府图卷」隐喻——动作如同手在纸上移动笔、揭开覆纸、按下印信，而非屏幕特效。所有动效有物质依据（§1.2 原则 C），无凭空发光、无弹性过冲式卡通感。**信息呈现优先于动效表演**（§1.2 原则 A）：动效不得延迟玩家读取关键军情。

### 5.1 运动原则

- **纸面运动，非屏幕运动**：元素的出现 = 笔迹落纸 / 覆纸揭开 / 印信按下；消失 = 墨迹淡入纸色 / 覆纸盖回。位移走短距、走克制弧线，不做大幅飞入。
- **语气决定动效强度**：三语气（事实/推测/命令，§3.3）的动效权重递增——事实安静即时，推测轻柔渐显，命令有「落定」的重量感。
- **不可逆即有重量**：命令承诺（行动状态，§2.3）的动效是全游戏最「重」的——印信封存、路径锁定，须让玩家体感「这一下收不回」。
- **天气与战争状态的动效来自系统，不来自滤镜**（§2.4）：迷雾流动、风向飘动是世界状态可视化，不是装饰。

### 5.2 缓动曲线规范

| 曲线名 | CSS/Unity 近似 | 用途 | 手感 |
|---|---|---|---|
| 墨入（Ink-In） | ease-out（cubic 0.0,0.0,0.2,1.0） | 信息/面板出现、笔迹落纸 | 快起慢收，如墨被纸吸入 |
| 纸落（Settle） | ease-in-out（0.4,0.0,0.2,1.0） | 面板移动、图卷平移 | 平稳，无过冲 |
| 印定（Stamp） | ease-in 末段轻微回弹 ≤4%（0.3,0.0,0.0,1.0 + 微回） | 命令承诺、印信封存 | 下压有力、定格干脆 |
| 雾散（Fade-Field） | linear 或极缓 ease | 迷雾/视野/天气叠加层渐变 | 无方向感，缓慢均匀 |

**禁用曲线**：弹性过冲 >4%（卡通 bounce）、回旋（back ease 大幅）、屏幕抖动作为常规反馈（仅限战败/失城等重大叙事节点，且须可在「减少动态」下关闭）。

### 5.3 状态转场帧数（@60fps 基准）

时长以 `interaction-patterns.md` §Animation Standards 的反馈分层为准，本表给视觉帧数落地：

| 动效类别 | 时长 | 帧数@60fps | 缓动 | 备注 |
|---|---|---|---|---|
| 即时反馈（hover/选中高亮） | 80ms | ~5 帧 | 墨入 | P2 即时反馈；不可省略 |
| 信息卡片/面板出现 | 150ms | ~9 帧 | 墨入 | 内容须在动效结束前已可读 |
| 图卷平移/缩放至战役区 | 250ms | ~15 帧 | 纸落 | §2.2 判断状态进入 |
| 推测层虚线渐显 | 200ms | ~12 帧 | 墨入 | 透明度 0→100%，区别于事实层的即时 |
| 命令路径锁定（朱批落定） | 300ms | ~18 帧 | 印定 | §2.3；全游戏最重动效 |
| 印信封存（已承诺标记） | 350ms | ~21 帧 | 印定 | 含 ≤4% 回弹定格 |
| 五状态间氛围转场（光照/饱和度） | 400ms | ~24 帧 | 纸落 | §2 相邻状态切换信号 |
| 迷雾/视野/天气叠加渐变 | 500ms | ~30 帧 | 雾散 | §2.4；非阻塞，玩家可同时操作 |
| 战后因果链逐步展开（P5） | 每步 200ms | ~12 帧/步 | 墨入 | §2.5；可整体跳过（P12） |

**全局约束**：
- 任何**阻塞玩家输入**的动效 ≤ 350ms。超过则改为非阻塞（玩家可在动效进行中继续操作）。
- 状态转场动效不得叠加超过 2 层同时进行，防止视觉噪声压过军情读取。
- 全部动效遵循 P12：受「减少动态」开关控制（见 §7.3）——开启后所有非必要动效退化为即时切换，仅保留 ≤80ms 的功能性反馈。

### 5.4 强调动效（克制清单）

仅以下情形允许强调动效，其余一律安静处理：

| 触发 | 强调动效 | 强度上限 |
|---|---|---|
| 命令承诺落定 | 印信下压 + 路径实线化 | 重（印定曲线，可回弹 ≤4%） |
| 期限/断粮警告激活 | 警戒琥珀边框脉冲 2 次后驻留 | 中（脉冲周期 600ms，仅 2 次，非常驻闪烁） |
| 条件链应验（胜负关键节点） | 相关节点单次高亮扩散 | 中（§2.5「应验」满足感，非庆典闪烁） |
| 侦察新情报抵达 | 来源图标单次墨入 + 时间戳更新 | 轻 |

**绝对禁止**：常驻闪烁/呼吸光作为信息载体（违反 P3，且对光敏感用户有害——见 §7.3）；用动效区分敌我或语气（动效是辅助，形状/颜色/标签才是主编码）。

## Section 6 — 音频视觉协同

> 音频视觉等价契约的**行为基线**在 `design/ux/interaction-patterns.md` §Sound Standards 已定义（每个关键音频反馈须有等价视觉反馈，反之亦然——服务于无障碍与无声游玩）。本节补充**视觉侧**的具体对应设计。完整声音设计待 audio 团队介入后单独成文（`design/audio/`）；本节只锁定「视觉须与音频对齐」的契约，不规定音色。

### 6.1 音视等价原则（无障碍硬约束）

**任何承载信息的音频，必须有等价的视觉反馈；任何承载信息的视觉强调，应有可选的音频伴随。** 游戏须在**完全静音**下可全程通关，无信息丢失（对应 `accessibility-requirements.md` 听觉无障碍基线）。

### 6.2 音频触发 → 视觉反馈对应表

| 游戏事件 | 音频角色 | 等价视觉反馈 | 视觉为权威？ |
|---|---|---|---|
| 命令承诺落定 | 印信「咚」/ 落笔声 | 印信封存动效 + 朱批路径实线化（§5.3） | ✅ 视觉权威，音频仅增强 |
| 期限/断粮警告 | 低沉警示音 | 警戒琥珀脉冲 + 文字「期限将至/补给告急」（P8） | ✅ 视觉权威 |
| 侦察新情报抵达 | 轻「沙」纸响 | 来源图标墨入 + 时间戳刷新（§5.4） | ✅ 视觉权威 |
| 条件链应验 | 沉稳定音（非高亢） | 节点高亮扩散（§2.5） | ✅ 视觉权威 |
| 回合/时段推进 | 更漏/钟点声 | 时间状态条推进（P8） | ✅ 视觉权威 |
| 非法操作/校验失败 | 短促否定音 | 元素轻震 1 次 + 红框 + 错误文字（P2/P3） | ✅ 视觉权威 |
| 环境氛围（五状态） | 状态环境底噪（市井/案前/战阵） | §2 光照·饱和度·色彩重心切换 | 双轨，互不依赖 |

**设计锁**：上表所有「视觉权威」项中，**视觉反馈是信息主载体，音频是冗余增强**。删除音频不得造成信息丢失。反向（仅有音频无视觉）一律禁止。

### 6.3 视觉对音频的节奏让位

当音频有明确节拍（如更漏推进、战鼓）时，对应的视觉脉冲/推进动效**对齐音频节拍点**，避免音画错位的廉价感。但视觉**不依赖**音频时钟驱动——音频关闭时视觉按自身 §5.3 帧数独立运行。

## Section 7 — 无障碍视觉标准

> 本节整合 `design/accessibility-requirements.md`（WCAG 2.1 AA 基线，MVP 15 项必达）的**视觉落地规格**。对比度基线见 §4.3，冗余编码见 §4.4/§4.5（P3），动效可关闭见 §5.3（P12）。本节补充：文本缩放、减少动态、焦点指示器、色彩外编码总则。

### 7.1 文本缩放

| 项目 | 规格 |
|---|---|
| 缩放档位 | 100%（默认）/ 125% / 150% 三档，全局生效 |
| 缩放对象 | 全部 B1/B2/C1/N1/N2 级文字（§8.2 字号体系）按比例放大 |
| 布局响应 | 容器须支持文本溢出处理：面板高度自适应或内部滚动，**不截断、不重叠** |
| 下限保护 | 150% 档下 C1（地图标签）须仍不超出节点可用区；超出则该标签降级为图标 + tooltip |
| 验证 | 三档 × 三分辨率（1080p/1440p/4K）= 9 组合须目视验证关键屏无截断（主菜单/HUD/暂停，见 `design/ux/`） |

文本缩放为 MVP 必达项（`accessibility-requirements.md`）。最小绝对字号下限仍受 §8.2 约束（100% 档 C1 = 11px@1080p）。

### 7.2 焦点指示器（键盘可达，PC 主输入）

平台为 PC、键鼠为主（`technical-preferences.md`），所有交互元素须键盘可达、焦点可见：

| 项目 | 规格 |
|---|---|
| 焦点框 | 2px 实线描边，朱批朱红 H12 S72 L40（与背景对比 ≥ 3∶1，§4.3） |
| 焦点框偏移 | 元素外缘 +2px，不遮挡内容 |
| 对比保障 | 焦点色与**相邻底色**对比 ≥ 3∶1；浅底用朱批朱红，深底（战争/夜袭状态）切换为金白高亮 |
| 不依赖颜色 | 焦点同时以「描边 + 轻微缩放 ≤2%」双编码，色盲/低视力用户可辨（P3） |
| 焦点顺序 | 与视觉阅读顺序一致（左→右、上→下、信息层先于命令层）；由 UX 屏规格逐屏定义 |
| 禁止 | 仅悬停（hover-only）才出现的可交互状态（`technical-preferences.md` 平台硬约束） |

### 7.3 减少动态（Reduce Motion）

| 项目 | 规格 |
|---|---|
| 开关 | 全局「减少动态」选项，MVP 必达（P12） |
| 开启效果 | §5.3 全部动效退化为即时切换；保留 ≤80ms 功能性反馈（hover/选中），保证可用性不损失 |
| 强制关闭项 | 任何脉冲/扩散/位移动效（§5.4）一律停用，改为静态状态呈现（如警告 = 静态琥珀框 + 文字，不脉冲） |
| 光敏保护 | 全游戏禁止 >3 次/秒的闪烁（WCAG 2.3.1）；战败/失城等含强动效的叙事节点须可被本开关降级 |

### 7.4 色彩外编码总则（P3 汇总）

承接 §4.3/§4.4/§4.5，统一声明：**任何信息不得仅靠颜色传达。** 每处颜色编码须至少叠加以下一种冗余：

| 信息维度 | 颜色 | 冗余编码 1 | 冗余编码 2 |
|---|---|---|---|
| 三语气（事实/推测/命令） | §3.3 配色 | 线型（实/虚/箭头） | 图标 + 标签 |
| 三势力（魏/蜀/吴） | §4.4 旗色 | 纹样（网格/山纹/波浪） | 边框形状 + 文字印记 |
| 风险/期限/断粮 | 警戒琥珀 | 图标（叉号/沙漏） | 文字提示 |
| 情报层级（事实/报告/推测/未知） | §4.3 配色 | 线型 + 来源图标 | 时间戳 / 置信标注 |

### 8.1 2D 精灵分辨率层级

所有 @1x 分辨率基于 1080p（1920×1080）参考显示器。@2x 版本用于 1440p 及 4K 显示。

| 资产类别 | @1x 分辨率 | @2x 分辨率 | 图集策略 | 备注 |
|---|---|---|---|---|
| 地图底图（手绘图卷纹理） | 2048×2048 | 4096×4096 | 不进图集，独立纹理 | 每地图场景 1 张；内存影响见 8.5 |
| 区域节点图标 | 64×64 | 128×128 | 进 MapIcons Atlas（2048×2048） | 含 selected / hover / disabled 三态变体 |
| 路线 / 道路元素（瓦片） | 16×16 | 32×32 | 进 MapTile Atlas（2048×2048） | 含实线/虚线/点线变体 |
| 人物半身像 | 256×340 | 512×680 | 按势力分组，每组 1 个 2048×2048 Atlas | 每势力 ≤15 幅；状态叠加独立精灵 |
| 人物状态叠加层（伤/疲/声望图标） | 24×24 | 48×48 | 进 CharStatus Atlas（1024×1024） | 最多 3 个同时显示于肖像右下角 |
| UI 图标（通用） | 32×32 | 64×64 | 进 UIIcons Atlas（2048×2048） | 含信息层级图标（事实/报告/推测/未知） |
| 势力旗帜 / 徽标 | 48×48 | 96×96 | 进 FactionBadge Atlas（1024×1024） | 含三势力旗色+纹样版本 |
| 地图叠加层（天气 / 视野 / 控制） | 512×512（柔边渐变） | 1024×1024 | 不进图集，独立渲染层 | 按需动态加载，非常驻 |
| 事件插图（关键叙事） | 480×270（16∶9 内嵌） | 960×540 | 不进图集，Addressable 按需加载 | MVP 阶段 ≤5 幅 |
| 场景背景 | 1920×1080 | 3840×2160 | 不进图集，独立纹理 | 当前场景加载，非常驻 |

**@1x / @2x 运行时加载策略**：

Unity 6.3 UI Toolkit 在 Panel Settings 中通过 Scale Mode 支持分辨率感知缩放。建议配置：
- 1080p → 加载 @1x 图集
- 1440p → 加载 @2x 图集（地图底图可先用 @1x + 双线性放大，目视验证后决定是否升级）
- 4K → 强制 @2x 图集；地图底图使用 4096×4096 版本

### 8.2 字体与排版规范

**字体族方向**：

| 用途 | 类型 | 推荐方向 | 备注 |
|---|---|---|---|
| 中文正文 / 面板信息 | 衬线（宋体类） | 思源宋体（Source Han Serif CN）或方正宋体，OFL 授权 | 与图卷手绘质感协调 |
| 中文标题 / 章节名 | 衬线偏碑帖感 | 方正隶变简体或同类风格，须确认授权 | 演义气势；不用于 14px 以下 |
| 数值 / 状态数字 | 无衬线 | 思源黑体（Source Han Sans CN）或 Noto Sans SC | 数字辨识度优先，衬线数字在小尺寸下难读 |
| 英文 / 拉丁字符回退 | 无衬线 | Noto Sans，与思源黑体配对 | 本地化扩展备用 |

所有字体须通过 TextMeshPro SDF 渲染（Unity 6.3 推荐方案），以确保各分辨率下边缘清晰。

**字号体系（基于 1080p，单位 px；其他分辨率线性缩放）**：

| 层级 | 名称 | 1080p | 1440p | 4K | 字重 | 典型用途 |
|---|---|---|---|---|---|---|
| T1 | 大标题 | 28px | 37px | 56px | Bold 700 | 章节名、势力名 |
| T2 | 章节标题 | 22px | 29px | 44px | SemiBold 600 | 面板标题、人物名 |
| T3 | 小标题 | 18px | 24px | 36px | Medium 500 | 子分类、卡片标题 |
| B1 | 正文 | 15px | 20px | 30px | Regular 400 | 说明文字、情报正文 |
| B2 | 辅助正文 | 13px | 17px | 26px | Regular 400 | 次级说明、来源标注 |
| C1 | 注释 / 标签 | 11px | 15px | 22px | Regular 400 | 地图标签、时间戳 |
| N1 | 数值显示 | 16px | 21px | 32px | Medium 500，无衬线 | 兵力、资源数字 |
| N2 | 小数值 / 趋势 | 12px | 16px | 24px | Regular 400，无衬线 | 变化量、区间估计 |

**最小可读字号下限**：

| 分辨率 | 绝对下限 | 临界需目视验证 |
|---|---|---|
| 1080p | C1 11px | N2 12px（须在真实显示器验证） |
| 1440p | C1 15px | — |
| 4K | C1 22px | — |

低于下限的信息不渲染为文字，改用图标表达。

**行距与字距**：
- 中文正文行距：1.6 × 字号（B1 在 1080p 下行距 24px）
- 标题行距：1.3 × 字号
- 中文字符间距：保持字体默认（不人工调整）
- 同一面板内至多 3 级字重（如 Bold 标题 + Regular 正文 + Medium 数值）

### 8.3 格式与图集

**文件格式**：

| 资产类型 | 源文件格式 | Unity 导入后压缩 | 透明通道 | 原因 |
|---|---|---|---|---|
| 所有 2D 精灵（人物 / 图标 / UI） | PNG | BC3（DXT5） | 是（RGBA） | 保留透明 Alpha 渐变质量 |
| 地图底图（不透明） | PNG | BC7 | 否（RGB） | 无透明需求时 RGB 压缩效率更高 |
| 地图叠加层（柔边） | PNG | BC3（DXT5） | 是（RGBA） | 须保留 Alpha 渐变 |
| 字体 | TTF / OTF | TextMeshPro SDF 生成 | — | SDF 在各分辨率下清晰 |
| 事件插图 | PNG | Addressable Bundle 内 BC7 | 否 | 大图按需加载，优先压缩率 |

**Sprite Atlas 规则**（Unity Sprite Atlas v2，Unity 6.3）：

- 每个 Atlas 最大尺寸：2048×2048（@1x）/ 4096×4096（@2x）
- 同一 Atlas 内的精灵须来自同一功能层（UI 图标不与地图图标混合）
- Atlas 按显示层分组，减少 Overdraw：UI 一组 / 地图一组 / 人物一组
- Padding：4px per sprite（防止 UV 出血）
- 启用 Tight packing（Unity Atlas 默认，提高填充率）

### 8.5 纹理 / 内存预算

**总体约束**（来自 technical-preferences.md）：
- 峰值工作内存：8GB
- Draw Call 上限：≤ 1000 帧/游戏期间

**内存分配方向**：

| 层级 | 估算占用 | 说明 |
|---|---|---|
| Unity 运行时 + C# 堆 | ~1.5 GB | 引擎固定开销 |
| 游戏状态（模拟数据） | ~500 MB | 地图状态、人物数据、回合记录 |
| 音频流 | ~300 MB | 环境音 + 音效 |
| **纹理内存（见下方细分）** | **目标 ≤ 400 MB（@1x）/ ≤ 800 MB（@2x 部分启用）** | — |
| 操作系统 + 其他缓存 | ~300 MB–1.3 GB | 缓冲余量 |

**纹理内存细分目标**（压缩后估算）：

| 资产类别 | @1x 压缩后 | @2x 增量 | 加载策略 |
|---|---|---|---|
| 地图底图（1 张/场景） | ~16 MB | +48 MB | 当前场景加载 |
| MapIcons Atlas × 2 | ~8 MB | +24 MB | 常驻 |
| MapTile Atlas | ~4 MB | +12 MB | 常驻 |
| 人物肖像 Atlas（3 势力 × 2） | ~60 MB | +180 MB | 常驻当前势力；其余 Addressable 按需 |
| CharStatus Atlas | ~2 MB | +6 MB | 常驻 |
| UIIcons Atlas × 2 | ~16 MB | +48 MB | 常驻 |
| FactionBadge Atlas | ~4 MB | +12 MB | 常驻 |
| 地图叠加层（天气/视野等） | ~16 MB | +48 MB | 按需加载，非常驻 |
| 事件插图 ≤5 幅 | ~20 MB | +60 MB | Addressable 按需加载 |
| 场景背景 | ~8 MB | +24 MB | 当前场景加载 |
| **合计** | **~154 MB** | **+462 MB** | |

**理想值 vs 受限值的明确取舍**：

> **不存在实质冲突**：若在 4K 显示器上启用全部 @2x 资产，估算纹理内存约 616 MB，总工作内存约 2.9 GB，在 8GB 系统上仍有充足余量。Draw Call 方面，典型 2D 策略场景（UI + 地图 + 人物）估算约 50–150 draw calls，远低于 1000 上限。因此当前技术约束**不构成资产质量的实际瓶颈**。
>
> **真正的制约是制作成本**：高精度 @2x 资产（尤其是地图底图和全部人物肖像）的绘制工作量，而非内存或渲染性能。
>
> **MVP 阶段策略**：@1x 资产优先，验证玩法后再制作 @2x。地图底图先以 @1x（2048×2048）+ Unity 双线性放大评估 4K 视觉质量，不可接受时再提升。此策略与 art-direction.md §MVP 资产边界原则一致：在玩法验证完成前不批量生产高精度资产。

## Section 9 — 制作管线与签核

### 9.1 资产生产工作流

```
GDD/UX 规格 → /asset-spec（逐资产规格 + 生成提示）→ entity-inventory.md（清单）
   → 草模/占位（灰度，验证布局与可读性）→ 一审（信息可读性，§1.2 原则 A）
   → 精绘 @1x → 无障碍验证关卡（§7.5）→ 二审（AD 签核）→ 入图集（§8.3）→ 入库
   → @2x 仅在玩法验证后按需补（§8.5 MVP 策略）
```

**关键关卡**：
- **占位优先**：所有屏幕/系统先用灰度占位资产验证布局与信息可读性，再投入精绘成本（呼应 vertical slice「功能原型而非完整美术」原则）。
- **可读性一审先于美观二审**：任何资产先过「信息能否在目标尺寸下读取」（§3.1 缩略图可读性、§4.3 对比度），再谈风格。
- **无障碍是入库前置**，非事后补丁（§7.5 六项验证）。
