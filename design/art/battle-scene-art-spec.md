## 1. 战场底图（逐地形战场背景）

### 1.1 功能定位

单张全屏横版背景，作为「方格战场」（5 区网格卡片 UI，已定布局方案）的底图背景层，方格卡片叠加其上。因此**构图必须留白克制、避免细节抢戏**——网格 UI 会覆盖画面中/下段大部分区域，背景的角色是"环境气氛"而非"需要玩家凝视的细节画面"。这与已锁定的坚城/遮蔽/主菜单背景一贯的留白构图哲学完全一致（延续 `midjourney-prompt-pack.md` §7 系列的构图裁决标准）。

### 1.5 平原（Plain）不复用「粮道」资产的判断依据（已更正）

## 2. 区域节点框 与 邻接连线（新增资产族——此前遗漏）

> 对齐 GDD §6：`mapnode_zonebattle_[zoneid]_[terrain?]_64` / `maproute_zonebattle_adjacency_16`。GDD 定义"5 区通用节点形状 + 正面区随地形换皮肤"，本节定义节点框与连线的具体视觉。

### 2.1 节点框（`mapnode_*`）

- **通用形状**：5 区（正面/侧翼/粮道/遮蔽/预备）共用同一基础卡片形状——圆角矩形战略图版（art-bible §3.2 已定义的地图节点视觉语言的战场层延伸），旧绢米底色 + 1.5px 山水墨描边，四角轻微磨损纸边效果（呼应"军府活图卷"整体质感）。
- **正面区地形皮肤**：仅正面区节点在通用形状基础上，右上角叠加一枚小型单色（山水墨，不用彩色，避免与旗帜阵营色抢主次）地形图腾蚀刻纹样，5 选 1：坚城=城楼剪影小纹、隘口=双峰夹隘剪影小纹、渡口=水波纹小纹、平原=平野地平线小纹、遮蔽=林冠剪影小纹。其余 4 区节点不挂地形纹样（其地形固定，不作为战术信息需要强调）。
- **命名**：`mapnode_zonebattle_[zoneid]_64.png`（通用 4 区：`flank`/`supply`/`cover`/`reserve`）+ `mapnode_zonebattle_front_[terrain]_64.png`（正面区 5 皮肤：`fortified`/`pass`/`ford`/`plain`/`cover`）。

  **TODO**：`[zoneid]` 的确切代码侧字符串取值（本节假定为 `front`/`flank`/`supply`/`cover`/`reserve`，对应 GDD 中文角色名"正面/侧翼/粮道/遮蔽/预备"）需与 unity-specialist 核对 `ZoneBattleView.cs`/相关枚举的实际命名一致，本规格先按语义占位。

### 2.2 邻接连线（`maproute_*`）

- 复用 art-bible §3.2/§3.3 既定的"三语气线条语言"——区域间的邻接关系是**结构性事实**（哪些区相邻、哪些区可调兵，是规则常量而非情报），因此连线固定走**事实语气**：实线、山水墨深灰、无箭头（邻接关系视为双向可调兵，不用箭头强调方向性；若 GDD 后续定义"调兵方向"需要箭头，另需 game-designer 明确后再补规格）。
- 不使用虚线（虚线保留给"推测情报"专用语义，连线是结构常量不应借用推测语气，避免和雾态推测信息产生混淆）。
- **命名**：`maproute_zonebattle_adjacency_16.png`（16px 宽的线段贴图/九宫格拉伸贴图，供 UI 层按实际连线长度拉伸渲染）。

## 3. 兵力队列图元 Sprite

### 3.1 造型选择：旗帜阵列（非兵卒剪影堆叠、非填充条）

**推荐依据**：

1. **缩放稳定性**：旗帜是简单几何轮廓（三角/燕尾旗面 + 直旗杆），在 art-bible §3.1 定义的缩略尺寸下仍保持清晰可辨的"数量感"；兵卒剪影在小尺寸下容易糊成一团色块，尤其多个剪影重叠时轮廓会互相吞噬（Gestalt 完形原则中的"闭合"在密集小剪影堆叠时容易失败）。
2. **离散计数直觉**：旗帜天然带有"计数"的文化联想（旌旗蔽日=兵多，孤旗=兵弱），比兵卒剪影更容易被"数出来"而非"目测面积大小"——与 GDD F1 定义的"离散图元计数"公式天然同构。
3. **与既有语言呼应**：art-bible §3.2 战略图层已把"旗帜/路线"作为主要图形词汇，沿用旗帜可与战略图层视觉语言呼应，降低玩家学习新符号的成本。
4. **反全知兼容性**：旗帜剪影在"雾态"下整体降低透明度后仍保持"这里有旗"的存在感、不暴露精确数量；兵卒剪影雾化后容易变得完全不可辨认。

### 3.2 资产架构（已按 GDD 更正——单一基础图元按状态着色，非"每档位一张图"）

**更正记录**：本文件 v1.0 曾设计"每个兵力档位一张不同图（`tier1`/`tier2`/…/`grand` 大纛）"，即每种数量对应不同的画面。经与 GDD F1 核对，GDD 的实际呈现机制是**同一枚基础图元按当前档位数在 UI 层重复堆叠 N 次**（图元本身只有"阵营 × 状态"的组合数量，不需要为每个档位单独出图）。本节按此更正：

- **每阵营仅需 1 枚基础旗帜图元**，按 3 种状态导出：`own`（我方）/ `enemy`（敌方·已侦察）/ `fogged`（敌方·未侦察，见 §6 反全知问号徽标一节）。4 阵营 × 3 状态 = 12 张基础图。
- **档位数量**（每图元≈该区容量 15%、封顶值等）由 GDD F1 公式定义，本文件不重复定义数值，只关心"档位数量确定后，这些图元在画面上怎么摆、封顶后怎么显示"：
  - 正常档位（未封顶）：基础图元并排堆叠 N 次，堆叠间距/是否重叠交由 UI 实装（不在本文件的美术资产范围内，图元本身已按"单枚小尺寸"设计以支持密集堆叠仍可读）。
  - **封顶溢出**：达到 GDD F1 的封顶档位数后，不再堆叠更多图元，而是在图元行尾追加一枚**数字徽标**（"+N"样式），不再使用旧版"大纛"特殊图形。徽标造型：小型圆角矩形纸质芯片，山水墨深底 + 宣纸白数字，与旗帜共享同一画面调性但明确是"UI 数字信息"而非"另一种旗"，避免玩家误读为"第二种旗帜含义"。

  **TODO（范围确认）**："+N" 徽标是否需要美术专属贴图，还是由 UI 层纯文字渲染（无需美术资产）——待与 ui-programmer 确认 UI Toolkit 文字排版能否直接满足观感需求；若可以，本条美术资产可整条撤销，改为 UI 层实现，不阻塞发布。

## 8. 反全知问号徽标 与 雾态统一规则（跨资产统一）

### 8.1 问号徽标（新增独立资产，非合并进旗帜本体）

**更正记录**：本文件 v1.0 曾把"问号"直接画进一枚专属的"问号旗"整体图元（`unit_banner_unknown_fog_32.png`），与"推测虚线旗"并列为两种雾态细分。经对照 GDD §R5，正确架构是**分层叠加**而非"整旗替换"：

- 未侦察状态下，兵力图元本体仍是 §3 定义的 `troopicon_[faction]_fogged_32.png`（同一阵营基础图元，按当前档位估计值堆叠、约 50% 不透明度），**不做成一枚独立的"问号造型旗帜"**。
- 问号是一枚**独立叠加徽标**，居中覆盖在图元堆叠区域上方（不逐枚重复，整组只叠加 1 枚），命名 `uiicon_tier_unknown_32.png`，造型为素色（山水墨深灰）圆形徽记内嵌一个手绘笔触问号，呼应印章质感但刻意不用朱红（朱红是"命令"语气专属色，问号属于"未知情报"不能借用该色，避免语义冲突）。
- 虚线描边（推测语气，参照 §3.3 蓝灰虚线）加在整组图元堆叠的外轮廓上，与问号徽标、半透明图元三者叠加，共同构成"未侦察"态的完整呈现，而非各自独立的三选一状态。

**本文件对 GDD R5 呈现口径的读解（如有出入以 game-designer 校正为准）**：未侦察态下，兵力图元本体**保留该阵营真实的旗面颜色**（因为玩家在发起该场战斗前已通过战略层知道对手是哪个阵营，"未侦察"隐藏的是**精确兵力数字与将领身份**，不是"敌方是哪个阵营"这条已知信息）；图元数量按 GDD F2 公式给出的区间估计中位值渲染，而非完全省略计数。若此读解与 GDD 原意不符（例如 GDD 实际希望未侦察态连阵营色也一并隐藏为中性色），**请 game-designer 明确指出**，本文件据此改一处（把 `fogged` 状态的旗面色改为统一中性灰蓝 H35 S8 L55），改动范围很小，不影响其余章节。

### 8.2 雾参数（复用既定值，不新创语言；FogIconOpacity 为本资产专属数值，不与 §3.2 战略图雾参数混淆）

- 兵力图元 `fogged` 态不透明度：约 50%（GDD 专属调参值，区别于 art-bible §3.2/§10.5 战略图雾的 60-70% opacity——两者服务于不同资产、不同尺度，不视为矛盾，仅在本节明确记录两处数值的来源分别是战场层 GDD 与战略图层 art-bible，避免未来维护者误以为是同一参数被写错）。
- 虚线描边：蓝灰 H215 S18 L55，对齐三语气体系"推测=虚线蓝灰"既定语法。
- 8px feather 软边 + 纸纹噪点：沿用 §3.2/§10.5 既有雾参数，用于图元边缘的柔化处理。

### 8.3 三语气体系在战场层的落地（规则整合，不引入新决策）

1. **地貌永远可见，情报永远可雾化**（§10.5 既定原则"地貌可见，情报不可见"沿用到战场层：背景/节点框/连线是地貌与结构，永不雾化；旗帜数值、将领身份、涌现标记的敌方来源是情报，按侦察状态雾化）。
2. **已侦察 = 事实语气**：实色 + 实线（`own`/`enemy` 两态、节点框、连线、在途/溃散叠加均属此列）。
3. **命令 = 命令语气**：实线 + 朱红 + 印章质感（姿态图标 §5 专属）。
4. **未侦察/区间估计 = 推测语气**：虚线 + 蓝灰 + 半透明（兵力图元 `fogged` 态 + 问号徽标）。
