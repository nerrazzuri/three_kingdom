## 完整游戏的弧线（game-concept）

> **城池太守开局** → 守城/治理/发育 → 君主任务/攒功绩名望 → 晋升掌权/扩张夺城 → **终极抉择**：
> - **忠臣线** → 逐阶到大都督 → **继承基业成君主**
> - **自立线** → 关系网三分支自立 → **成势力争霸**
> → 君主级国策 / 多势力争霸 / 架空史推进 → **统一乱世（终极结局）**

横跨：全图（多州多城）× 全演义时间线（讨董→官渡→赤壁→三分→北伐…）× 全势力（诸侯并立）。

## ⚠️ 设计状态分层（决定能不能直接实现）

| 范围 | 设计状态 | 含义 |
|---|---|---|
| 太守阶段 + 三层内核（守城/生涯/世界/战斗/情报/外交入口…） | ✅ 已有 GDD（001~016）、内核已实现 | 可直接**装配+深化** |
| 全图/全时间线/全势力 AI/完整外交/完整战场 | ⚠️ 有 GDD 但只切片深度 | 需**深化到全量**（多为扩展，非新设计） |
| **继承基业→君主玩法 / 自立→争霸 / 君主级国策 / 统一胜利条件 / 完整8阶权限** | ❌ **gdd-014 §Future Scope，无 GDD** | **必须先补设计（新 GDD + ADR）再实现** |
| 美术/音频/完整 UI/全局平衡/发布 | 部分（art-bible v1、UX 三屏）| 内容+打磨 |

**结论**：Phase A/B/C 主要是装配与深化（已设计）；**Phase D（终局/君主/争霸）须先走设计管线（GDD→架构→epic）**；Phase E 内容打磨。

## 阶段总览（A→E）

| 阶段 | 目标 | 完成后状态 | 设计前置 |
|---|---|---|---|
| **A 可玩骨架** | 一段太守循环端到端跑通（窄但完整） | **首个可测游戏**：开局→守城→治理→事件/任务→晋升/自立触发，Unity 可玩 | 无（已设计） |
| **B 完整世界铺开** | 从一区域铺到全图/全时间线/全势力 | 在真三国世界里玩太守，全图行军、历史事件网、多势力 AI | 无（深化已有 GDD） |
| **C 中后期纵深** | 掌权/扩张/完整战场/完整官阶 | 玩到掌权扩张：多城多军团、野战攻城多战场、8 阶梯队 | 部分（8阶权限差异需补细则） |
| **D 终局：君主/争霸/统一** | 终极抉择落地 + 君主/争霸 + 统一胜利 | **能通关**：继承成君主或自立争霸→统一乱世→结局 | ❌ **先补 GDD/ADR** |
| **E UI/美术/音频/平衡/发布** | 完整表现 + 全局平衡 + 打磨 | 完整成品可发布 | art-bible 扩展 |

**依赖主线**：A → B → C → D → E（E 贯穿）。A 是必经脊梁；D 须在其设计补完后才能实现。

## Phase A — 可玩骨架（垂直切片端到端，= epic-013~019）

> 必经第一步：先把一段循环装成能跑的，才谈得上铺开。完成 = 首个可测游戏。

- **A1 会话装配内核 CampaignSession**：统一会话持生涯+世界+战斗；日界按全局结算顺序（FIX-2）推进三层；战果写回生涯/世界（经 004，FIX-9 已证通）。**脊梁**。
- **A2 场景配置化 + 战役目录（含 CON-5）**：SliceScenario→不可变 SliceScenarioData + ScenarioCatalog 多场景 + 多城 CitySeed + 配置校验。
- **A3 回合间治理/推进循环**：治理跨日 + 君主任务 + 招揽 + 晋升申请 + campaign loop（推进世界/触发历史事件与任务→下一情境）。
- **A4 敌方 AI 便宜 80%（战术层）**：gdd-016 §MVP，替换脚本敌人为会决策的对手。
- **A5 Meta 三屏 UI**：生涯面板 / 战略态势 / 治理·战役选择。
- **A6 首个循环内容（一区域）**：1-2 城 + 数条历史事件 + 数任务 + 前 2-3 阶门槛 + 1-2 战役场景。
- **A7 端到端可玩验证 + 平衡**：Unity 运行时全接 + playtest + 端到端测试。**= 首个可测游戏。**

（各模块详细目标/范围/验收见本文件 git 历史 v1，或开工时落 epic。）

## Phase B — 完整世界铺开（横向：一区域 → 全三国）

- **B1 全图世界地图**：GDD_003 完整——多州多城拓扑、行军/路线/地形、全图寻路（现为切片最小图）。
- **B2 全演义历史事件网络**：GDD_015 完整时间线——讨董→官渡→赤壁→三分→北伐…事件网 + reachability 触发 + 分叉传播在全图（现仅机制，无全量事件数据）。
- **B3 全势力建模 + 势力级 AI 战略层**：全诸侯势力存续/疆域/关系（FactionRecord 全量）+ GDD_016 战略层（够得着范围）+ 抽象结算（够不着范围）驱动全图所有势力。
- **B4 完整外交**：多势力结盟/敌对/背叛/外交国策（GDD_012§8 扩展到全势力 + 长线外交）。
- **B5 完整情报/战争迷雾**：全图情报、信使延迟、势力级认知（GDD_007 全量）。

## Phase C — 中后期纵深（纵向：太守 → 掌权扩张）

- **C1 完整官阶梯队**：8 阶 + 各阶兵权/治理范围/战略参与权差异（GDD_014 完整；**8阶权限差异细则需补**）。
- **C2 掌权/扩张玩法**：多城治理、独领战区、多军团/战区调度、征兵体系。
- **C3 完整战斗与战场**：不止守城——野战/攻城/伏击/水战/夜袭…多战场 + 六杠杆全量招式库（GDD_010 完整 Future 部分）。
- **C4 君主任务库 + 随机事件**：丰富任务、剧情事件、关系延伸（联姻/家族/师承，gdd-014 Future）。

## Phase D — 终局：君主 / 争霸 / 统一 ❌ 先补设计

> **本阶段绝大部分无 GDD（gdd-014 §Future Scope）。每个模块须先走 `/design-system`→`/design-review`→`/architecture-decision`→`/create-epics`，再实现。**

- **D0 终局设计批次（GDD + ADR）**：为下列写 GDD_017+ —— 君主级玩法、势力争霸、统一胜利条件与结局体系、架空史长线推进。
- **D1 终极抉择落地**：继承基业 / 自立 两条终局路径真正可走（非"结局后玩法简化"）。
- **D2 君主级玩法**：成君主后国策、任命、势力经营、多军团争霸调度。
- **D3 自立争霸 + 架空史**：自立势力发展、与诸侯争霸、GDD_015 玩家势力圈脱稿的架空史推进。
- **D4 胜利条件与结局**：统一乱世 + 多结局（忠臣继业 / 自立称霸 / 流浪…）、通关流程。

## Phase E — UI / 美术 / 音频 / 平衡 / 发布（贯穿 + 收尾）

- **E1 完整 UI**：所有阶段的屏（沿 UX 规范），占位 UXML→正式。
- **E2 美术资产**：角色/场景/图标/UI（守原创红线①，art-bible 扩展）。
- **E3 音频**：音乐/音效（audio direction）。
- **E4 全局平衡 pass**：全游戏经济/难度/雪球/双线/争霸平衡（FIX-5 护栏全量落实+验证）。
- **E5 发布准备**：性能预算、存档迁移链、本地化、打磨、release checklist。

## 建议执行顺序

1. **先 Phase A 全做**（A1→A7）：拿到首个可测游戏 + 证明架构可铺开。**A1 是脊梁，先开。**
2. **Phase B 铺开**到完整世界（可与 A6/A7 内容部分重叠）。
3. **Phase C 纵深**到掌权扩张。
4. **Phase D 之前先开设计批次（D0）**：补君主/争霸/统一的 GDD+ADR，过审查，再实现 D1~D4。
5. **Phase E 贯穿**（UI/美术随各阶段产出补，平衡/发布收尾）。

每模块走标准管线：`/create-epics`/`/create-stories` → `/dev-story` → `/code-review` → `/story-done` → commit+push。每模块完成即有可测/可玩产物。
