# Epic: 多城与战区循环（Multi-City / Theater Loop / M12）

> **Layer**: Feature
> **Architecture Module**: M12（`full-game-loop-module-plan` §M12）
> **Governing ADR**: ADR-0008（城池控制权唯一权威 + 变更事件）· ADR-0009（会话装配）· ADR-0004（确定性）
> **Status**: Draft（需补设计锚点后转 Ready；风险已标注"复杂度指数膨胀，须先定权限/委任机制"）
> **设计来源**: 模块计划 §M12 + 2026-07-04 用户设计对话（占城 C 使玩家真会直辖多城）

## 背景与问题

选定占城方案 C（epic-029 story-004）后，玩家**真的会直辖多城**。若每城都要玩家点内政，
必成苦役、并滑向 4X 微操（违反 game-concept"不是三国志复制品/全知面板"）。M12 的答案是
**委任 + 权限层级**：玩家掌管范围随官阶变大，但次要城靠**委任下属**打理，玩家只碰关键决策。

## 设计裁定（继承 + 对齐）

1. **掌管范围随官阶**：官阶梯队（Rank 0–7）本就是掌管轴——单城→多城→一州数城→战区。升官解锁更大范围。
2. **委任打理**：直辖但不亲管的城 → 委任 `DelegatedGovernor`（AI 下属），玩家只做关键决策（征调/战区令）。
3. **玩家仍不全知**：多城态仍经情报投影呈现，不给全知势力面板（反全知）。
4. **仍受约束**：多城不等于无限资源——人/粮/令/情报四重约束照旧；委任 AI 不越权。

## 砍 scope 尺子自检（强制）

- 多城喂给什么？更大的补给/征募/兵力池喂战争；更高权限喂生涯（编制/战区令）。**委任抽象掉日常微操，只留"改变战争边界"的决策。喂不到的城市经营细节即砍。**
