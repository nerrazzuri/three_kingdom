# s23b 补充验证报告：出征战斗流程（HUD 走通）

时间：2026-07-07
范围：补充 s23 冒烟测试中 step4（战斗）被标记 BLOCKED 的部分。验证出征入口（隐藏在右侧「敌情探报」列，需下滚至「出征攻城」卡）能否从 HUD 完整走通：授权→组装→发起→进入 ZoneBattle→推进→结算→返回 HUD。
说明：本次仅在 Unity 编辑器 Play 模式内验证 UI 可达性与前端表现，未修改任何代码、未 git pull/push/commit。

## 步骤结果

### 步骤1 打开工程/Play模式进入HUD — PASS
主菜单→新游戏→选择「刘玄德·小沛」开局（就此起家按钮点击两次才生效，为已知历史遗留小问题，不阻塞）→进入HUD。

### 步骤2 定位「出征攻城」卡 — PASS
在HUD右侧敌情探报列，依次下滚过 派出侦察/派出袭扰/设伏诱敌 后，找到出征攻城卡。
卡片可见：请缨受命（君主授权）按钮、目标行下邳：未授权（须君主受命）、组装出征（选目标）按钮（此时禁用）、布势路线标签，正面强攻按钮部分可见。

### 步骤3 请缨受命（君主授权） — PASS
点击后：请缨受命按钮变灰禁用；目标行变为下邳：可攻（授权·敌控城）；组装出征按钮变为可用。
Console：1 info / 0 error。
截图：step3-authorized.png

### 步骤4 组装出征→选路线→加副将/增兵 — PASS
点击组装出征→选择路线正面强攻→点击加副将1-2次→点击增兵+100。
预览行显示：目标 下邳｜路线 正面强攻｜预计战力 795·士气 0.769943｜未侦察（战力从基础680随加副将/增兵提升至795，第二次点加副将无进一步变化，疑似封顶）。
发起出征、守城迎敌（区域防御战）按钮均可见。
截图：step4-plan.png

### 步骤5 发起出征→进入ZoneBattle — PASS
点击发起出征后，窗口标题变为...ZoneBattle - Windows...，确认场景切换成功，进入独立战斗场景。
界面显示：出征·虎牢关 — 战场区域（注：标题写虎牢关而非下邳，与目标名不一致，可能是路线/关隘命名差异，未深究，建议开发确认），第1/6回合，各区态势面板（遮蔽高地/侧翼隘口/★正面关城 我500|敌467/预备后方/敌粮道），排兵布阵面板。
Console：1 info / 0 error。
截图：step5-battle-enter.png

### 步骤6 守方具名守将验证 — 未通过/GAP（非崩溃）
推进一回合（亲自打·推进回合(敌军亦动)）：第1/6→第2/6，★正面关城 我500→351｜敌467→420，Console保持0 error。
反复查看左侧各区态势面板全部5个分区（遮蔽高地/侧翼隘口/★正面关城/预备后方/敌粮道），均只显示我N|敌M数字兵力，未见任何具名武将文本。
点击★正面关城分区行文字、点击敌方数字，均无展开/详情弹出。
右侧涌现兵法板块在观察期间始终为空，未见战报或武将名录内容。
结论：本次未能在ZoneBattle战斗UI中肉眼确认守方（Defender）具名守将——界面仅展示数值化的分区兵力对比，没有找到显示具体守将姓名的入口。这不代表后端逻辑未执行（单测覆盖之外未做代码级验证），但从玩家可见角度，具名守将尚未在前端呈现，建议开发确认是否有意省略或UI未接上。
截图：step6-defenders.png（当前最佳证据画面，为分区数值面板）

点击结算战果并返回后，窗口标题变回...Hud - Windows...，自动返回HUD，界面正常显示（相位/性情/争霸信息行、日期行、下方两栏面板），无残留战斗UI或异常遮挡。

## 总判定
出征战斗链路（请缨受命→组装出征→发起出征→ZoneBattle→推进回合→结算返回HUD）在Unity编辑器Play模式下端到端可以走通，全程Console 0 error、无崩溃、无红色报错。
唯一未达成的验证项是守方是否显示具名守将——本轮UI只呈现数值化兵力对比，未发现具名武将展示入口，标记为GAP（非阻断性，非崩溃），建议后续与开发确认该UI表现是否符合当前批次（GDD批B/C）预期。
其余环节（步骤1/2/3/4/5/7/8）判定：PASS。
