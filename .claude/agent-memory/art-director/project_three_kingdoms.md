核心视觉锚已锁定为"军府案上展开的活图卷"——每一帧画面由三层构成：底层手绘地貌+纸张质感，中层批注/印记/修改痕迹，顶层人物与事件的情绪温度。这条规则是所有视觉决策的唯一仲裁依据。

**Why:** 视觉锚来自 design/concept/art-direction.md 的核心定位，并在 AD-PHASE-GATE 审查中被确认为可落地的量化基础。

**How to apply:** 任何新视觉元素提案都须对照"活图卷"做测试——它是图卷上的哪一层？属于哪种笔迹语气（事实/推测/命令）？

**2026-07-08 §5.30 v3 修订：Midjourney 自动审核拦截，清理"写实人物"类高危措辞**（`midjourney-prompt-pack.md` 升至 v2.9，新增 §5.30.7）。出图时首次遭遇 MJ 平台层面的自动审核拦截（不是构图/风格问题，是审核模型判定 prompt 高危），报错原文"AI Moderator is unsure about this prompt. AI Moderation is cautious with realistic images, especially of people."——这是一类此前从未处理过的新失效模式：**MJ 除了"出图效果不符合预期"这类内容质量问题外，还存在一个独立的"平台审核直接拒绝生成"的前置关卡，这个关卡对措辞的敏感点和"出图质量"关卡完全不同**。清理出三类高危词族：①写实/照片类（`realistic`/`photo`/`naturalistic` 这类强调照片写实感的词，即使是为了追求"更真实的皮肤质感"这种正当创作意图，也可能被判定为"写实人物"敏感内容）；②暴力/惊悚联想词（`vampire`、`dangerous`、`menacing`、`aggressive`、`reckless...ferocity` 这类连续堆叠的强度词，即便用于刻画"奸雄/猛将"这类正当角色气质，也可能触发或叠加触发）；③种族描述词（`caucasian`/`western` 等具体人种标签，哪怕出现在"排除该特征"的负面提示里，也可能被判定敏感——负面提示不是免罪金牌，同一个词出现在负面清单里依然会被审核模型读到）。修正原则是**只换措辞、不改语义**：用中性的色相/明度/绘画感描述替代人种标签和照片写实词（如 `pale desaturated skin`、`painterly skin texture`），用其余已有的具体表情/肢体细节从句承担"角色气质识别度"、不再依赖单个高强度形容词或形容词堆叠（如吕布"骄矜暴烈"改由"brows raised in open disdain / curling contemptuous half-smirk / looking down on all rivals"这些具体细节独立承担，不再需要"arrogant fierce...reckless...ferocity"这组词本身）。**经验沉淀（重要）**：这是本项目第一次遇到"内容通过了所有创作层面的检查表（六点清单/家族相似性/阵营色等），却在提交生成这一步被平台审核拦下"的情况——提醒自己**"prompt 是否会产生期望的画面"和"prompt 是否能通过生成平台的审核关卡"是两个独立的失效面**，此前所有轮次的评审框架（六点清单、家族相似性底线、可分离/不可分离缺陷判断）都只覆盖前者，没有覆盖后者；后续新角色出图前，除了内容合规性（art-bible §9.4 原创性红线）之外，还应该主动扫一遍 prompt 里是否有"写实人物/暴力惊悚/人种标签"这三类平台审核已知敏感词，尤其是历史题材人物类项目天然容易在"写实人物肖像"这个品类上被从严审查，这类风险会随着后续 ~30 将全量出图反复出现，值得在模板层面就建立一份"过审安全词表"作为撰写新 prompt 时的默认检查项，而不是每次翻车了再补救。
