## 判断依据（裁定时用的尺子）

裁定不是"三国志有就抄"，而是用以下尺子衡量。这些是项目已锁定的约束（CLAUDE.md / ADR / GDD）：

1. **核心循环**：太守/武将生涯、有限情报、战前谋划、战斗后果、历史分支。
2. **强制设计锁**（不可违反）：
   - 兵法是条件组合，不是无条件技能按钮。
   - 军师建议而**不自动**排兵布阵 / 不替玩家做最优决策。
   - 战斗是核心特色之一，但**不能压倒人物、关系、城市、外交和政治**。
   - 失败必须产生**可继续**状态（非删档）。
   - 反全知（AI/UI 不读世界真值）、确定性可复现、所有平衡值数据驱动。
3. **红线**：零复制现有三国题材游戏/作品资产（机制可借鉴，**具体表达必须自创**）。
4. **里程碑容量**：不把大项一次性判 Must；拆成最小可交付。

## ⚠️ 关键发现：参考文档的 M13/M14 与 module-plan 不一致

参考文档（§使用方式）对里程碑的定义，与权威 `production/full-game-loop-module-plan-2026-06-28.md §6` 的后半程**不是一回事**：

| 模块 | 参考文档定义 | module-plan §6 定义 | 一致性 |
|------|-----|-----|-----|
| M11 | 外交 | 外交战略约束 | ✅ |
| M12 | 多城/军团委任 | 多城战区 | ✅ |
| **M13** | **科技/政策/势力成长** | **君主与争霸（Sovereign/Hegemony）** | ❌ 冲突 |
| **M14** | **亲族/婚姻/关系深层事件** | **统一与终局（Victory/Ending）** | ❌ 冲突 |
| M15 | 教程/难度/首小时 | 表现/UX/反馈 | 🟡 部分 |
| M16 | 内容/宝物/剧本/编辑器 | 内容/平衡/发布 | ✅ 大致 |

**含义**：参考文档实际上**悄悄改写了后半程路线图**——把 module-plan 里硬阻塞的"君主争霸 / 统一终局"（缺 GDD_017/018，CLAUDE.md 红线"必须先补 GDD+ADR"）替换成了"科技政策 / 亲族婚姻"。

这等价于在问一个未决的根本问题：**我们的终盘是什么？**（见文末"方向性决策"）。团队需要先对齐 M13–M16 到底指什么，否则后半程裁定无基准。

## 桶三：和我们方向不一致 / 需谨慎（评审重点）

> 这些若不加约束直接做，会**偏离核心或违反设计锁**。建议团队逐项决断。

| 功能 | 不一致点 | 建议处理 |
|------|---------|---------|
| **科技/政策/势力成长** | ⚠️ 是**"君主全局经营"视角**；我们核心是**"太守/武将生涯"**。三国志政策槽是君主玩的，我们玩家多数时间是太守 | **暂缓 + 需方向决策**：要么只在玩家自立成势力后解锁，要么 Reject。**不要默认塞进路线图** |
| **婚姻/义兄弟/亲族（全量）** | ⚠️ 易膨胀成**生活模拟**，偏离"判断与计谋"核心 | **只做关系标签**（义兄弟/仇敌影响关系权重，复用 `RelationshipState`），不做子女/婚姻事件网。Later |
| **复杂商路/贸易经济** | ⚠️ 违反设计锁——城市治理**必须喂给战争**（M03 的"筛选尺子"）；纯经济不反哺战争即偏离 | **Reject 纯经济**；只保留"反哺战争/外交"的补给路线 |
| **全自动委任黑箱** | ⚠️ 违反设计锁"军师建议而不自动执行 / 玩家判断" | 委任做成**可解释的授权**，不是甩手自动（已并入桶一 M12 约束） |
| **大而全编辑器/DLC** | 抢核心产能；非核心闭环 | Later/backlog，除非定位明确要 UGC |
| **君主争霸 / 统一终局**（module-plan M13/M14） | ⚠️ **可能整体偏离**——统一天下是 4X 的胜利条件；我们核心是"一个人的三国人生"（太守视角、失败可继续、判断计谋） | **需方向决策**（见下） |

## 两个方向性决策（必须由团队定，AI 不能替定）

这两个问题决定后半程（M13–M16）怎么走，也决定要不要写 GDD_017/018：

### 决策一：我们的"终盘"是什么？

- **选项 A — 统一天下（4X 终局）**：玩家最终目标是统一，需 GDD_017（君主争霸）+ GDD_018（统一终局）+ 配套 ADR。工作量大，且与"太守视角/失败可继续/一个人的人生"有张力。
- **选项 B — 一段三国人生**：终盘是"位极人臣 / 自立一方 / 功败垂成"皆有意义的人生结局，而非统一胜利。更贴合现有设计锁（失败可继续、生涯循环），后半程改为"生涯结局 + 关系事件 + 历史见证"。
- **影响**：选 A → 走 module-plan 原 M13/M14；选 B → 走参考文档重解读的 M13（势力成长）/M14（关系事件），且"统一终局"可能 Reject 或大幅简化。

### 决策二：科技/政策做不做、何时做？

- 它偏"君主全局经营"，与"太守视角"张力明显。
- 选项：(a) Reject；(b) 仅玩家自立成势力后解锁的轻量政策槽；(c) 重新定义为"太守可影响的治理方针"（贴合 M03 城市治理，而非君主科技树）。

## 近期执行建议（若方向决策暂缓，也可先做这些）

**两个最高优先，且都不是"加功能"：**

1. **M11 外交**——核心支柱4，底座已有，闭环必需。
2. **M15 首小时教程 / 可进入性**——让 11 个已贯通的 Domain 循环**真正可玩**。继续堆 Domain 模块边际收益递减，"让人能玩到"是断崖式收益。

**建议顺序**：M11 外交 → M12 多城委任（非黑箱）→ **转 M15 让它能玩** → 再回头按"决策一/二"的结论决定 M13/M14。
强契合"计谋核心"的**策反/离间**、**战法内容包**、**招揽**可穿插进 M11/M06 增量做，不必单列大里程碑。

**暂不动**：科技政策（待决策二）、君主争霸/统一终局（待决策一 + 缺 GDD_017/018）、婚姻亲族全量、编辑器、复杂商路。
