## 评审修正边界（v2，团队反馈——权威约束，优先于下文表格）

下列边界经团队评审确立，**Claude Code 后续据此判断时必须遵守**；与下文表格如有出入，以本节为准：

```text
M11/M12 = Must, but design-first, not implementation-ready.
M13/M14 must follow current module-plan unless a formal roadmap revision is made.
M09 additions are design debt / incremental GDD, not a restart of M09.
Historical events: minimum event pipeline = Should; full Romance event network = Later.
Access rights / logistics corridors belong to M11 and should not be postponed entirely.
Treasure: minimal reward-loyalty loop = Should; full item economy = Later.
M15 = UX/readability/onboarding; full difficulty matrix = M16.
```

逐条说明：
1. **M11/M12 = Must，但 design-first**：列为 Must 指"应进入交付范围"，**不等于 implementation-ready**。两者均须**先补 GDD/ADR**（M11：外交 GDD 或 GDD_012 扩展 + ADR；M12：多城战区设计 + ADR），再进入实现。
2. **M13/M14 不得由本 verdict 重新定义**：君主争霸/统一终局若要改方向，**必须正式更新 `production/full-game-loop-module-plan-2026-06-28.md`**，不能只在本 gap verdict 里改口径。下文"关键发现"只是**提出问题**，不构成路线图变更。
3. **M09 不重开**：忠诚、任官、背叛等作为 **M09 的增量设计债 / 后续新 GDD**，不推翻已完成的 epic-022（M09）。
4. **历史事件拆开**：最小事件流程（触发器 + 3–5 代表事件 + 生产流程）= Should；全演义大规模事件网 = Later。
5. **通行权/后勤通道不全放 Later**：它们是 M11 外交与战略约束的重要内容，至少 Should，归 M11。
6. **宝物拆开**：最小"赏赐→忠诚"闭环 = Should；完整宝物收集/交易/套装/特产 = Later。
7. **M15 = UX/可读性/可进入性（onboarding）**，不止"首小时教程"；完整难度参数矩阵更偏 M16。

## 判断依据（裁定时用的尺子）

裁定不是"三国志有就抄"，而是用以下尺子衡量。这些是项目已锁定的约束（CLAUDE.md / ADR / GDD）：

1. **核心循环**：太守/武将生涯、有限情报、战前谋划、战斗后果、历史分支。
2. **强制设计锁**（不可违反）：
   - 兵法是条件组合，不是无条件技能按钮。
   - 军师建议而**不自动**排兵布阵 / 不替玩家做最优决策。
   - 战斗是核心特色之一，但**不能压倒人物、关系、城市、外交和政治**。
   - 失败必须产生**可继续**状态（非删档）。
   - 反全知（AI/UI 不读世界真值）、确定性可复现、所有平衡值数据驱动。
3. **红线**：零复制现有三国题材游戏/作品资产（机制可借鉴，**具体表达必须自创**）。
4. **里程碑容量**：不把大项一次性判 Must；拆成最小可交付。

## ⚠️ 关键发现：参考文档的 M13/M14 与 module-plan 不一致

参考文档（§使用方式）对里程碑的定义，与权威 `production/full-game-loop-module-plan-2026-06-28.md §6` 的后半程**不是一回事**：

| 模块 | 参考文档定义 | module-plan §6 定义 | 一致性 |
|------|-----|-----|-----|
| M11 | 外交 | 外交战略约束 | ✅ |
| M12 | 多城/军团委任 | 多城战区 | ✅ |
| **M13** | **科技/政策/势力成长** | **君主与争霸（Sovereign/Hegemony）** | ❌ 冲突 |
| **M14** | **亲族/婚姻/关系深层事件** | **统一与终局（Victory/Ending）** | ❌ 冲突 |

## 桶三：和我们方向不一致 / 需谨慎（评审重点）

> 这些若不加约束直接做，会**偏离核心或违反设计锁**。建议团队逐项决断。

| 功能 | 不一致点 | 建议处理 |
|------|---------|---------|
| **科技/政策/势力成长** | ⚠️ "君主全局经营"视角，与"太守/武将生涯"核心张力 | ✅ **已决（2026-06-30）**：君主全局科技树 **Reject**；只做"治理方针"（重农/重商/重武/安民）**并入 M03/M12**，不设独立模块 |
| **婚姻/义兄弟/亲族（全量）** | ⚠️ 易膨胀成**生活模拟**，偏离"判断与计谋"核心 | **只做关系标签**（义兄弟/仇敌影响关系权重，复用 `RelationshipState`），不做子女/婚姻事件网。Later |
| **复杂商路/贸易经济** | ⚠️ 违反设计锁——城市治理**必须喂给战争**（M03 的"筛选尺子"）；纯经济不反哺战争即偏离 | **Reject 纯经济**；只保留"反哺战争/外交"的补给路线 |
| **全自动委任黑箱** | ⚠️ 违反设计锁"军师建议而不自动执行 / 玩家判断" | 委任做成**可解释的授权**，不是甩手自动（已并入桶一 M12 约束） |
| **大而全编辑器/DLC** | 抢核心产能；非核心闭环 | Later/backlog，除非定位明确要 UGC |
| **君主争霸 / 统一终局**（module-plan M13/M14） | ⚠️ 纯 4X 框架与"一个人的三国人生"核心偏离 | ✅ **已决（2026-06-30）选 C**：M13/M14 已正式修订为"生涯结局与传承 / 人生落幕与史册"；争霸/战略AI/问鼎保留为人生走向与最难结局，只去掉 4X 框架 |

## 两个方向性决策（2026-06-30 已定）

> 见 `docs/reviews/endgame-direction-decision-2026-06-30.md`；已落入 `full-game-loop-module-plan` 正式修订（M13/M14 段 + §6 切分表）。

### 决策一：终盘是什么？ → **选 C（一段三国人生）**

骨架 = 一段三国人生（主角是**人**：会死/可传承/多结局/失败不删档）；**争霸、战略 AI、问鼎天下皆保留**，作为人生后期走向与最难结局；**只去掉 4X 框架**（主角是不死势力/统一唯一目的/输了删档）。M13/M14 已正式修订为 **生涯结局与传承 / 人生落幕与史册**。

### 决策二：科技/政策做不做？ → **不做君主全局科技树（Reject）；只做"治理方针"**

采纳选项 (c)：重新定义为"太守/势力可影响的**治理方针**"（重农/重商/重武/安民等数据驱动方向选择），**并入 M03/M12 城市治理**，作为"喂给战争的筛选尺子"延伸；**不设独立科技模块**，君主全局科技树 **Reject**。

## 近期执行建议（若方向决策暂缓，也可先做这些）

**两个最高优先，且都不是"加功能"：**

1. **M11 外交**——核心支柱4，底座已有，闭环必需。
2. **M15 UX / 可读性 / 可进入性（onboarding）**——让 11 个已贯通的 Domain 循环**真正可玩**。继续堆 Domain 模块边际收益递减，"让人能玩到"是断崖式收益。（完整难度参数矩阵归 M16）

**建议顺序**：M11 外交 → M12 多城委任（非黑箱）→ **转 M15 让它能玩** → 再回头按"决策一/二"的结论决定 M13/M14。
强契合"计谋核心"的**策反/离间**、**战法内容包**、**招揽**可穿插进 M11/M06 增量做，不必单列大里程碑。

**暂不动**：M13/M14 终盘（方向已定 C，但属后期内容，待游戏能玩后做）、君主全局科技树（已 Reject）、婚姻亲族全量、编辑器、复杂商路。
