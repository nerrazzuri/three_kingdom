# ADR-0015 — 纪元与世界快照模型（Era Calendar & World Snapshot）

> **Status**: Proposed（2026-07-05；待用户/审查裁定 Accepted。落地 GDD_026 空降者·纪元开局与一生。）
> **相关**: GDD_026（纪元开局与一生）· GDD_001 游戏时间 · GDD_003 世界地图 · GDD_014 生涯/自立 · GDD_015 历史事件 · GDD_025 武将标签（EraStage）· #1 PlayableStart · ADR-0004（确定性）· ADR-0005（存档版本/迁移）· ADR-0009（CampaignSession 装配）· ADR-0003（数据驱动配置）

## Context（背景）

GDD_026 把开局升级为"空降者任选**年份 × 城池**、活出一生"。这要求三件当前架构没有的东西：
1. **公元纪年层**——世界须知道"现在是公元几年"，历史事件与空降者寿命按年计；而现有 `WorldTime` 只有抽象日-段/AbsoluteIndex，无日历。
2. **按年的世界快照**——每个锚点年一张 `{势力/城归属 + 君主 + 各城在职武将}`；现有 `World` 表（#1 PlayableStart）是**单一隐含年**的骨架。
3. **武将生卒/布防 + 空降者寿命**——"该年该城武将归你"要求武将有生卒年与"任职城×年"，且空降者有确定性寿命。

核心问题：如何在**不破坏现有确定性/存档/装配基座**、不使回合数爆炸、不把连续 96 年都做成内容的前提下，引入日历、快照与寿命，并与既有 `WorldTime`、`PlayableStart`、`GeneralDossiers`、历史事件系统自洽。

## Decision（决策）

**D1 公元纪年层叠加于日-段之上，不重写 WorldTime。**
新增轻量纪元态 `{ StartYear, YearsElapsed }`，派生 `CurrentYear = StartYear + YearsElapsed`。既有 `WorldTime`（日-段/AbsoluteIndex）**保持不变**，继续驱动战役/治理的细粒度；`YearsElapsed` 只在**年度边界**（每 `SegmentsPerYear` 段）递增。历史事件（GDD_015）与寿命门改以 `CurrentYear` 判定。年推进为**确定性纯函数**（ADR-0004）——不引入新随机源。

**D2 锚点年离散快照，非连续年滑块。**
世界快照只在**精选锚点年**提供（命名剧本；首发 190）。快照 = 不可变**配置**（`anchorYear → {FactionRecord[], CityOwnership[], CityGarrison[城→武将id]}`），构建期校验、指纹保护（ADR-0003），**不入存档**。现有 `World` 表泛化为"按年"的快照集：默认 190 快照即当前 17 席/37 城骨架的按年化，保证既有默认剧本**字节级不变**（#1 的护栏延续）。理由：连续 96 年 = 96 套快照，内容与确定性代价不可行；离散锚点给历史意义又可分期补。

**D3 PlayableStart 由"手写"泛化为"快照 × 选城"程序化生成。**
`PlayableStart` 不再仅是手写常量，而可由 `(anchorYear, cityId)` 程序化派生：玩家势力 = 快照中该城 Owner、治所 = 该城、部曲 = 该年该城在职武将（D4）。可选城集排除各势力**君主治所**（玩家恒为太守，接 GDD_014）。现有 3 个手写开局作为等价特例保留（回归护栏）。

**D4 武将生卒/布防扩 GeneralDossiers，为唯一权威。**
`GeneralDossier` 增 `BirthYear/DeathYear` 与 `Placement(anchorYear → CityId)`。在世/在职判定为纯函数 `Available(g,Y) ⟺ BirthYear+SERVICE_MIN_AGE ≤ Y ≤ DeathYear`；`EraStage`（GDD_025）由 `Y − BirthYear` 派生。布防与快照城归属**构建期自洽校验**（某将布防城须存在且属某势力）。`GeneralDossiers` 仍是武将数据唯一权威（#2），无第二真值源。

**D5 空降者寿命种子化确定性派生。**
`Lifespan = LIFESPAN_BASE + round((2u−1)·LIFESPAN_SPREAD)`，`u = DeterministicRandom(sessionSeed ⊕ SALT_LIFE).NextUnit()`——注入式确定性流（复用 ADR-0004/0006 随机基座），**非掷骰**、随存档可复现、同人设/会话恒定。`DeathYear = StartYear + Lifespan`。寿终/退隐 → 一世结算 → 传承（`HeirChain`，接 GDD_014）或重开。寿终**非 game-over**，是可续的自然落幕。

**D6 纪元/一生入存档（版本化 DTO），快照/生卒/布防为配置。**
存档新增权威字段 `StartYear/YearsElapsed/EntryAge/Lifespan/Retired/HeirChain` → 显式版本化 DTO + 逆序迁移链（ADR-0005）。快照/生卒/布防属**配置**（构建期、指纹校验，ADR-0003），不入存档。round-trip：恢复后当前年/寿数/在职武将集恒等（ADR-0004）。

**D7 反全知延伸至选城与己身。**
选城时只投影名声/已知信息——武将隐藏量级档（战阵档/谋略档）与隐秘心（忠诚/野心）**不因归你管而暴露为数字**（接 GDD_007/025）。空降者寿数对玩家只给**定性档**（春秋鼎盛/年事渐高/风烛残年），不给精确倒计时。

## Consequences（后果）

**正面**
- 开局从"选诸侯"跃为"选时空活一生"，兑现 game-concept 的人生模拟支柱；#1/#2/#3 成为其自然基座（零重写）。
- 日历层轻量叠加，既有 `WorldTime`/战斗/治理/存档基座不动；确定性、反全知、存档三条红线全保。
- 离散锚点 + 分期布防让内容**可增量**：先 190，后逐年补，架构一次到位。
- 武将生卒/布防让"名将随时代登场/老去"成真，深化 GDD_025 时代分段。

**负面 / 代价**
- 新增内容负担：武将**生卒年 + 历年布防表**（首发 48 将 190 布防，扩谱时逐年补）——这是主要投入。
- 历史事件需从抽象日-段窗**改键为公元年窗**（GDD_015 既有事件迁移一次）。
- 存档格式升版（+纪元/一生字段），需迁移链一条。
- `SegmentsPerYear`（年↔日-段映射）需与 GDD_001/004 联调定值（暂列 Open）。

**中性 / 约束**
- 玩家恒为**太守而非君主**（君主治所不可选）——与 GDD_014 生涯弧一致，非本 ADR 新增自由度。
- 快照为不可变配置，新增锚点年 = 加一张快照数据 + 该年布防，运行期/引擎零改动。

## Alternatives Considered（备选）

- **连续任意公元年滑块**：弃。96 套逐年快照内容爆炸，且多数年份枯燥、确定性/校验成本高。
- **把年份塞进 WorldTime 抽象序**（不引真实公元）：弃。失去历史意义与生卒/事件的真实锚，"活在三国"的沉浸打折。
- **武将布防另立数据源**（独立于 GeneralDossiers）：弃。违反单一权威，易与档案/快照失自洽；扩 GeneralDossier 字段更收敛。
- **空降者无寿命/固定 280 终点**：弃（用户裁定）。寿命让"选年"真正有取舍、兑现"完成一生"。

## Guardrails（护栏 · 供 story 引用）

- 默认 190 快照须使既有默认剧本**字节级不变**（延续 #1 护栏；`WorldSkeletonTests` 等为证）。
- 年推进/寿命纯函数、种子化、无新随机源（ADR-0004）。
- 快照城归属与武将布防**构建期自洽校验**，非法引用加载期拒（ADR-0003）。
- 选城恒排除君主治所；生成的 PlayableStart 对现有 3 手写开局等价（回归）。
- 存档新增字段走版本化 DTO + 迁移链，旧档可迁移或稳定拒（ADR-0005）。
- 反全知不破：选城/己身寿数皆不暴露隐藏量级档与精确数（GDD_007/025）。
