# ADR-0017 — 武将运行时人生态（GeneralState）

| 字段 | 值 |
|------|----|
| **状态** | Accepted（2026-07-07） |
| **决策者** | 用户 + Claude |
| **落地** | GDD_027 #1 / GDD_005 |

## 背景

武将系统此前"档案层做得多、活人层做得少"：`GeneralDossier`（静态档案，"本质如何"）+ 派生归属，
但缺一个**长期持久化的运行时态**——武将不会因这一局的经历（被背叛/救命/负伤/被俘/受赏）而改变。
GDD_005 要求人物有当前主君、忠诚变化、健康、被俘、记忆，并向关系/军议/战斗/AI 输出人物投影。

## 决策

### D1 静态档案与运行时态分离
`GeneralDossier`（不变，答"这个人本质是什么样"）与 `GeneralState`（答"这一局里经历了什么"）分离。
`GeneralState`（Domain/Characters）持：当前主君 `Faction`、驻城 `Location`、忠诚 `Loyalty(0..100)`、
健康 `Health(康健/负伤/重创)`、疲劳 `Fatigue`、被俘 `CaptiveOf`、记忆 `Memories`。**不可变（With\* 返新）**、确定性、无随机源。

### D2 记忆是"活人"核心
`MemoryEvent`（被背叛/受赏/救命/羞辱/宽宥/拔擢 + 施与者 + 年 + 权重）经 `GeneralLifeService.Remember`
**联动忠诚**（救命+15/背叛−20…×权重）并长期留痕。叛离风险 `RiskOf` 由忠诚 − 被背叛记忆派生（反全知定性档，无数字）。

### D3 台账为唯一权威运行时态，惰性铸入
`GeneralLedger`（可变容器）按需登记 per-将态；未登记者由 `GeneralLifeSeeding.Seed`（气质忠义倾向→初始忠诚 + 归属→主君/驻城）惰性铸初态。

### D4 持久化（可推演不足以覆盖，须显式入档）
忠诚漂移/伤病/记忆/被俘为**权威可变态**，无法派生 → 须显式纳入版本化存档（ADR-0005 DTO + 迁移链）。
`GeneralState`/`GeneralLedger` 字段皆值/枚举/id，为持久化设计（`Entries` 为 DTO 读写入口）。
**本 ADR 落地领域基座 + 服务 + console 演示 + 单测；存档 DTO/迁移 + 消费方全接（关系/AI/战斗/军议）为后续实施步。**

### D5 确定性
一切演化纯函数、无随机源（ADR-0004）；招降/伤病判定若需随机，注入种子流。

## 后果

**正面**：500 武将从"可查询档案"变为"会在世界里持续变化的人"；被背叛/救命长期改写向背，
支撑"人物·关系·名声塑造战争"的卖点；纯函数可测；不锁场景。
**代价**：新增权威可变态 → 增存档面（须迁移，逆序链再演练）；消费方全接是渐进工程（本 ADR 只落基座 + 一处 console 消费）；
初始忠诚/记忆权重为可调平衡值，待手感校准。

## ADR 依赖
ADR-0004（确定性）· ADR-0005（存档版本/迁移）· ADR-0016（武将归属层——归属为 GeneralState 主君/驻城的派生初值来源）。
