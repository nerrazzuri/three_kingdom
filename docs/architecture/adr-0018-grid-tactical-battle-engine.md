# ADR-0018 — 确定性格子战术战斗引擎（Deterministic Grid Tactical Battle Engine）

> **Status**: Accepted（2026-07-20；用户原型验证手感通过后裁定，取代命名区域战斗）
> **Supersedes**: ADR-0012（确定性区域战斗引擎——命名区域图模型）
> **相关**: GDD-028（格子战术战斗，本决策落地）· GDD-021（区域战斗，被取代）· GDD-010（兵法沙盒·涌现条件）· GDD-012（补给）· GDD-011（士气疲劳）· GDD-001（时间/DaySegment）· ADR-0004（确定性）· ADR-0006/0013（敌方AI）· ADR-0009（会话装配）· ADR-0005（存档）

## Context（背景）

ADR-0012 定"战斗=命名区域图（非坐标）+ 回合纯函数状态机 + 条件涌现"，其 D2 明确**否决坐标网格**，理由是"退化为兵种克制微操、滑向 SLG"。落地后（GDD-021 + 换皮 M1-M3 呈现），用户实玩判定：命名区域"更像棋盘、参与感薄"——胜负被前置到战前准备，开打后回合选择摆动小，缺**空间/当下参与感**（没有"半路遭遇被迫决断""绕后断粮""诱敌入隘伏击"这类地形驱动的临场博弈）。

用户提出并经**丢弃式原型（prototypes/grid-battle）验证手感通过**的战斗模型：**真地形格子地图 + 兵种定速行军（设目的地→按时间段推进→按速走）+ 半路遭遇临机抉择 + 时间段推进=相邻格交战 + 补给硬约束（充足回血/不足叛逃+强攻数倍伤亡）+ 地形赋能策略（隘口伏击、粮仓火攻断粮）**。

核心架构问题：如何在**引入坐标网格 + 连续移动 + 时间流**的同时，仍守住项目铁律与差异化——① 确定性（ADR-0004）；② 不退化为"兵种克制三角微操"（ADR-0012 当年的顾虑）；③ 保留"兵法=地形/时机涌现，非按钮"的差异化（GDD-010）；④ 反全知（GDD-007）；⑤ 复用而非重造已有的补给/士气/AI/存档设计。

## Decision（决策）

**D1 战斗为确定性时间段状态机，纯 Domain。**
`GridBattleState`（网格 + 各单位位姿 + 世界钟 + 双方补给 + AI 记忆/种子）是权威态。一个时间段推进 = 确定性纯函数：`Advance(state, commands, config, rng) → state'`。无 `MonoBehaviour`/`UnityEngine`、**无 float 于权威路径**（坐标为整数格、乘子用整数百分比/定点，ADR-0004）、无隐式随机。

**D2 战场 = 整数格子网格 + 地形（取代 ADR-0012 D2 命名区域）。**
`BattleGrid` = 宽×高的格子场 + 每格 `TerrainKind`（平地/山[不可通行]/隘口/林地/粮仓/…）+ 双方粮仓坐标。**本决策显式解除 ADR-0012 D2"无坐标"负向不变量**——坐标网格由用户裁定采纳（原型验证）。地图数据驱动、构建期不可变配置（ADR-0003），支持多张地图（不同地形→不同布局→不同策略）。**防退化护栏见 D8。**

**D3 兵种定速行军 = 设目的地 + 按段推进（复用命令契约）。**
每单位有 `TroopType`（骑/步/弓…）→ 数据驱动的**速度**（格/段）与**射程**。玩家下"设目的地"命令（前置校验→稳定错误码→无部分写入）；每次时间段推进，单位按速度沿**确定性寻路**（整数 BFS，绕不可通行/占用）向目的地移动。骑兵快（少段到）、步/弓慢（多段到）。移动是引擎推进的结果，非玩家逐格操控。

**D4 时间用世界钟驱动（复用 GDD-001 / DaySegment）。**
战斗时钟 = `WorldTime`（`WorldDay` + `DaySegment`：黎明/白昼/黄昏/夜）。**一次"进行" = 推进一个 DaySegment**。`夜` 段赋能夜袭（近战突袭加成）、`白昼/黄昏` 赋能火攻（可视纵火）——时机成为地形策略的一维（复用 GDD-010 时机门）。

**D5 交战 = 每段相邻格结算（确定性优先序）。**
每段推进末，**相邻（含斜角）的敌我单位互相造伤**；弓兵按射程可隔格放箭（不吃近战反击）。伤害整数结算，多单位按 `(side, unitId)` 规范序遍历、同时计算后统一写回（可复现）。攻守统一：一套引擎两向。

**D6 地形/时机涌现条件（保留并空间化 GDD-010，取代按钮）。**
兵法条件按**格子位置 + 时机**涌现，非按钮：
- **伏击**（复用 `TacticTag.Ambush`）：单位藏于**林地**、且相邻**隘口**有敌进入 → 伏击加成（大幅首击）。
- **火攻**（复用 `TacticTag.FireAttack`）：单位抵敌**粮仓**相邻 + 白昼/黄昏 → 焚仓。
- **夜袭**（复用 `NightRaid`）：夜段近战加成。
门不齐不成型、不打标签（延续 TR 负向不变量）。**无同名"计策按钮"**。

**D7 补给硬约束（复用 GDD-012，接现有忠诚/凝聚）。**
双方各持**补给度**（整数）。守住己方粮仓→逐段回升；粮仓被焚/被断→下滑。**补给充足→单位逐段少量回血；补给告急→① 部队叛逃减员（接 GDD-006/忠诚·凝聚）② 该方单位战斗伤亡按倍率放大（强攻数倍伤亡，接 GDD-011 士气疲劳）**。粮仓与补给线成为**地图战略目标**（落地"绕后断粮"）。

**D8 半路遭遇决策点 + 反全知 + 防退化护栏。**
- **遭遇门**：行军中单位与敌新相邻 → 暂停给玩家抉择（继续突进/就地据守/后撤避战）——"当下参与感"的核心接缝。
- **反全知**（GDD-007）：玩家对敌只见情报投影（位置/兵力估计随侦察时效），隐蔽区（林中藏兵）未被侦破不入对方可见面。类型层玩家可见结构不含敌真值。
- **防退化护栏（回应 ADR-0012 顾虑）**：① **无兵种克制三角减益**（延续 ADR-0011 兵种杠杆非克制）——兵种差异体现在速度/射程/涌现资格，非石头剪刀布；② 差异化仍在**地形+时机涌现的兵法**（伏击/火攻/断粮），非单纯格子微操；③ 胜负由"布势+补给+时机+地形利用"决定，非数值堆叠。

**D9 确定性 + 会话装配 + 存档（复用 ADR-0004/0009/0005）。**
整数/定点、种子化 AI（ADR-0006/0013 注入点）、同种子+同命令序→同状态哈希、可回放、可**战中存档续战**。战斗由 Application（ADR-0009 装配脊梁）从出征（GDD-019 六维准备→初始部署/补给/兵种输入）或守城发起、纳入日界全局序与统一存档信封（ADR-0005）。

## Consequences（后果）

**正面**
- 战斗获得**空间/当下参与感**：行军、半路遭遇决断、诱敌入隘、绕后断粮、择夜突袭——地形与时机成为真策略。
- 仍确定性可测（整数/定点 + 种子 → 同哈希）、可回放、可存档续战。
- **复用而非重造**：补给（GDD-012）、士气疲劳（GDD-011）、兵法涌现（GDD-010）、时间（GDD-001）、AI（ADR-0006/0013）、存档（ADR-0005）、六维准备（GDD-019 作为初始输入）全部接回，兵法/忠诚/凝聚等既有 Domain 可挂接。

**负面 / 代价**
- **取代 ADR-0012**：命名区域引擎（`ZoneBattle*`）与其战斗侧测试大半作废/迁移；换皮 M1-M3 呈现被格子取代（部分 UI 概念可复用）。工期周级、分期。
- Domain 面更大（网格/寻路/移动/补给/涌现/段结算/命令）→ 需充分单测（确定性/寻路/涌现门/补给曲线/段序/存档）。
- 引入坐标 → 须以 D8 护栏持续防"退化为通用 SLG 微操"；平衡面更大（速度/射程/补给曲线/伏击倍率）。
- **2.5D 呈现**为独立且较重的美术/渲染工作量（等距投影 + 深度排序），列为呈现分期，不阻塞 Domain。

## Alternatives Considered（备选）

- **A 维持 ADR-0012 命名区域（换皮加强）**：成本低但用户实玩判定参与感不足、"更像棋盘" → 否决。
- **B 连续实时（RTS）**：过重、且与回合确定性/存档续战冲突 → 否决。用户明确要**按时间段推进（回合式）**，非连续实时。
- **C 本决策：整数格子 + 时间段确定性状态机 + 地形/时机涌现 + 补给硬约束** → 采纳（原型验证手感通过）。

## Engine Compatibility（引擎兼容）

纯 C# Domain，无 `UnityEngine`，`dotnet test`（NUnit）可验。整数坐标、乘子用整数百分比/定点 Q16.16、无 float 权威路径（ADR-0004）。地图/兵种/补给配置经构建期不可变配置管线（ADR-0003）。Presentation 薄壳只读投影渲染（Unity 6.3 LTS UI Toolkit / 后续 2.5D 等距）。

## Compliance / Guardrails

- 纯函数段推进、无随机、整数/定点（ADR-0004）；AI 随机走 ADR-0013/0006 注入流。
- 负向不变量（断言）：**无兵种克制三角减益**（延续 ADR-0011）；差异化在地形/时机涌现，非纯格子数值微操。
- 反全知：类型层玩家可见面不含敌真值；林中藏兵双向不互见直到侦破。
- 命令前置校验、稳定错误码、无部分写入。
- 装配层不拥规则（ADR-0009）：战斗规则在 Domain，发起/存档编排在 Application。
- 坐标为整数格；乘子（伏击/夜袭/补给伤亡）用整数百分比，不得引入 float 到状态哈希路径。
