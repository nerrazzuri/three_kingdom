# ADR-0016 — 武将归属层与全局融入契约

- **状态**：Accepted（2026-07-06）
- **落地**：GDD_027（武将归属·任用·全局融入）
- **关联**：ADR-0002（分层）· ADR-0003（数据驱动）· ADR-0004（确定性整数/定点）· ADR-0008/0010（城控制权）· GDD_025/026

## 背景

500 员武将此前只被「施计画像」与「地图棋子」消费，未进战斗阵容、招揽、内政、军师、关系、事件——是「数据库」而非「玩法」。需让武将成为贯穿全局的 NPC 连接层，且**不依赖任何具体场景/剧本**（横向解耦，可与任意纪元盘配合）。

## 决策

### D1 归属为「(武将 × 年) → 在职/在野」的纯函数派生
新增薄数据 `baseFaction`（一将一势力标签）；`AffiliationOf(将, 年)` 由 `baseFaction + 生卒(GDD_026) + WorldAt(年)` 纯函数派生，**无场景依赖、确定性、可复现**。`baseFaction` 未登记 → 默认在野（中立可招）。著名叛降由**少量纪元覆盖数据**校正，不写引擎分支。

### D2 组织单位为「城武将册」，上限 20，太守调拨
城以武将册（≤20）为组织单位。NPC 城由归属派生填充并按 `max(战阵档,谋略档)` 裁剪到 20；玩家城由太守显式调拨。内政官/军师/军队主将副将**皆从城册点选**。军队编制固定 **1 主将 + 可选 1 副将**，允许多军并进。

### D3 归属层置于 Application/Scenarios，消费方各自查询（依赖倒置）
归属/角色/册派生为 Application 层纯服务（Domain 数据 + WorldAt 盘），**六消费方（招揽/内政/军师/关系/战斗/事件）各自按需查询，互不耦合**；归属层不反向依赖任何消费方。表现层只读投影（ADR-0002）。任用（玩家指派）为经 Command/Service 的授权状态，纳入存档。

### D4 确定性与存档
派生纯函数（同盘同年同结果）；任用状态（玩家城册指派）为权威态，纳入版本化存档 + 状态哈希（ADR-0004/0005）。角色/难度/加成均整数/定点，禁 float 权威路径。

### D5 分期落地、每阶段独立可测
P1 地基 → P2 招揽 → P3 任用 → P4 内政 → P5 军师 → P6 关系 → P7 战斗 → P8 事件 → P9 表现层。前 8 阶段纯 C# 无场景依赖、单测覆盖；P9 表现层 ★编辑器验证。事件引擎（P8）已由招牌试水系统化为完整引擎（见 D6）。

### D6 演义事件覆盖层（P8 完整）：可推演不入档，优先于 baseFaction 派生
事件引擎（`LoreEvents`）登记招牌关目表（桃园/温酒/三英/三顾/官渡·许攸/白门楼/赤壁/水淹/走麦城/五丈原…），每桩锚定具名武将 + 纪元/玩家势力触发，产出 typed 效果 `LoreEffect`：**登场（Introduce）/ 移籍（Reassign）/ 斩杀（Slay）**。效果由 `OverridesAt(ctx)` 逐年扫描 [AnchorYear, CurrentYear] 确定性重放累积成 `LoreOverrides`，被 `GeneralAffiliations.AffiliationOf` **优先消费**（斩杀→Absent；移籍→换势力，覆盖优先序高于 baseFaction）。
关键性质：触发只依赖**已持久化态**（纪元盘/当前年/玩家势力/在世门），故覆盖层**可推演、不入存档**——读档时 `OverridesAt` 重算即复原，无需新 DTO 字段/迁移（沿用 ADR-0015「发觉门派生自持久化态」范式）。纯函数、幂等，满足 ADR-0004 确定性。`AffiliationOf`/`RosterOf` 以可选 `LoreOverrides` 参数接入（默认 null = 原派生行为，零破坏面）。

## 后果

**正面**：500 武将转为玩法杠杆；横向解耦，任意纪元/剧本复用；纯函数可测；不锁场景。
**代价**：`baseFaction` 是简化（跳槽者需纪元覆盖，接受近似；部分由 D6 移籍效果在事件时点纠正）；城册裁剪可能漏掉次要史实布防；任用入存档增加存档面（v1.0→需迁移，逆序链首次演练）。事件覆盖层已完整落地（D6），触发于纪元锚点年当轮即生效——运行期各消费方（croster/gov/council/army）是否透传 `LoreOverrides` 及「锚点年当轮 vs 推进后」触发时机为独立集成/表现层调优线（★需运行期验证），非引擎缺口。

## 备选（未采纳）

- **垂直切片**（只做一个剧本做深）：被否——武将融入应横向解耦，不锁单场景（用户裁定）。
- **每回合 500 将常驻 agent 模拟**：被否——太重；归属按需派生即可，不做常驻。
