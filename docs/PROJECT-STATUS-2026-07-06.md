# 项目现状与质量把控报告 — 三国策略游戏

**日期**：2026-07-06 ｜ **引擎**：Unity 6.3 LTS ｜ **语言**：C# ｜ **分支**：main ｜ **提交数**：288

> 本报告供质量分析师把控。**如实陈述**已做/未做/停泊/存疑，不粉饰。结论：**引擎/系统/内容层完成度高且工程纪律强，但视觉/体验/调优层尚早期**——是一个「文本可通关、可玩性完整、但美术与手感未做」的阶段。

---

## 1. 这是什么游戏

单机、回合制、确定性的三国策略模拟。玩家为「空降者」，在某历史纪元（184~234）选一势力/城起家，走 **太守→晋升→自立→争霸→统一** 的生涯；核心玩法是**多维出征准备 + 命名区域战场的兵法沙盒战斗 + 人心杠杆（离间/策反/攻心）+ 反全知情报**。设计立意：**不靠数值面板与胜率条，靠名声、性情、情报与条件成型**取胜。

## 3. 工程质量信号（强项）

- **四层架构严格**（ADR-0002）：Domain **零 `UnityEngine` 引用**（已核）；Presentation 只读投影，玩家操作必经 Command/Service。
- **确定性**（ADR-0004）：权威结算用整数/定点 Q16.16 + 注入式种子随机（非 float 掷骰）；状态哈希；同种子同结果有测试。
- **数据驱动**（ADR-0003）：平衡数值出自版本化配置，非硬编码（设计约束）。
- **存档**（ADR-0005）：显式版本化 DTO + 原子写入；round-trip 状态哈希一致有测试；当前 schema v1.0。
- **测试纪律**：每系统有单测；关键契约（时间推进确定性、战役结算、存读档、配置校验、Command 前置校验）均覆盖。
- **反全知一致性**：情报、人才、战略图棋子都贯彻「只给估计/隐藏可发觉」，设计立意落地一致。

## 6. 可玩性现状

- **文本控制台（`dotnet run --project src/Console`）**：**完整游戏循环可玩** —— 选纪元选城/推进/治理/情报军议/君命/出征全流程/守城/施计/自立/传承/存读档/战略图/武将录。这是当前最完整的可玩形态。
- **Unity（11 屏，UI Toolkit 灰盒）**：主菜单 / 开局选择 / HUD / 武将录 / 外交 / 多城战区 / **战略图** / 区域战斗 / 暂停 / 无障碍设置 / 战败。已编辑器验证：11 开局可选、纪元公元年正确、反全知战略图雾正确、屏间导航通。**均为无美术的功能灰盒**。

## 8. 诚实短板与风险（质量把控重点）

### 高优先（影响「是不是个游戏」的体验）
- **零美术**：所有 Unity 屏是灰盒 UI Toolkit；无立绘/图标/地图图形/动画；多屏背景透出天空盒（面板非不透明底）。**「战略大地图（DOTween 立绘版）」scaffold 仍停泊 `parked/campaign-map-scaffold`**（缺 Input System/URP/DOTween 依赖）——当前战略图是功能性列表，非视觉地图。
- **无音频、无叙事文本**：系统在，但过场/对白/事件文案基本未写。
- **平衡未调**：数值为占位，**无 playtest / balance 报告**。战斗/经济/招揽的手感与曲线未经打磨。

### 中优先（工程/质量）
- **Unity 屏无自动化测试**：view model（纯 C#）有测，但 **UXML 绑定/场景装配仅靠截图人工验证**（项目标准里 UI 属 ADVISORY 门禁）。回归风险靠每次重跑 computer-use。
- **内容广但布防非穷尽**：500 武将，但各纪元地图**布防是策展子集**（每纪元约 5~25 城具名，已加密但未铺满 36 城）；世界盘 36 城是真实 ~100+ 城的简化。
- **人才招揽只接 3 原型**：反全知招揽机制完整，但未推广到 500 谱（地图有反全知雾，但「从地图直接招揽某将」未接）。
- **存档迁移链未经实战**：schema 恒为 v1.0，**逆序迁移链尚无第二版本可验**（架构在，未演练）。
- **`float`/`double` 出现在 9 个 Domain 文件**：需审计是否触碰权威结算路径（ADR-0004 只允许非权威 Presentation 用 float）。**建议分析师核这 9 处**。

### 低优先（已知、可接受）
- **无性能/soak 证据**：预算已定（60fps / 8GB），但未实测 profile。
- **单机**：无多人（设计即单机）。
- **敌方 AI 深度**：架构一次到位，深度仍在迭代（可能显得「不够聪明」）。
- **发觉标记**：已入存档（派生自持久化侦察态），但地图反全知的「发觉」渠道目前只有「占城 + 派探目标势力」两条。

## 9. 建议分析师优先审查

1. **玩起来是不是好玩**：拿控制台跑 2~3 个纪元通关，评估**循环乐趣与手感**（这是自动化测试照不到的盲区）。
2. **战斗与经济的平衡曲线**：是否存在支配性策略、是否有意义的取舍。
3. **`float`/`double` 的 9 处 Domain 用法** 是否违反确定性红线。
4. **Unity 屏的真实可用性**：11 屏在无美术下是否已能构成「可理解的游戏」，还是必须先补美术才能评估。
5. **内容的历史正确性抽检**：500 武将的生卒/归属/羁绊、8 纪元的城池归属是否经得起考据。
6. **「从数据到体验」的最短路径**：当前最大瓶颈是美术/地图可视化，评估**接回 parked 地图 scaffold vs 继续用 UI Toolkit 灰盒**的投入产出。

---

## 10. 一句话结论

**内核扎实、系统齐全、内容丰富、工程纪律强、可文本通关**；**但它现在是一台"跑得起来的引擎 + 一副骨架"，还不是一款"能给玩家看的游戏"**——差的主要是美术、音频、叙事文案与平衡打磨，以及把 500 武将/8 纪元的深度真正接进可视可玩的 Unity 表现层。
