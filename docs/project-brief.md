# 《三国演义：兵法沙盒》项目简报

## 文档状态

- 阶段：项目初始化
- 基线日期：2026-06-21
- 默认技术：Unity + C#
- 产品形态：离线单机三国沙盒战略 RPG
- 当前实现许可：仅文档与规划；禁止进入 gameplay、UI 或 Scene 实现

## 一句话定位

玩家以一名身处三国乱世的武将度过人生、经营关系与势力，并通过侦察、地形、天气、时机、后勤和人心亲手创造兵法成立条件。

## 核心体验

本项目不是把计策包装成技能按钮，而是让玩家理解局势、改变条件、承担代价并利用结果。战争由人物、城市、外交、关系和政治共同塑造；战果也必须反向改变这些系统。

## 产品支柱

1. 人物人生系统
2. 关系与派系系统
3. 城市经营与政治系统
4. 外交与天下局势系统
5. 兵法沙盒战斗系统

## 目标玩家

- 喜欢三国人物与历史情境，而不只追求战斗数值的玩家。
- 喜欢通过信息、准备和取舍解决问题的策略玩家。
- 接受失败、撤退、失城和改投等可持续叙事结果的沙盒玩家。

## 核心产品承诺

- 兵法由可观察、可干预、可组合的世界条件产生。
- 军师提供信息整理、风险判断和候选思路，不替玩家执行最优解。
- 非战斗行为能改变战争条件，战争结果能改变人物与天下局势。
- 失败生成新局面，而不是轻易毁档或强迫读档。
- 所有规则可配置、可测试、可追踪、可存档并可演进。

## 阶段门禁

进入 gameplay 实现前必须依次完成并审查：概念文档、核心系统地图、MVP 边界、首批 GDD、跨系统矛盾报告、技术架构、控制清单、相关 method specs 与 vertical slice 计划。

## 当前成功标准

`/start` 完成的标准是建立可追踪的文档基线，明确项目方向、范围、架构边界与后续门禁；不以可运行游戏或 Unity 工程为交付物。
